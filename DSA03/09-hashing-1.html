01Introduction & Motivation

The Hotel Analogy

Imagine a hotel with 5 rooms. To track which rooms are occupied, you can use a simple boolean array indexed by room number. But if room numbers can range from 1 to 10⁹ (arbitrary IDs like 5588, 9955, 25…), a boolean array of size 10⁹ is impractical. The solution is a HashMap.

Small hotel — boolean array works

Room 2Occupied
Room 3Not occ.
Room 4Not occ.
Room 5Not occ.

Arbitrary IDs — HashMap needed

5588Occupied
25Not occ.
35Not occ.
45Occupied
9955Occupied
Core Idea A HashMap (also called a map or dictionary) stores key–value pairs and provides O(1) average-time access by key — regardless of the key's magnitude or type. It eliminates the need to allocate a huge array indexed by value.

Two Core Data Structures

🗂️
HashMap
Stores key → value pairs. Keys are unique; values can be anything. Use when you need to associate data with a key (e.g., element → count).
🧩
HashSet
Stores only keys (no associated value). Use when you only need to track existence or uniqueness (e.g., "have I seen this element before?").

02HashMap Use Cases — Choosing the Right Type

The generic type parameters of HashMap encode exactly what you're storing:

Example 1 →
Population of each country
India→145M, USA→65M, Russia→14M, China→147M
HashMap<String, Long> map;
Example 2 →
Number of states per country
India→29, USA→50, China→25, Russia→21
HashMap<String, Integer> map;
Example 3 →
Names of all states
India→[Andhra Pradesh, U.P, M.P, Karnataka, …], USA→[New York, Texas, …]
HashMap<String, List<String>> map;
Example 4 →
Population of each state in each country
India→{Karnataka→50M, Maharashtra→72M, …}, USA→{Texas→18M, …}
HashMap<String, HashMap<String, Long>> map;
Rule Keys must be unique. If you add a key that already exists, the value is overwritten. Values can be any type — primitives, objects, lists, or even other HashMaps.

03Operations & API — All O(1) Average

Every standard HashMap and HashSet operation runs in O(1) average time due to the underlying hash table. (Worst case is O(N) due to hash collisions, but this is rare with a good hash function.)

HashMap — Key–Value Pairs
O(1)map.put(key, value) — add / overwrite
O(1)map.get(key) — retrieve value
O(1)map.containsKey(key) — search
O(1)map.remove(key) — delete
O(1)map.size() — count of entries
O(1)map.getOrDefault(k, 0) — safe get
HashSet — Keys Only
O(1)set.add(key) — insert
O(1)set.remove(key) — delete
O(1)set.contains(key) — search
O(1)set.size() — count of elements
getOrDefault Pattern In Java, map.getOrDefault(key, 0) returns 0 if the key is absent, avoiding a NullPointerException. This is idiomatic for frequency counting: map.put(x, map.getOrDefault(x, 0) + 1);

Brute Force — O(Q × N)

For every query element, scan the entire array and count occurrences. With Q = 10⁵ and N = 10⁶ this is 10¹¹ operations — far too slow.

TimeO(Q × N) SpaceO(1)

Optimized — Build a Frequency Map O(N + Q)

Pre-process the array once into a frequency map (element → count). Then answer each query in O(1) with a simple lookup.

Queries: 2→3, 8→3, 3→2

Java — Frequency Map Approach
HashMap<Integer, Integer> map = new HashMap<>();

// Phase 1: Build frequency map — O(N)
for (int i = 0; i < N; i++) {
    map.put(arr[i], map.getOrDefault(arr[i], 0) + 1);
}

// Phase 2: Answer queries — O(Q), each lookup O(1)
for (int i = 0; i < Q; i++) {
    int element = queries[i];
    System.out.println(map.getOrDefault(element, 0));
}
TimeO(N + Q) SpaceO(N)

06Count of Distinct Elements

Problem Given an array of N integers, return the count of distinct (unique) values.

Examples:

[3, 5, 6, 5, 4]  →  4 distinct (3, 5, 6, 4)
[3, 3, 3]  →  1 distinct
[1, 1, 1, 2, 2]  →  2 distinct (1, 2)

A HashSet automatically enforces uniqueness — adding a duplicate is silently ignored. So simply add all elements and return the set's size.

Java — Distinct Count
HashSet<Integer> set = new HashSet<>();

for (int i = 0; i < N; i++) {
    set.add(arr[i]);   // duplicates silently ignored
}

return set.size();   // = number of distinct values
TimeO(N) SpaceO(N)

07Longest Substring Without Repeating Characters

Problem — "GOODness" of a String Given a string A, find the length of the longest substring that contains no repeating characters. Solve in O(N) time.

Input 1: "abcabcbb" → 3 ("abc")
Input 2: "AaaA" → 2 ("Aa")

Brute Force — O(N³)

Consider every possible substring (O(N²) pairs of start/end indices), and for each one check all characters for duplicates (O(N) check). Total: O(N³).

TimeO(N³)

Optimized — Sliding Window "Acquire & Release" — O(N)

Use two pointers i (right expansion pointer) and j (left shrink pointer), plus a HashSet representing the current window's characters. The algorithm:

  • Acquire: Try to expand the window by moving i right. If str[i] is not in the set, add it — the window grows.
  • Release: If str[i] is already in the set (duplicate!), shrink from the left: remove str[j] from the set and increment j. Keep releasing until the duplicate is gone.
  • After each expansion, update maxLength = max(maxLength, i − j + 1).

Visual Trace — "abcdmdcmopqrmm"

The longest window without repeats is "abcdm" → length 5, then "dcmopqr" → length 7.

When i reaches the second 'd' (index 5), it's already in the set. Release from j until 'd' is removed, then acquire again.

Java — Sliding Window O(N)
HashSet<Character> set = new HashSet<>();
int maxLength = 1, j = 0;

for (int i = 0; i < N; i++) {
    char ch = str.charAt(i);

    if (set.contains(ch)) {
        // RELEASE: shrink from left until duplicate is removed
        while (str.charAt(j) != ch) {
            set.remove(str.charAt(j));
            j++;
        }
        set.remove(str.charAt(j));  // remove the duplicate itself
        j++;
    }

    // ACQUIRE: expand window to include ch
    set.add(ch);
    maxLength = Math.max(maxLength, i - j + 1);
}

return maxLength;
TimeO(N) SpaceO(N)
Why O(N)? Each character is added to the set at most once (when i passes it) and removed at most once (when j passes it). Both i and j only move forward. So the total number of set operations is at most 2N, giving O(N) overall.

08First Non-Repeating Element

Problem Given N elements, find the first element that appears exactly once. If no such element exists, return −1.
[1, 2, 3, 1, 2, 5] → 3 (first element with freq = 1)
[4, 3, 3, 2, 5, 6, 4, 5] → 2
[4, 1, 1, 4] → −1 (all elements repeat)

Brute Force — O(N²)

For every element, scan the entire array to count its occurrences. Return the first element with count 1.

Optimal — Frequency Map O(N)

Two passes: build a frequency map in pass 1, then iterate the original array in pass 2 and return the first element with frequency 1.

Java — First Non-Repeating
HashMap<Integer, Integer> map = new HashMap<>();

// Pass 1: build frequency map
for (int i = 0; i < N; i++) {
    map.put(arr[i], map.getOrDefault(arr[i], 0) + 1);
}

// Pass 2: find first element with frequency 1
for (int i = 0; i < N; i++) {
    if (map.get(arr[i]) == 1) {
        return arr[i];
    }
}
return -1;  // no non-repeating element
TimeO(N) SpaceO(N)
Why Two Passes? The first pass is needed to know each element's total frequency before declaring anything "non-repeating." You can't decide in a single pass because a later occurrence might invalidate an earlier first-time element.

09Key Ordering in HashMap

A common trap: iterating over a HashMap does not give keys in insertion order or sorted order. Keys appear in arbitrary (hash-determined) order.

arr = [5, 2, 7, 3, 6]

You might print GCD of elements in order 1, 7 — or any permutation.

⚠ Keys will be present in random order in HashMap.

🔀
HashMap (Java)
Unordered. Use when iteration order doesn't matter and you only care about fast lookup.
📋
LinkedHashMap (Java)
Preserves insertion order. Use when you need to iterate in the order elements were first added.
📊
TreeMap (Java)
Keys sorted in natural order. O(log N) operations. Use when sorted iteration is required.

★Quick Recap

HashMap
Key→value store. Keys unique. All core ops (put, get, containsKey, remove) are O(1) average.
HashSet
Keys-only store. Automatically ignores duplicates. add/remove/contains all O(1). size() = distinct count.
Frequency Map
Build in O(N), then answer Q queries in O(1) each. Total O(N+Q) vs brute O(Q×N).
Distinct Count
Add all elements to a HashSet; return set.size(). O(N) time and space.
Sliding Window
Two pointers i and j. Acquire (add to set) on right; Release (remove from set) on left when duplicate found.
First Non-Repeating
Two passes: build fmap, then scan array returning first element with fmap[x] == 1. O(N) time/space.
Key Ordering
HashMap = unordered. LinkedHashMap = insertion order. TreeMap = sorted order (O(log N) ops).
getOrDefault
map.getOrDefault(k, 0) safely returns 0 for absent keys. Idiomatic for freq counting.
