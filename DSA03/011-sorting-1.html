01Count Sort — Smallest Number from Digits

Problem Given an array of digits (0 ≤ digit ≤ 9), rearrange them to form the smallest possible number. Return the result as a sorted digit array.

[6,3,4,2,7,2,1] → [1,2,2,3,4,6,7]
[4,2,7,3,9,0] → [0,2,3,4,7,9]

Idea 1 — Use Inbuilt Sort

Simply call the standard sort on the array. The sorted order is the smallest arrangement.

TimeO(N log N) SpaceO(depends on algo)

Idea 2 — Count Sort O(N)

Since digits are limited to 0–9 (range = 10), we can exploit this small range. Instead of comparing elements, count how many times each digit appears, then reconstruct the sorted array by writing each digit value its count number of times.

arr = [9, 1, 2, 5, 4, 2, 1, 2, 5, 8]

Java — Count Sort (digits 0–9)
int[] farr = new int[10];

// Phase 1: Count each digit
for (int i = 0; i < N; i++) {
    farr[arr[i]]++;
}

// Phase 2: Reconstruct sorted array
int k = 0;
for (int i = 0; i <= 9; i++) {
    for (int j = 0; j < farr[i]; j++) {
        arr[k] = i;
        k++;
    }
}
TimeO(N + 10) = O(N) SpaceO(1) — farr is constant size
Limitation Count sort only works when the value range R is small. If values can be up to 10⁹, allocating farr[10⁹] would require gigabytes of memory — completely impractical. Use count sort only when the range is small (e.g., digits 0–9, scores 0–100).

02Count Sort — Handling Negative Numbers

When the array contains negative numbers, we can't use a digit index directly. The fix: compute the min and max, allocate a frequency array of size max − min + 1, and use val − min as the index (shifting all values so the minimum maps to index 0).

arr = [5, 2, −3, 4, −2, 9, −1, 2]  |  min=−3, max=9  |  range = 9−(−3)+1 = 13

Java — General Count Sort (with negatives)
// Step 1: Find min and max
int min = arr[0], max = arr[0];
for (int i = 1; i < N; i++) {
    min = Math.min(min, arr[i]);
    max = Math.max(max, arr[i]);
}

// Step 2: Build frequency array of size (range)
int R = max - min + 1;
int[] farr = new int[R];
for (int i = 0; i < N; i++) {
    farr[arr[i] - min]++;   // shift by -min to get non-negative index
}

// Step 3: Reconstruct
int k = 0;
for (int i = min; i <= max; i++) {
    for (int j = 0; j < farr[i - min]; j++) {
        arr[k++] = i;
    }
}
TimeO(N + R) SpaceO(R)

where R = max − min + 1 is the range.

03Merge Two Sorted Arrays

Problem Given two sorted arrays a[N] and b[M] (elements may be up to ±10⁹), merge them into a single sorted array of size N+M.

a = [2, 4, 7, 8, 12]   b = [3, 5, 6, 7]   →   [2, 3, 4, 5, 6, 7, 7, 8, 12]

Idea 1 — Copy & Sort

Copy both arrays into a result array of size N+M, then sort it. Simple but wastes the fact that both inputs are already sorted.

TimeO((N+M) log(N+M))

Idea 2 — Two-Pointer Merge O(N+M)

Use pointer i for array a and pointer j for array b. At each step, pick the smaller of a[i] and b[j], place it in result array c, and advance the corresponding pointer. When one array is exhausted, copy the remaining elements of the other.

a = [2, 4, 7, 8, 12]   b = [3, 5, 6, 7]

i→2 vs j→3: pick 2.   i→4 vs j→3: pick 3.   i→4 vs j→5: pick 4.   …   b exhausted, copy [8,12].

Java — Two-Pointer Merge
int[] c = new int[N + M];
int i = 0, j = 0, k = 0;

// Main comparison loop
while (i < N && j < M) {
    if (a[i] <= b[j]) { c[k++] = a[i++]; }
    else               { c[k++] = b[j++]; }
}

// Copy remaining elements of a (if any)
while (i < N) { c[k++] = a[i++]; }

// Copy remaining elements of b (if any)
while (j < M) { c[k++] = b[j++]; }

return c;
TimeO(N + M) SpaceO(1) extra (output excluded)
Why ≤ Not < in the Comparison? Using a[i] <= b[j] (not strict <) means when elements are equal, we take from a first. This preserves the relative input order among equal elements — a key property for stable sorting.

04Merge Sort — Divide & Conquer

Merge sort is a divide-and-conquer algorithm. It recursively splits the array in half, sorts each half independently, then merges the two sorted halves using the two-pointer merge from Section 03.

The Three Steps

✂️
Divide
Find mid = (lo + hi) / 2. Split array into arr[lo..mid] and arr[mid+1..hi]. Recurse on each half.
🔁
Base Case
If lo == hi (single element), return immediately. A single element is trivially sorted.
🔗
Conquer (Merge)
Both halves are now sorted. Merge them in O(N) using the two-pointer merge step.
Java — Merge Sort
void mergeSort(int[] arr, int lo, int hi) {
    if (lo == hi) return;   // Base case: single element

    int mid = (lo + hi) / 2;
    mergeSort(arr, lo, mid);           // Sort left half
    mergeSort(arr, mid + 1, hi);      // Sort right half
    merge2SortedSubarrays(arr, lo, mid, hi);  // Merge both halves O(N), O(N)
}

05The Merge Step — merge2SortedSubarrays

Both halves arr[lo..mid] and arr[mid+1..hi] are already sorted by the recursive calls. We merge them in-place using a temporary array, then copy the result back.

Java — merge2SortedSubarrays
void merge2SortedSubarrays(int[] arr, int lo, int mid, int hi) {
    int[] temp = new int[hi - lo + 1];   // temp array of size = window
    int i = lo, j = mid + 1, k = 0;

    // Compare and pick smaller from each half
    while (i <= mid && j <= hi) {
        if (arr[i] <= arr[j]) { temp[k++] = arr[i++]; }
        else                   { temp[k++] = arr[j++]; }
    }

    // Copy remaining elements of left half
    while (i <= mid)  { temp[k++] = arr[i++]; }

    // Copy remaining elements of right half
    while (j <= hi)   { temp[k++] = arr[j++]; }

    // Write temp back into arr[lo..hi]
    for (int x = 0; x < temp.length; x++) {
        arr[lo + x] = temp[x];
    }
}
TimeO(hi − lo + 1) = O(N) SpaceO(N) for temp array

06Time Complexity Analysis of Merge Sort

The recursion tree has log₂N levels (since each level halves the problem size). At every level, the total work across all merges is exactly N elements processed. Therefore:

Total work = (work per level) × (number of levels) = O(N) × O(log N) = O(N log N)
LevelSubproblemsSize eachTotal work
L01NN
L12N/2N
L24N/4N
⋮⋮⋮N
L(log N)N1N
Total levels = log₂NN × log₂N
TimeO(N log N) SpaceO(N)
Space — Why O(N)? The temp array allocated inside each merge call is not all live simultaneously. At any point, only the merge calls on the current root-to-leaf path are active, and the total size of their temp arrays is O(N). The space is bounded by the maximum stack depth times the size of the merge window at each level.

07In-place & Stable Sort

In-place Sorting

A sorting algorithm is in-place if it uses only O(1) extra memory (beyond the input array). Standard merge sort uses O(N) extra space for the temporary merge array — it is not in-place.

Stable Sorting

A sorting algorithm is stable if elements that compare as equal maintain their original relative order. This matters when sorting complex objects by one key — equal keys should preserve original input ordering.

Homework Is standard Merge Sort stable or not? Think about the comparison in the merge step: arr[i] <= arr[j] takes the left element when equal — this preserves the original relative order. Answer: Merge Sort IS stable (as long as the ≤ comparison is used).

08Real-world Application — Gmail All Inboxes

Scenario Google's Gmail "All Inboxes" feature merges email timestamps from multiple accounts into a single chronological feed. Each account's emails are already sorted by time — so the problem is exactly merge two sorted arrays.

Given two sorted timestamp arrays A and B from two email accounts, produce a merged array sorted in ascending order — so the user sees all emails in chronological order.

A = [1, 5, 6, 9]   B = [2, 4, 8]

Merged = [1, 2, 4, 5, 6, 8, 9]

Use the two-pointer merge from Section 03 directly.

Extension — K sorted arrays If there are K email accounts (K sorted arrays), the generalization uses a min-heap of size K. Pop the minimum, add it to the result, push the next element from the same array. Total time O(N log K) where N is the total number of emails.

★Quick Recap

Count Sort
Build frequency array farr[R], then reconstruct. O(N+R) time, O(R) space. Only practical when R (range) is small.
Negative Offset
For negatives: range = max−min+1. Index = val−min. Shift values up by |min| to make all indices non-negative.
Merge Two Sorted
Two pointers i and j. Pick smaller, advance its pointer. Copy remainder. O(N+M) time, O(1) extra space.
Merge Sort
Divide (split at mid), recurse both halves, merge. T(N) = 2T(N/2) + O(N) → O(N log N) time, O(N) space.
In-place
Sorting algorithm uses O(1) extra memory. Merge sort is NOT in-place (uses O(N) for temp array).
Stable Sort
Equal elements preserve their original relative order. Merge sort IS stable when using ≤ in the comparison.
