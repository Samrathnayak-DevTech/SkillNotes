K=6 → true (1+5).   K=22 → false (11+11 needs same element twice).   K=8 → true (9+−1? no, but −2+10? no; actually 9+−1? Let's check: no 10 in array. Wait — the notes say K=8 true. That's 4+4 at indices 4 and 8.) ✓

Brute Force — O(N²)

Check all pairs (i, j) with i < j.

Java — Brute Force
for (int i = 0; i < N; i++) {
    for (int j = i + 1; j < N; j++) {
        if (arr[j] == K - arr[i]) return true;
    }
}
return false;
TimeO(N²) SpaceO(1)

Why "Build Set First" Fails for K = 8 with arr = [4, 4, …]

If you add all elements to a set first then check set.contains(K − arr[i]), a set with a single 4 would report K=8 as true even when only one 4 exists — because the set can't distinguish "is there a different element equal to 4?" from "is 4 anywhere in the set?"

Optimized — HashSet "On the Go" — O(N)

Check for the complement before adding the current element. This ensures we only match with elements seen earlier — never with the element itself.

Java — HashSet On the Go
HashSet<Integer> set = new HashSet<>();

for (int i = 0; i < N; i++) {
    // Check complement BEFORE adding current element
    if (set.contains(K - arr[i])) return true;
    set.add(arr[i]);
}
return false;
TimeO(N) SpaceO(N)
The "On the Go" Pattern Always check then add. The set at step i contains only elements at indices 0 through i−1. So a successful match guarantees two distinct indices. This is the canonical O(N) pair-detection pattern.

Pairs summing to 10: (2,8) and (8,2) combos. Frequency map: 2→3, 5→3, 8→2.   When we see a 2 at index 2, we need an 8 — there are 0 eights in map yet. When we see 8 at index 4, complement is 2 — there are 2 twos already → ans += 2. And so on. Total = 9.

HashMap "On the Go"

For each element, its complement is K − arr[i]. If the complement is already in the map, all previous occurrences of that complement form valid pairs with the current element. Add map.get(K − arr[i]) to the answer.

Java — Count Pairs with Sum K
HashMap<Integer, Integer> map = new HashMap<>();
int ans = 0;

for (int i = 0; i < N; i++) {
    // Count how many previous elements form a valid pair
    int complement = K - arr[i];
    if (map.containsKey(complement)) {
        ans += map.get(complement);
    }

    // Add current element to map AFTER checking
    map.put(arr[i], map.getOrDefault(arr[i], 0) + 1);
}
return ans;
TimeO(N) SpaceO(N)

i=1: arr[1]=5, complement=5, map has no 5 → ans=0, add 5→1.
i=2: arr[2]=2, complement=8, no 8 → ans=0, add 2→2.
i=3: arr[3]=5, complement=5, map has 5→1 → ans=1, add 5→2.

03Subarray with Sum = 0 (Exists?)

Problem Given N elements, check whether any contiguous subarray has sum equal to 0. (1 ≤ N ≤ 10⁵)

Brute Force — O(N³) or O(N²)

Enumerate all subarrays (O(N²) start/end pairs) and compute each sum from scratch (O(N)) → O(N³). Using prefix sums reduces to O(N²).

Key Insight — Prefix Sum Duplicates

Define the prefix sum psum[i] = arr[0] + arr[1] + … + arr[i]. The sum of subarray arr[j+1..i] equals psum[i] − psum[j]. This sum is 0 when psum[i] = psum[j] — that is, when the same prefix sum repeats.

sum(arr[j+1 .. i]) = psum[i] − psum[j] = 0 iff psum[i] == psum[j] → A repeated value in psum means a zero-sum subarray exists.

Edge Case — Prefix Sum Itself = 0

If psum[i] = 0, the subarray from index 0 to i has sum 0. To handle this uniformly, insert 0 into the set initially before scanning. Then "sum 0 seen before" correctly triggers a match.

Example — arr = [−2, −1, 3, 5] psum = [−2, −3, 0, 5]. At i=2, psum=0. Without 0 pre-inserted, the check "set.contains(0)" is false. With 0 pre-inserted, it correctly returns true because arr[0..2] = [−2, −1, 3] sums to 0.
Java — Subarray Sum = 0
HashSet<Integer> set = new HashSet<>();
set.add(0);   // handle subarrays starting from index 0

int sum = 0;
for (int i = 0; i < N; i++) {
    sum += arr[i];
    if (set.contains(sum)) return true;
    set.add(sum);
}
return false;
TimeO(N) SpaceO(N)

04Subarray with Sum = K (Exists?)

Problem Given an array of N integers and target K, check if any contiguous subarray has sum equal to K. (1 ≤ N ≤ 10⁶)
arr=[2,3,9,−4,1,5,6,2,5], K=11 → true (subarray [2,3,9,−4,1]=11)
arr=[2,3,9,−4,1,5,6,2,5], K=10 → true
arr=[5,10,20,100,105], K=110 → false

Key Insight — Prefix Sum Difference

The sum of subarray arr[j+1..i] = psum[i] − psum[j]. This equals K when psum[j] = psum[i] − K. So for each running prefix sum, check if sum − K was seen before.

sum(arr[j+1..i]) = K iff psum[i] − psum[j] = K iff psum[j] = psum[i] − K = sum − K → Check if (sum − K) is in the set.
Java — Subarray Sum = K
HashSet<Integer> set = new HashSet<>();
set.add(0);   // handle subarrays starting from index 0

int sum = 0;
for (int i = 0; i < N; i++) {
    sum += arr[i];
    if (set.contains(sum - K)) return true;  // key change from sum=0 version
    set.add(sum);
}
return false;
TimeO(N) SpaceO(N)
Sum=0 as a Special Case The subarray-sum-0 problem is just the K=0 case: sum − K = sum − 0 = sum. So the sum=0 version checks set.contains(sum) and the sum=K version checks set.contains(sum − K). One unified solution handles both.

05Longest Subarray with Sum = 0

Problem Given an array of N integers, find the length of the longest contiguous subarray whose elements sum to 0.

Extending the Detection Idea

We know that if psum[i] = psum[j] (with i > j), then the subarray arr[j+1..i] has sum 0 and length i − j. To maximize this length, we want the earliest (smallest j) at which each prefix sum value was seen. Store the first occurrence index in a HashMap.

ans = max(ans, i − map.get(sum)) Insert (0 → −1) initially so that if psum[i] = 0, the subarray arr[0..i] of length i − (−1) = i + 1 is counted.

Java — Longest Subarray Sum = 0
HashMap<Integer, Integer> map = new HashMap<>();
map.put(0, -1);   // virtual index: psum=0 "first seen" before index 0

int sum = 0, ans = 0;

for (int i = 0; i < N; i++) {
    sum += arr[i];

    if (map.containsKey(sum)) {
        // Same prefix sum seen before: subarray arr[map.get(sum)+1 .. i] has sum 0
        ans = Math.max(ans, i - map.get(sum));
    } else {
        // Only store FIRST occurrence — we want the longest span
        map.put(sum, i);
    }
}
return ans;
TimeO(N) SpaceO(N)
Don't Overwrite First Occurrence The else branch is critical: only insert into the map if the key is absent. If you overwrite an existing entry, you'll lose the earliest index and compute a shorter (not longest) subarray. The code checks containsKey and only puts if the key is absent.

★Quick Recap

Pair Sum Exists
HashSet on the go: check set.contains(K−x) THEN add x. Prevents matching an element with itself.
Count Pairs with Sum K
HashMap on the go: ans += map.get(K−x) then increment map[x]. Counts all earlier matching elements.
Subarray Sum = 0
Pre-insert 0 into set. Check if running prefix sum has been seen before. O(N) time and space.
Subarray Sum = K
Same as sum=0 but check set.contains(sum−K). Pre-insert 0. O(N) time and space.
Longest Subarray Sum = 0
HashMap stores first occurrence of each prefix sum. On repeat: update ans = max(ans, i − map.get(sum)). Pre-insert 0→−1.
"On the Go" Pattern
Always check the hash structure FIRST, then add the current element. Prevents self-matching and maintains correct ordering.
