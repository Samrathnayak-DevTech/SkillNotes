01

What Are Intervals?

An interval is a contiguous range defined by a start time a and an end time b, written [a, b]. Two intervals overlap if one starts before the other ends.

Overlap Condition Intervals [a₁, b₁] and [a₂, b₂] overlap when a₂ ≤ b₁ (assuming a₁ ≤ a₂ after sorting by start time). If a₂ > b₁, they are non-overlapping.

Overlap Examples

Edge Case — Touching Intervals Intervals that share exactly one endpoint, like (3, 6) and (6, 10), are considered overlapping. Their merged result is [3, 10]. The condition is a₂ ≤ b₁ (not strictly less than).
02

Merging Two Intervals

Given two intervals [a₁, b₁] and [a₂, b₂] (with a₁ ≤ a₂), the rule is:

java — merge two intervals
// [a1,b1] and [a2,b2], assume a1 <= a2 (sorted by start)
if (a2 > b1) {
    // Non-overlapping: gap between the two intervals
    // Return both intervals unchanged
} else {
    // Overlapping: merge into one interval
    int mergedStart = Math.min(a1, a2);   // always a1 since a1 <= a2
    int mergedEnd   = Math.max(b1, b2);   // take the farther endpoint
}
Analogy Imagine two meetings on a calendar. If meeting B starts before meeting A ends, you can't schedule anything in between — they merge into one blocked-out period. The merged period starts when the earlier meeting starts and ends when the later meeting ends.
03

Merging N Intervals — O(N)

Problem Statement Given a list of intervals already sorted by start time, merge all overlapping intervals and return the sorted list of non-overlapping intervals. (1 ≤ N ≤ 10⁵)

Idea: Track the current merged interval using a running start s and end e. Iterate through every interval. If the next interval overlaps with [s, e], extend e. If it does not overlap, finalize [s, e] and start a new one.

Worked Example

Input: [(0,2), (1,4), (5,6), (6,8), (7,10), (8,9), (12,14)]

java — merge N intervals O(N)
List<int[]> mergeIntervals(int[][] intervals, int n) {
    int s = intervals[0][0];
    int e = intervals[0][1];
    List<int[]> ans = new ArrayList<>();

    for (int i = 1; i < n; i++) {
        if (intervals[i][0] <= e) {          // overlapping
            e = Math.max(e, intervals[i][1]); // extend end
        } else {                              // non-overlapping
            ans.add(new int[]{s, e});        // finalize current
            s = intervals[i][0];              // start new interval
            e = intervals[i][1];
        }
    }
    ans.add(new int[]{s, e});               // add the last interval
    return ans;
}
Time: O(N)
Space: O(1) extra
Prerequisite — Must Sort First This algorithm assumes intervals are pre-sorted by start time. If the input is unsorted, sort it first: Arrays.sort(intervals, (a,b) → a[0] - b[0]). The total complexity with sorting becomes O(N log N).

Quiz

Input (already sorted): [(1,10), (2,3), (4,5), (9,12)]

Trace it: s=1, e=10. All subsequent intervals start ≤ 10, so they all merge. Last interval (9,12) extends e to 12.

Answer: [[1, 12]]

04

First Missing Natural Number

Problem Statement Given an unsorted array of N integers (which may include negatives and numbers larger than N), find the first missing positive natural number (i.e., the smallest positive integer not present in the array).

Examples

ArrayFirst Missing NaturalWhy
[3, −1, 1, 2, 7]41,2,3 present; 4 missing
[9, 2, 6, 4, −8, 1, 3]51,2,3,4 present; 5 missing
[1, 0, −5, −6, 4, 2]31,2 present; 3 missing
[1, 2, 5, 6, 4, 3]71–6 all present
[1, 2, 3, 4]51–4 all present
Key Observation — Bounded Answer The answer always lies in the range [1, N+1]. Why? If the array contains all of 1, 2, …, N, then the answer is N+1. Otherwise, some number in 1…N is missing. This bound is critical for the optimal approach.
05

Approach 1 — Brute Force O(N²)

Idea: Iterate from 1 to N+1. For each candidate, scan the entire array to check if it is present. Return the first candidate not found.

java — brute force O(N²)
int firstMissingBF(int[] arr, int n) {
    for (int candidate = 1; candidate <= n + 1; candidate++) {
        boolean found = false;
        for (int x : arr) if (x == candidate) { found = true; break; }
        if (!found) return candidate;
    }
    return n + 1;
}
Time: O(N²)
Space: O(1)
06

Approach 2 — Sort O(N log N)

Idea: Sort the array. Then scan linearly looking for a gap in the sequence 1, 2, 3, …

java — sort approach
Arrays.sort(arr);    // e.g. [-4, -2, -1, 1, 2, 3, 5, 7]
int candidate = 1;
for (int x : arr) {
    if (x == candidate) candidate++;  // found it, look for next
    else if (x > candidate) break;    // gap found
}
return candidate;
Time: O(N log N)
Space: O(depends on sort)

Better, but can we do it in O(N) without sorting?

07

Approach 3 — Index Marking O(N)

Core Insight — Array as a Hash Table Since the answer is in [1, N+1], we can use the array itself as a presence table. Map "number k is present" to "index k−1 is negative." Three passes do it all without extra space.

Three-Pass Algorithm

①

Pass 1 — Neutralize Out-of-Range

Replace every non-positive number (≤ 0) and every number > N with N+2. These are irrelevant — they can never be the answer and must not corrupt the sign-encoding in pass 2.

②

Pass 2 — Mark Presence

For each element, take its absolute value ele. If 1 ≤ ele ≤ N, negate arr[ele−1]. A negative value at index k means the number k+1 is present in the original array.

③

Pass 3 — Find First Positive

Scan the array. The first index i where arr[i] > 0 means number i+1 was never marked — it is the answer. If no such index exists, return N+1.

Why abs() in Pass 2? A number may already have been negated by a previous iteration of pass 2. Taking abs(arr[i]) gives us the original value to use as an index, regardless of whether it was already negated. This is the key trick that makes the algorithm correct.

java — index marking O(N) time, O(1) space
int firstMissingPositive(int[] arr, int n) {
    // Pass 1: replace non-positive & out-of-range values with N+2
    for (int i = 0; i < n; i++) {
        if (arr[i] <= 0) arr[i] = n + 2;
    }

    // Pass 2: mark presence — negate arr[ele-1] for each valid ele
    for (int i = 0; i < n; i++) {
        int ele = Math.abs(arr[i]);          // recover original value
        if (ele <= n) {
            arr[ele - 1] = Math.abs(arr[ele - 1]) * -1;
        }
    }

    // Pass 3: first positive index i → answer is i+1
    for (int i = 0; i < n; i++) {
        if (arr[i] > 0) return i + 1;
    }
    return n + 1;
}
Time: O(N)
Space: O(1)

Next Permutation — Problem Statement

Problem Statement Given an array of N integers representing a permutation, rearrange it into the next lexicographically greater permutation. If no greater permutation exists (array is in descending order), rearrange to the smallest permutation (ascending order). Must be done in-place without extra memory.

Examples

InputOutputWhy
[1, 2, 3, 4, 5][1, 2, 3, 5, 4]Swap last two
[5, 4, 2, 2, 1][1, 2, 2, 4, 5]Fully descending → reverse all
[9, 8, 2, 5][9, 8, 5, 2]Swap 2 and 5 (just greater)
[9, 8, 2, 5, 4][9, 8, 4, 2, 5]Swap 2 with 4, then reverse suffix

Intuition — The Dip

Think of the array as a height graph read right-to-left. A fully descending suffix (like 9, 8, 6, 5, 4) is already the largest arrangement of those digits — there is no way to make it bigger. The first element that breaks the descent (going left) is the dip — the pivot that must change to produce the next permutation.

4-Step Algorithm — O(N)

Step 1
Find the dip. Scan right-to-left. Find the largest index dip such that arr[dip] < arr[dip+1]. If no dip exists, the array is fully descending — skip to step 4.
Step 2
Find the just-greater element. Scan right-to-left from the end to find the smallest element in the suffix that is strictly greater than arr[dip]. Call its index j.
Step 3
Swap. Swap arr[dip] with arr[j]. This makes position dip as small as possible while still being larger than before.
Step 4
Reverse the suffix. Reverse arr[dip+1 … N−1]. Since the suffix was descending before the swap, reversing it makes it ascending — the smallest possible arrangement for those digits.
Why is the suffix always descending? By definition, dip is the rightmost index where arr[dip] < arr[dip+1]. Everything to the right of dip must therefore be non-increasing (otherwise a larger dip index would exist). So after swapping, reversing the suffix always gives the smallest possible tail.

Code & Final Examples

java — next permutation O(N) in-place
void nextPermutation(int[] arr) {
    int n = arr.length;
    int dip = -1;

    // Step 1 — find dip (rightmost i where arr[i] < arr[i+1])
    for (int i = n - 1; i > 0; i--) {
        if (arr[i - 1] < arr[i]) {
            dip = i - 1;
            break;
        }
    }

    if (dip == -1) {         // fully descending → wrap to smallest
        reverse(arr, 0, n - 1);
        return;
    }

    // Step 2 & 3 — find just-greater element and swap
    for (int i = n - 1; i > dip; i--) {
        if (arr[i] > arr[dip]) {
            // swap arr[i] and arr[dip]
            int tmp = arr[i]; arr[i] = arr[dip]; arr[dip] = tmp;
            break;
        }
    }

    // Step 4 — reverse suffix from dip+1 to end
    reverse(arr, dip + 1, n - 1);
}

void reverse(int[] arr, int l, int r) {
    while (l < r) {
        int tmp = arr[l]; arr[l] = arr[r]; arr[r] = tmp;
        l++; r--;
    }
}
Time: O(N)
Space: O(1)

Quick Recap

Merge Two Intervals

Overlap when a₂ ≤ b₁. Merged result: [min(a₁,a₂), max(b₁,b₂)]. Touching endpoints count as overlapping.

Merge N Intervals

Requires sorted input. One pass: extend end if overlap, else finalize and start fresh. O(N). Add O(N log N) sort if unsorted.

Missing Natural Number — Bound

Answer is always in [1, N+1]. Use this bound to avoid extra space by encoding presence in the array itself.

Index-as-Hash Trick

Three passes: replace invalids with N+2 → negate arr[ele−1] for each valid ele → first positive index + 1 is the answer.

abs() in Pass 2

Always use abs(arr[i]) to recover the original value — earlier iterations may have already negated it.

Next Permutation — Dip

Rightmost index where arr[i] < arr[i+1]. If none exists, reverse the whole array to get the smallest permutation.

Suffix is Descending

The suffix to the right of the dip is always descending (by definition of dip). This guarantees step 2 finds the smallest valid replacement and step 4's reverse produces the smallest tail.

Real-World — Merge Intervals

Merge intervals appears as the "best time for maintenance" problem: merge all active-user intervals, then find the largest gap between them.
