01Introduction to Recursion

What is Recursion?

Recursion is a programming technique where a function calls itself to solve a problem by reducing it to a smaller instance of the same problem — called a sub-problem.

Definition Recursion = solving a problem by breaking it into smaller instances of the same problem, until you reach a case small enough to solve directly (the base case).

Why Learn Recursion?

Recursion is not just one technique — it is the foundation of many major algorithmic domains:

🔙
Backtracking
Explore all possibilities and undo bad choices — think N-Queens, Sudoku solver, permutations.
🌳
Trees & Graphs
Tree traversals (inorder, preorder, postorder), DFS on graphs — all naturally recursive.
⚡
Dynamic Programming
DP = recursion + memoization. Every DP problem starts as a recursive formulation.
🔀
Sorting Algorithms
Merge Sort and Quick Sort are divide-and-conquer algorithms implemented recursively.
Analogy Russian nesting dolls (Matryoshka): To see the smallest doll, you open the outer one, then the next, and so on — each step is the same action on a smaller version. The base case is the innermost doll, which cannot be opened further.

02Function Call Tracing

Before diving into recursive calls, it helps to understand how the call stack works for ordinary (non-recursive) nested function calls. Each function call creates a new stack frame holding its local variables. The frame is destroyed when the function returns.

Example: Nested Calls

C++
int add(int x, int y) { return x + y; }
int mul(int x, int y) { return x * y; }
int sub(int x, int y) { return x - y; }

void main() {
    int x = 10, y = 20;
    // Evaluated inside-out:
    // add(10, 20) = 30
    // mul(30, 30) = 900
    // sub(900, 75) = 825
    print(sub(mul(add(x, y), 30), 75));
    // Output: 825
}

How the Stack Looks

When print(sub(mul(add(x,y), 30), 75)) executes, the call stack builds up from the innermost call outward:

print x = 825
sub x=900, y=75 → 825
mul x=30, y=30 → 900
add x=10, y=20 → 30
main x=10, y=20

↑ Each frame is pushed onto the stack when called, and popped when it returns.

Key Rule Nested expressions are always evaluated inside-out. The innermost function call returns first, and its result is passed to the next level up.

03Sum of First N Natural Numbers

This is the canonical first example to understand recursive thinking. The goal: compute sum(N) = 1 + 2 + 3 + … + N without a loop.

Identifying the Sub-problem

Notice that:

sum(N) = 1 + 2 + 3 + … + (N−2) + (N−1) + N = sum(N−1) + N

sum(N−1) is a smaller instance of the same problem — that is the sub-problem. We keep reducing until we reach sum(0) = 0, which is trivially known.

Recursive Code

C++
// Assumption: sum(N) calculates and returns the sum of first N natural numbers.
int sum(int N) {
    if (N == 0) return 0;    // Base case
    return sum(N - 1) + N;    // Main logic: sub-problem + current contribution
}

Dry Run — Tracing sum(5)

sum(0)returns 0
→ 0 + 1 = 1
sum(1)sum(0) + 1 → 1
→ 1 + 2 = 3
sum(2)sum(1) + 2 → 3
→ 3 + 3 = 6
sum(3)sum(2) + 3 → 6
→ 6 + 4 = 10
sum(4)sum(3) + 4 → 10
→ 10 + 5 = 15
sum(5)sum(4) + 5 → 15

Final answer: sum(5) = 15 ✓

04Steps for Writing Recursive Code

Every recursive solution follows a three-step mental framework:

Step 1 — Assumption
Decide exactly what your function will do (and return) for the given problem. Write it as a comment. Do not think about how just yet.
Step 2 — Main Logic
Break the problem into a smaller sub-problem of the same type and combine the result. Trust your assumption — pretend the recursive call works correctly.
Step 3 — Base Case
Identify the smallest input(s) for which the answer is trivially known. Return directly without making any recursive call.
Common Mistake Forgetting or writing an incorrect base case leads to infinite recursion and a stack overflow error. Always define a base case that is actually reachable.
Tip — Leap of Faith When writing Step 2, do not try to mentally trace the entire recursive chain. Trust that sum(N−1) is correctly computed — that is the "leap of faith" that makes recursion manageable.

05Factorial of N

What is a Factorial?

The factorial of a non-negative integer N, written N!, is the product of all positive integers from 1 up to N.

N! = 1 × 2 × 3 × … × (N−2) × (N−1) × N = (N−1)! × N

Special case: 0! = 1 (by convention, and confirmed mathematically).

Example: 5! = 1 × 2 × 3 × 4 × 5 = 120

Recursive Formulation

factorial(N) = factorial(N−1) × N factorial(0) = 1 ← base case

Code

C++
// Assumption: factorial(N) returns the value of N!
int factorial(int N) {
    if (N == 0) return 1;         // Base case: 0! = 1
    return factorial(N - 1) * N;  // Main logic
}

Dry Run — factorial(4)

factorial(0)returns 1
→ 1 × 1 = 1
factorial(1)factorial(0) × 1 → 1
→ 1 × 2 = 2
factorial(2)factorial(1) × 2 → 2
→ 2 × 3 = 6
factorial(3)factorial(2) × 3 → 6
→ 6 × 4 = 24
factorial(4)factorial(3) × 4 → 24

Result: 4! = 24 ✓

Complexity

TimeO(N) SpaceO(N)

There are N + 1 function calls (from N down to 0), each doing O(1) work. The call stack holds at most N + 1 frames simultaneously, giving O(N) space.

06Print 1 → N in Increasing Order

Print all integers from 1 to N in ascending order using recursion — no loops.

Key Insight

To print 1, 2, …, N, first print 1, 2, …, N−1 (sub-problem), then print N. The print of N happens after the recursive call returns — this is called executing in the unwinding phase.

C++
// Assumption: pI(N) prints 1, 2, ..., N in order.
void pI(int N) {
    if (N == 0) return;   // Base case: nothing to print
    pI(N - 1);           // ① Recurse first (go deeper)
    print(N);            // ② Print on the way back up
}
Before recursive call ①

Lines here execute while going down the recursion (before the call).

After recursive call ②

Lines here execute while coming back up — during the unwinding phase.

Because print(N) is placed after the recursive call, numbers print from smallest to largest as the stack unwinds.

For pI(5): output is 1, 2, 3, 4, 5

Complexity

TimeO(N) SpaceO(N)

07Print N → 1 in Decreasing Order

Now print from N down to 1. A real-world analogy from the notes: a Whirlpool washing machine timer that counts down from the set time to 0 — each minute displayed in decreasing order.

Problem — Whirlpool Timer A user sets a timer for A minutes. Display each minute counting down: A, A−1, A−2, …, 1. Implement with recursion.

Key Insight

To print N, N−1, …, 1: print N first (before the recursive call), then recurse for N−1.

C++
// Assumption: pD(N) prints N, N-1, ..., 1 in order.
void pD(int N) {
    if (N == 0) return;   // Base case
    print(N);            // ① Print BEFORE recursing (going down phase)
    pD(N - 1);           // ② Recurse
}

For pD(4): output is 4, 3, 2, 1

Comparison: Inc vs Dec

VariantPrint positionOutput for N=4
Increasing (pI)After recursive call1, 2, 3, 4
Decreasing (pD)Before recursive call4, 3, 2, 1

Complexity

TimeO(N) SpaceO(N)

08Key Observation — Before & After the Recursive Call

This is one of the most important mental models in recursion. Inside any recursive function, the code is split by the recursive call into two zones:

Zone A — Before the recursive call

These lines run while going down the recursion tree (from larger N toward the base case). Each new frame executes them immediately before calling itself again.

↓ recursive call ↓
Zone B — After the recursive call

These lines run while coming back up (unwinding). The deepest frame's Zone B runs first, followed by each parent frame on the way back.

Why This Matters Swapping a print statement from before to after the recursive call completely reverses the output order — no other change needed. This principle is heavily used in tree traversals (preorder vs postorder) and backtracking.

09Time & Space Complexity

General Formulas

Time Complexity = O(number of function calls × work per call) Space Complexity = O(maximum recursion depth × space per frame)

Time Complexity Explained

For a function like factorial(N), there are exactly N + 1 calls (N, N−1, …, 1, 0). Each call does a constant amount of work (one comparison, one multiplication, one return). Therefore:

factorial — TimeO(N)

Space Complexity Explained

The space complexity of a recursive algorithm equals the maximum depth of the call stack at any single point in time, multiplied by the space used per frame.

Important We do not add up all frames ever created. At any moment, only the frames on the current path from root to the active call are live on the stack. Once a call returns, its frame is popped and that memory is freed.

For factorial(N), the maximum stack depth is N + 1:

factorial — SpaceO(N)

Recursive Formulation

fib(0) = 0 fib(1) = 1 fib(N) = fib(N−1) + fib(N−2) for N ≥ 2

Code

C++
// Assumption: fib(N) returns the Nth Fibonacci number.
int fib(int N) {
    if (N <= 1) return N;              // Base cases: fib(0)=0, fib(1)=1
    return fib(N - 1) + fib(N - 2);   // Two recursive sub-problems
}

Recursion Tree — fib(4)

Time Complexity of Fibonacci

Unlike factorial (one call per level), Fibonacci makes two recursive calls per invocation, forming a binary tree of calls. The number of calls at each level doubles:

LevelCalls
L01
L12
L24
L38
……
LN2ᴺ

Total calls ≈ 1 + 2 + 4 + … + 2ᴺ = 2ᴺ⁺¹ − 1 ≈ 2ᴺ

fib — TimeO(2ᴺ)
Warning O(2ᴺ) is exponential and becomes extremely slow very quickly (e.g., fib(50) would require trillions of calls). The fix is memoization (caching results), which reduces it to O(N). This is introduced in the Dynamic Programming module.

Space Complexity of Fibonacci

Despite the exponential number of total calls, the maximum stack depth at any instant is only N. The algorithm always explores one branch fully to the base case before starting the other branch. At no point are more than N frames on the stack simultaneously.

fib — SpaceO(N)
Insight — Branch-by-Branch Execution When fib(N) calls fib(N−1), it fully resolves the entire left subtree before starting the right branch fib(N−2). At any moment, only one root-to-leaf path is active on the call stack.

★Quick Recap

Recursion
A function that calls itself with a smaller input to solve sub-problems, stopping at a base case.
Three Steps
Assumption → Main Logic (sub-problem) → Base Case. Always write in this order.
Base Case
The trivially-known input where recursion stops. Without it you get a stack overflow.
Call Stack
Each call pushes a frame; returning pops it. Space complexity = maximum stack depth.
Before vs After
Code before the recursive call runs going down; code after runs while unwinding back up.
Factorial
N! = factorial(N−1) × N. Base: 0! = 1. Time O(N), Space O(N).
Print Inc / Dec
Put print after recursive call for ascending; before for descending. Both O(N) time & space.
Fibonacci (Naive)
fib(N) = fib(N−1) + fib(N−2). Two recursive branches → Time O(2ᴺ), Space O(N).
