How to Approach a Problem

Before diving into data structures, internalize a repeatable problem-solving loop. Every competitive programmer and software engineer uses some version of this — here is the one practiced in class:

⏱️
Start a 35-minute timer. Time-boxing forces focused effort instead of passive reading.
📖
Read carefully. Read the problem statement, all examples, and verify the expected inputs and outputs.
💡
Think brute force first. The simplest solution that is obviously correct — never skip this step.
📐
Check constraints and optimize. If N ≤ 105, you typically need O(N log N) or better. Use constraints to rule out approaches.
✏️
Dry-run on paper — this is the most critical step. Trace through your algorithm by hand on a small example before writing a single line of code.
✅
If solved in 35 minutes → move to the next question. If not → bookmark, take hints (left to right), watch the video explanation, or raise a TA request.
Tip After failing a problem, ask yourself: "What concept, if I had known it, would have let me solve this?" Then go revise that concept specifically. Targeted review beats re-reading everything.

Maximum Subarray Sum

Problem Statement Given an array arr[N] of integers (positive and negative), find the maximum sum among all contiguous subarrays. Constraint: 1 ≤ N ≤ 105.
Real-World Framing Imagine you are a cryptocurrency trader. Each element of the array is the daily price change for a coin. You want to choose a consecutive holding period (buy day → sell day) that maximizes your total profit. Because profit/loss is the sum of daily changes in that period, this is exactly the maximum subarray sum problem.

02

Approach 1 — Brute Force O(N³)

Idea: Enumerate every possible pair of start index i and end index j, then iterate from i to j to compute that subarray's sum. Track the maximum.

java / pseudo-code 
int maxSubarraySum(int[] arr, int n) {
    int ans = Integer.MIN_VALUE;

    for (int i = 0; i < n; i++) {           // start index
        for (int j = i; j < n; j++) {        // end index
            int sum = 0;
            for (int k = i; k <= j; k++) {   // sum subarray [i..j]
                sum += arr[k];
            }
            ans = Math.max(ans, sum);
        }
    }
    return ans;
}
Time: O(N³)
Space: O(1)

For N = 105, O(N³) = 1015 operations — this will time out. We need to optimize.

03

Approach 2 — Prefix Sum O(N²)

The inner loop in the brute force computes the same partial sums repeatedly. We can precompute a prefix sum array to answer any subarray-sum query in O(1).

Key Insight — Prefix Sum Define psum[i] = arr[0] + arr[1] + … + arr[i].
Then sum(i, j) = psum[j] − psum[i−1] for i > 0, or simply psum[j] when i = 0.
java / pseudo-code 
// Step 1 — build prefix sum array
int[] psum = new int[n];
psum[0] = arr[0];
for (int i = 1; i < n; i++) {
    psum[i] = psum[i - 1] + arr[i];
}

// Step 2 — use prefix sum for O(1) range queries
int ans = Integer.MIN_VALUE;
for (int i = 0; i < n; i++) {
    for (int j = i; j < n; j++) {
        int sum = (i == 0) ? psum[j] : psum[j] - psum[i - 1];
        ans = Math.max(ans, sum);
    }
}
return ans;
Time: O(N²)
Space: O(N)

Still O(N²) — better, but still too slow for N = 105. The prefix sum technique is, however, used directly in Sections 06 and 07.

04

Approach 3 — Carry Forward O(N²)

Instead of building a separate prefix array, we can compute the subarray sum incrementally by carrying it forward as we extend the right boundary j.

java / pseudo-code 
int ans = Integer.MIN_VALUE;

for (int i = 0; i < n; i++) {       // fix start
    int sum = 0;
    for (int j = i; j < n; j++) {    // extend end
        sum += arr[j];              // carry sum forward
        ans = Math.max(ans, sum);
    }
}
return ans;
Time: O(N²)
Space: O(1)

Same time complexity as the prefix-sum approach but uses O(1) space. The key observation that drives Kadane's Algorithm next is: when should we reset the sum to zero?

Observations That Lead to Kadane's

➕

All Positive

Take the entire array — sum of all elements is always optimal.

➖

All Negative

The best subarray is a single element — the maximum (least negative) one.

〰️

Mixed Signs

A negative prefix only drags down any subarray that includes it. Carry the running sum forward only when it is positive.

05

Approach 4 — Kadane's Algorithm O(N)

Core Insight If the running sum becomes negative, it is always better to start a fresh subarray from the next element rather than carry a negative sum into the future. A negative prefix only lowers any subarray sum it joins.
java — Kadane's Algorithm 
int kadane(int[] arr, int n) {
    int ans = Integer.MIN_VALUE;
    int sum = 0;

    for (int i = 0; i < n; i++) {
        sum += arr[i];
        ans = Math.max(ans, sum);
        if (sum < 0) sum = 0;   // discard negative prefix
    }
    return ans;
}
Time: O(N)
Space: O(1)

Continuous Sum Query — Suffix Increment

Problem Statement Given arr[N] (initially all zeros) and Q queries. Each query specifies an index idx and a value val. Increment every element from index idx to the last index by val. Return the final array. (1 ≤ N, Q ≤ 105)

Example

N = 7, queries: (3, +4), (1, +3), (4, −2)

idx0123456
after (3,+4)000+4+4+4+4
after (1,+3)0+3+3+7+7+7+7
after (4,−2)0+3+3+7+5+5+5
Final0337555

Naive Approach — O(Q × N)

For each query, iterate from idx to N−1 and add val. This is O(Q × N) = 1010 for maximum inputs — too slow.

Optimal Approach — Difference Array + Prefix Sum

Key Insight Instead of updating a range of elements, record only the start of the change in a difference array. A single prefix-sum pass at the end reconstructs the full result.

For a suffix increment starting at idx by val: set arr[idx] += val. Then take the prefix sum of arr to get final values.

java — suffix increment via difference array 
int[] suffixIncrementQuery(int n, int[][] queries) {
    int[] arr = new int[n];   // all zeros initially

    // Step 1 — mark the start of each change
    for (int i = 0; i < queries.length; i++) {
        int idx = queries[i][0];
        int val = queries[i][1];
        arr[idx] += val;
    }

    // Step 2 — prefix sum to propagate changes rightward
    for (int i = 1; i < n; i++) {
        arr[i] = arr[i - 1] + arr[i];
    }

    return arr;
}
Time: O(N + Q)
Space: O(1) extra
Analogy Think of each query as dropping a colored dye at position idx. The prefix-sum pass is like water flowing right — the dye spreads from its starting point to the end of the array.

Naive Approach — O(Q × N)

For each query, loop from s to e and add val. Too slow for large inputs.

Optimal Approach — Difference Array Trick

Key Insight — Difference Array To represent "add val to the range [s, e]", store:
  • arr[s] += val — marks the start of the increment
  • arr[e+1] -= val — marks the end of the increment (if e+1 < N)
After processing all queries, a single prefix-sum pass reconstructs the final array.
java — range increment via difference array 
int[] rangeIncrementQuery(int n, int[][] queries) {
    int[] arr = new int[n];

    // Step 1 — apply difference array updates
    for (int i = 0; i < queries.length; i++) {
        int s   = queries[i][0];
        int e   = queries[i][1];
        int val = queries[i][2];

        arr[s] += val;
        if (e + 1 < n) arr[e + 1] -= val;  // stop propagation
    }

    // Step 2 — prefix sum to restore actual values
    for (int i = 1; i < n; i++) {
        arr[i] = arr[i - 1] + arr[i];
    }

    return arr;
}
Time: O(N + Q)
Space: O(1) extra

Trapping Rain Water

Problem Statement Given arr[N] where arr[i] is the height of the i-th building, return the total amount of rainwater trapped across all buildings. (1 ≤ N ≤ 105)

Visualizing the Problem

Array: [2, 1, 3, 2, 1, 2, 4, 3, 2, 1, 3, 1] — total trapped water = 8 units

Core Observation

Formula Water trapped on top of building i is:
water[i] = min(left_max[i], right_max[i]) − arr[i]

where left_max[i] = maximum height in arr[0..i] and right_max[i] = maximum height in arr[i..N−1].
Analogy Imagine pouring water into the landscape. Each building is like a pillar. Water can only be held at position i if there are taller (or equal) pillars on both sides. The water level is capped by the shorter of the two boundary pillars — any water above that level spills over.

If left_max[i] or right_max[i] equals arr[i] itself (i.e., position i is the tallest on one side), then no water sits on top of it. The formula yields 0 or less, so we add max(0, water[i]) — although precomputing left/right max correctly already ensures non-negative values.

Brute Force — O(N²)

For every element, iterate left to find left_max and iterate right to find right_max, then apply the formula.

Time: O(N²)
Space: O(1)

Optimal — Precomputed Left/Right Max Arrays — O(N)

Precompute lmax[] in one left-to-right pass and rmax[] in one right-to-left pass. Then a final pass sums up all the trapped water.

java — trapping rain water O(N) 
int trapRainWater(int[] arr, int n) {
    // Step 1 — left-max for every index
    int[] lmax = new int[n];
    lmax[0] = arr[0];
    for (int i = 1; i < n; i++) {
        lmax[i] = Math.max(lmax[i - 1], arr[i]);
    }

    // Step 2 — right-max for every index
    int[] rmax = new int[n];
    rmax[n - 1] = arr[n - 1];
    for (int i = n - 2; i >= 0; i--) {
        rmax[i] = Math.max(rmax[i + 1], arr[i]);
    }

    // Step 3 — accumulate trapped water
    int ans = 0;
    for (int i = 0; i < n; i++) {
        ans += Math.min(lmax[i], rmax[i]) - arr[i];
    }
    return ans;
}
Time: O(N)
Space: O(N)

Quick Recap

Brute Force Subarray Sum

Three nested loops — enumerate every (i, j) pair and sum the subarray [i..j]. O(N³) time, O(1) space.

Prefix Sum Optimization

Precompute psum[i] so any range sum is O(1). Reduces subarray enumeration to O(N²).

Carry Forward

Incrementally extend the subarray sum as j grows. O(N²) time, O(1) space — no extra array needed.

Kadane's Algorithm

Keep a running sum; reset it to 0 whenever it goes negative. O(N) time, O(1) space — optimal.

Suffix Increment Query

Mark the start of each change with arr[idx] += val, then a single prefix-sum pass gives the result. O(N + Q).

Range Increment Query

Difference array trick: arr[s] += val and arr[e+1] -= val, then prefix sum. O(N + Q).

Trapping Rain Water

At each index: water = min(lmax, rmax) − height. Precompute both arrays in O(N), then sum. Total O(N) time and O(N) space.

init = INT_MIN

Always initialize ans = Integer.MIN_VALUE for max subarray problems — the answer can be negative.
