02Partition — Algorithm & Code

The two-pointer partition uses a left pointer l (starts at index 1, just after the pivot) and a right pointer r (starts at the last index). They march toward each other, swapping only when both point at misplaced elements.

Step 1 — Pick pivot Set pivot = arr[0], then l = 1, r = arr.length - 1.
Step 2 — Advance l While arr[l] ≤ pivot, increment l. This element is already on the correct side.
Step 3 — Retreat r While arr[r] > pivot, decrement r. This element is already on the correct side.
Step 4 — Swap If l ≤ r, swap arr[l] with arr[r], then do l++, r--. Repeat from Step 2.
Step 5 — Place pivot When l > r, swap arr[0] with arr[r]. The pivot is now at its correct sorted position.
Java — Partition (standalone)
int partition(int[] arr) {
    int pivot = arr[0];
    int l = 1, r = arr.length - 1;

    while (l <= r) {
        if (arr[l] <= pivot) {
            l++;                          // arr[l] is on the correct side
        } else if (arr[r] > pivot) {
            r--;                          // arr[r] is on the correct side
        } else {
            // arr[l] > pivot AND arr[r] <= pivot → swap
            int tmp = arr[l];
            arr[l] = arr[r];
            arr[r] = tmp;
            l++; r--;
        }
    }
    // Place pivot at its correct position
    int tmp = arr[0]; arr[0] = arr[r]; arr[r] = tmp;
    return r;  // pivot's final index
}
TimeO(N) SpaceO(1)
Key Observation After one call to partition, the pivot element is permanently in its correct sorted position. Every element to its left is ≤ pivot; every element to its right is > pivot.

Key Insight partition() is the engine; Quick Sort is just the driver. Each call permanently fixes one element. The algorithm terminates when every sub-array has length 0 or 1 (base case).
Java — Quick Sort (full implementation)
void quickSort(int[] arr, int lo, int hi) {
    if (lo >= hi) return;             // Base case: 0 or 1 element

    int pi = partition(arr, lo, hi);   // Pivot's final index
    quickSort(arr, lo, pi - 1);        // Sort left sub-array
    quickSort(arr, pi + 1, hi);        // Sort right sub-array
}

int partition(int[] arr, int lo, int hi) {
    int pivot = arr[lo];
    int l = lo + 1, r = hi;

    while (l <= r) {
        if (arr[l] <= pivot)       { l++; }
        else if (arr[r] > pivot)   { r--; }
        else {
            int tmp = arr[l]; arr[l] = arr[r]; arr[r] = tmp;
            l++; r--;
        }
    }
    int tmp = arr[lo]; arr[lo] = arr[r]; arr[r] = tmp;
    return r;
}

05Randomized Quick Sort

The fix for the worst case is elegantly simple: instead of always picking arr[lo] as pivot, pick a random element from the sub-array and swap it to position lo before calling partition.

Why Does Randomization Help?

For the worst case to occur at every level of recursion, the random pivot must be the minimum element each time. The probability of that:

P(picking min from N elements each time)

  = 1/N  ×  1/(N−1)  ×  1/(N−2)  ×  …  ×  1/1
  = 1 / N!

This is astronomically small — effectively impossible for any real input.

Java — Randomized Quick Sort
Random rand = new Random();

void randomizedQuickSort(int[] arr, int lo, int hi) {
    if (lo >= hi) return;

    // Pick a random index, swap it to lo so partition uses it as pivot
    int pivotIdx = lo + rand.nextInt(hi - lo + 1);
    int tmp = arr[lo]; arr[lo] = arr[pivotIdx]; arr[pivotIdx] = tmp;

    int pi = partition(arr, lo, hi);
    randomizedQuickSort(arr, lo, pi - 1);
    randomizedQuickSort(arr, pi + 1, hi);
}
Expected TimeO(N log N) Expected SpaceO(log N)
Tip In competitive programming and interviews, always prefer randomized Quick Sort or rely on Collections.sort() / Arrays.sort(). The key insight to understand is why randomization helps — it makes worst-case inputs extremely improbable by breaking the adversary's ability to construct them.

06Comparators

A comparator is a function supplied to a sorting algorithm to define a custom ordering. Instead of sorting by natural order (1, 2, 3…), you can sort by string length, factor count, or any arbitrary criterion.

The Return-Value Contract

A comparator receives two values a and b and must return an integer:

Return ValueMeaningEffect
negative (−ve)a should come before ba moves left
0Both are equivalentOrder unchanged
positive (+ve)b should come before ab moves left
Java / Python / JS / C#
  • Return −ve → first arg comes first
  • Return 0 → equal, no swap
  • Return +ve → second arg comes first
  • Ascending shortcut: return a - b;
  • Descending shortcut: return b - a;
C++
  • Return true → first arg comes before second
  • Return false → second comes first (or equal)
  • Must define a strict weak ordering
  • Never return true for equal elements — undefined behavior
Java — Basic ascending comparator
int compare(int val1, int val2) {
    return val1 - val2;  // negative → val1 first (ascending order)
}

// C++ equivalent
bool compare(int val1, int val2) {
    return val1 < val2;  // true → val1 comes before val2
}
Overflow Warning The subtraction trick (return a - b) can overflow for large integers (e.g., a = Integer.MAX_VALUE, b = -1). Use explicit comparisons (if (a < b) return -1; else if (a > b) return 1; else return 0;) when values may approach ±2×109.

Counting Factors Efficiently

Iterate i from 1 to √n. For each i that divides n, add 2 (both i and n/i). If i × i = n, add only 1 (perfect square).

Java — Count Factors + Sort with Comparator
int countFactors(int n) {
    int count = 0;
    for (int i = 1; i * i <= n; i++) {
        if (n % i == 0)
            count += (i * i == n) ? 1 : 2;
    }
    return count;
}

Arrays.sort(arr, new Comparator<Integer>() {
    public int compare(Integer a, Integer b) {
        int fa = countFactors(a);
        int fb = countFactors(b);
        if (fa < fb) return -1;      // fewer factors → a first
        else if (fa > fb) return 1;  // fewer factors → b first
        else return a - b;            // tie-break by value
    }
});

// Java 8 lambda equivalent
Arrays.sort(arr, (a, b) -> {
    int diff = countFactors(a) - countFactors(b);
    return diff != 0 ? diff : a - b;
});
TimeO(N log N · √Amax) SpaceO(log N)
Optimization Since A[i] ≤ 104, precompute all factor counts for 1…104 with a sieve in O(Amax log Amax). Then each comparator call is O(1), reducing total time to O(N log N + Amax log Amax).

08Problem: Largest Number

Problem Given a list of non-negative integers, arrange them so they form the largest possible number. Return the result as a string (since it may overflow any integer type).

Examples

nums = [10, 2]  →  "210"

"2" + "10" = "210" > "10" + "2" = "102"  →  2 should come first.

nums = [3, 30, 34, 5, 9]  →  "9534330"

The Key Insight

You can't sort by numeric value. For any two numbers a and b, compare the concatenation "ab" vs "ba". Place a before b if "ab" > "ba".

Analogy Imagine laying numbered tiles end-to-end. The question for each pair: which tile on the left makes the bigger two-tile number? The comparator answers that question for every pair, and the sort engine guarantees global consistency.
Java — Largest Number comparator
Arrays.sort(arr, new Comparator<Integer>() {
    public int compare(Integer a, Integer b) {
        String ab = a + "" + b;   // e.g. a=9, b=10 → "910"
        String ba = b + "" + a;   // e.g.          → "109"
        int i = 0, j = 0;
        while (i < ab.length() && j < ba.length()) {
            if (ab.charAt(i) < ba.charAt(j)) return  1; // ba is larger → b first
            if (ab.charAt(i) > ba.charAt(j)) return -1; // ab is larger → a first
            i++; j++;
        }
        return 0;
    }
});

// Cleaner Java 8+ version
Arrays.sort(arr, (a, b) -> (b + "" + a).compareTo(a + "" + b));
Java — Full solution including edge case (all zeros)
public String largestNumber(int[] nums) {
    Integer[] arr = Arrays.stream(nums).boxed().toArray(Integer[]::new);
    Arrays.sort(arr, (a, b) -> (b + "" + a).compareTo(a + "" + b));
    if (arr[0] == 0) return "0";   // All elements are 0
    StringBuilder sb = new StringBuilder();
    for (int n : arr) sb.append(n);
    return sb.toString();
}
TimeO(N log N) SpaceO(log N)
Edge Cases to Test All zeros → "0". Single element. Numbers with repeated digits in different orders: [34, 3] → "343", not "334".

Pivot Partition
Two-pointer technique that rearranges an array around a pivot in O(N) time, O(1) space. The pivot lands at its exact final sorted index.
Quick Sort
Recursively apply partition on left and right sub-arrays. Each call permanently places one element, giving average O(N log N) performance.
Best Case
Pivot always at midpoint → balanced binary tree → O(N log N) time, O(log N) call-stack space.
Worst Case
Sorted input + first-element pivot → degenerate linear tree → O(N²) time, O(N) call-stack space.
Randomized Quick Sort
Randomly select pivot to reduce worst-case probability to 1/N!, giving expected O(N log N) across all inputs.
Comparator Contract
Return −ve to put first arg first, +ve to put second arg first, 0 for equal. Enables sorting by any custom criterion.
Sort by Factors
Count factors in O(√n) per element; comparator uses factor-count as primary key and value as tie-breaker. Total: O(N log N · √Amax).
Largest Number
Compare concatenations "ab" vs "ba" for any pair. Sort descending by this rule, then join all elements as a string.
