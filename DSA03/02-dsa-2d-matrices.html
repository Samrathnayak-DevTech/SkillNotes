02

Approach 1 — Brute Force O(N×M)

Idea: Iterate over every element in the matrix. If any element equals k, return true. This completely ignores the sorted property.

java
boolean searchBF(int[][] mat, int n, int m, int k) {
    for (int i = 0; i < n; i++)
        for (int j = 0; j < m; j++)
            if (mat[i][j] == k) return true;
    return false;
}
Time: O(N×M)
Space: O(1)

Correct, but wasteful — it throws away the sorted structure entirely.

03

Approach 2 — Binary Search on Each Row O(N log M)

Idea: Since each row is individually sorted, apply binary search independently on every row. If any binary search finds k, return true.

java
boolean searchBS(int[][] mat, int n, int m, int k) {
    for (int i = 0; i < n; i++) {
        int lo = 0, hi = m - 1;
        while (lo <= hi) {
            int mid = lo + (hi - lo) / 2;
            if      (mat[i][mid] == k) return true;
            else if (mat[i][mid] <  k) lo = mid + 1;
            else                        hi = mid - 1;
        }
    }
    return false;
}
Time: O(N log M)
Space: O(1)

Better — but still does N independent binary searches. The column-sorted property is never used. We can do even better.

04

Approach 3 — Staircase Search O(N+M)

Key Insight Start at the top-right corner mat[0][M−1]. This position has a unique property: moving left decreases the value (row is sorted), and moving down increases the value (column is sorted). This means at every step we can eliminate exactly one row or one column.
Analogy Think of it like a staircase. You start at the top-right. If the current cell is too large, step left (eliminate the rightmost column). If too small, step down (eliminate the top row). You'll either land on the target or fall off the matrix.

java — staircase search
boolean searchStaircase(int[][] mat, int n, int m, int k) {
    int row = 0, col = m - 1;   // start: top-right corner

    while (row < n && col >= 0) {
        if      (mat[row][col] == k) return true;
        else if (mat[row][col]  > k) col--;   // too big → move left
        else                         row++;   // too small → move down
    }
    return false;
}
Time: O(N+M)
Space: O(1)
Why O(N+M)? At every step we either decrement col or increment row — never both. So the pointer travels at most N steps downward and M steps leftward before exiting the matrix. Total iterations ≤ N + M.

The bottom-left corner also works as a starting point — the logic mirrors: move right if current < k, move up if current > k.

Idea — Four Passes

Print the boundary in four separate sweeps. Each sweep prints exactly N−1 elements to avoid double-counting the corners:

→

Pass 1 — Top Row

Print N−1 elements of row 0, from col 0 to col N−2. Advance col.

↓

Pass 2 — Right Column

Print N−1 elements of col N−1, from row 0 to row N−2. Advance row.

←

Pass 3 — Bottom Row

Print N−1 elements of row N−1, from col N−1 to col 1. Retreat col.

↑

Pass 4 — Left Column

Print N−1 elements of col 0, from row N−1 to row 1. Retreat row.

java — print boundary elements
void printBoundary(int[][] mat, int n) {
    int row = 0, col = 0;

    // Pass 1 → top row: (0,0) to (0,N-2)
    for (int i = 1; i < n; i++) {
        System.out.print(mat[row][col] + " ");
        col++;
    }
    // Pass 2 ↓ right column: (0,N-1) to (N-2,N-1)
    for (int i = 1; i < n; i++) {
        System.out.print(mat[row][col] + " ");
        row++;
    }
    // Pass 3 ← bottom row: (N-1,N-1) to (N-1,1)
    for (int i = 1; i < n; i++) {
        System.out.print(mat[row][col] + " ");
        col--;
    }
    // Pass 4 ↑ left column: (N-1,0) to (1,0)
    for (int i = 1; i < n; i++) {
        System.out.print(mat[row][col] + " ");
        row--;
    }
}
Time: O(N)
Space: O(1)
Why N−1 per pass? Each corner cell is shared between two sides. Printing N elements per pass would count all four corners twice. By printing N−1 elements and starting each pass where the previous one left off, every corner is printed exactly once.

java — spiral printing
void spiralPrinting(int[][] mat, int n) {
    int row = 0, col = 0;

    while (n > 1) {
        // → top row
        for (int i = 1; i < n; i++) { System.out.print(mat[row][col] + " "); col++; }
        // ↓ right column
        for (int i = 1; i < n; i++) { System.out.print(mat[row][col] + " "); row++; }
        // ← bottom row
        for (int i = 1; i < n; i++) { System.out.print(mat[row][col] + " "); col--; }
        // ↑ left column
        for (int i = 1; i < n; i++) { System.out.print(mat[row][col] + " "); row--; }

        row++; col++;  // move inward to the next ring
        n -= 2;        // shrink the ring size
    }
    // handle the center cell when N is odd
    if (n == 1) System.out.print(mat[row][col]);
}
Time: O(N²)
Space: O(1)

What Is a Sub-Matrix?

A sub-matrix is any contiguous rectangular portion of a matrix. It is uniquely defined by two corners:

↖

Top-Left (TL)

The cell at the smallest row and smallest column of the sub-matrix. Also written as (r₁, c₁).

↘

Bottom-Right (BR)

The cell at the largest row and largest column. Also written as (r₂, c₂).

Key Facts A single element is a valid sub-matrix (TL = BR). The whole matrix is also a valid sub-matrix. For an N×M matrix, the total number of sub-matrices is:
N(N+1)/2 × M(M+1)/2 ≈ N²M²/4

Sum of All Sub-Matrix Sums — Brute Force

Problem Statement Given mat[N][M], find the sum of the sums of every possible sub-matrix. There are approximately N²M²/4 sub-matrices total.

Brute force idea: Enumerate every (TL, BR) pair using four nested loops — two for the TL corner and two for the BR corner. For each pair, iterate over the sub-matrix and sum its elements.

java — brute force O(N³M³)
long subMatrixSumBF(int[][] mat, int n, int m) {
    long ans = 0;
    for (int rsi = 0; rsi < n; rsi++)          // TL row
      for (int rei = rsi; rei < n; rei++)        // BR row
        for (int csi = 0; csi < m; csi++)        // TL col
          for (int cei = csi; cei < m; cei++) {  // BR col
            // iterate every cell in sub-matrix [rsi,csi]..[rei,cei]
            for (int r = rsi; r <= rei; r++)
              for (int c = csi; c <= cei; c++)
                ans += mat[r][c];
          }
    return ans;
}
Time: O(N³M³)
Space: O(1)

For N = M = 100 that is 1012 operations — completely infeasible. We need a fundamentally different approach.

09

Contribution Technique — O(N×M)

Core Insight Instead of summing each sub-matrix individually, ask: how many times does each element mat[i][j] appear across all sub-matrix sums? That count is its contribution. Multiply each element by its contribution and sum everything up.

Counting the Contribution of mat[i][j]

An element at position (i, j) appears in a sub-matrix if and only if:

  • The Top-Left corner of that sub-matrix is somewhere in the rectangle from (0,0) to (i,j) — that is (i+1) choices for the TL row × (j+1) choices for the TL column.
  • The Bottom-Right corner is somewhere in the rectangle from (i,j) to (N−1, M−1) — that is (N−i) choices for the BR row × (M−j) choices for the BR column.
Contribution of element at (i, j) count(i, j) = (i+1) × (j+1) × (N−i) × (M−j)

Visual Example — mat[i][j] where i=2, j=2 in a 5×4 matrix

Worked Example For element mat[1][2] in a 3×5 matrix (N=3, M=5):
TL choices = (1+1) × (2+1) = 2 × 3 = 6
BR choices = (3−1) × (5−2) = 2 × 3 = 6
Contribution count = 6 × 6 = 36 sub-matrices

Final Code

java — contribution technique O(N×M)
long subMatrixSumContribution(int[][] mat, int n, int m) {
    long ans = 0;
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < m; j++) {
            // number of sub-matrices containing mat[i][j]
            long tlChoices = (long)(i + 1) * (j + 1);
            long brChoices = (long)(n - i) * (m - j);
            ans += (long) mat[i][j] * tlChoices * brChoices;
        }
    }
    return ans;
}
Time: O(N×M)
Space: O(1)

Quick Recap

Staircase Search

Start at the top-right corner; move left if too big, down if too small. O(N+M) — eliminates one row or column at every step.

Why Top-Right?

It is the only corner where one neighbor is strictly smaller (left) and another is strictly larger (down), enabling unambiguous decisions.

Boundary Print

Four passes of N−1 elements each. Track a single (row, col) cursor that advances along the boundary. O(N) time, O(1) space.

Spiral Print

Reuse boundary printing in a while loop. After each ring: row++, col++, N−=2. Handle the odd-N center cell separately. O(N²).

Sub-Matrix Count

For an N×M matrix: N(N+1)/2 × M(M+1)/2 total sub-matrices, which is ≈ N²M²/4.

Contribution Formula

Element at (i,j) appears in (i+1)(j+1)(N−i)(M−j) sub-matrices. Multiply and sum for O(NM) total.

Use long!

Sub-matrix sums can overflow int. Always use long for the accumulator and the contribution product.

Staircase Starting Points

Both top-right and bottom-left corners work. Top-right: left=smaller, down=larger. Bottom-left: right=larger, up=smaller.
