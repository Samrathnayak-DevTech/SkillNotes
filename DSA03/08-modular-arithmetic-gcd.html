01The Modulo (%) Operator

A % B gives the remainder when A is divided by B.

Range of (A % B) → [0, B − 1]

No matter how large A is, A % B always falls within this finite range.

Why Do We Need %?

Many problems involve numbers that grow astronomically large — factorials, power functions, sum of N numbers where N ≤ 10⁹. Storing or comparing these raw values is impractical. Applying % m limits the range of any value to [0, m−1], preventing integer overflow and enabling fast computation.

Common Modulus in Competitive Programming The value 10⁹ + 7 (= 1,000,000,007) is a prime number commonly used as the modulus in contest problems. It fits in a 32-bit signed integer, and products of two values taken mod 10⁹ + 7 fit in a 64-bit long.

02Five Properties of Modular Arithmetic

These five identities are the foundation of every modular computation. Memorize them — they appear constantly in problem constraints like "print answer modulo 10⁹+7".

#PropertyFormulaExample
1 Addition (a + b) % m = ((a%m) + (b%m)) % m (3+4)%5 = (3+4)%5 = 2 ✓
(9+8)%5 = (4+3)%5 = 2 ✓
2 Multiplication (a × b) % m = ((a%m) × (b%m)) % m (4×5)%7 = (4×5)%7 = 6
=(4%7 × 5%7)%7 = (4×5)%7 = 6 ✓
3 Add m cancels (a + m) % m = a % m Since m%m = 0, adding m is a no-op mod m.
4 Subtraction (a − b) % m = ((a%m) − (b%m) + m) % m a=10, b=2, m=9: (1−2+9)%9=8 ✓
a=7, b=2, m=9: (7−2)%9=5 ✓
5 Power aᵇ % m = ((a%m)ᵇ) % m a%m is applied first; then raise to power b.
Subtraction Trap When computing (a − b) % m, the result of (a%m − b%m) can be negative (e.g., 1 − 2 = −1). Always add m before taking modulo to ensure a non-negative result: ((a%m) − (b%m) + m) % m.
Note on Division Division does not distribute over modulo the same way. (a / b) % m ≠ ((a%m) / (b%m)) % m in general. For modular division you need the modular multiplicative inverse, which is an advanced topic.

These two quiz problems from the notes demonstrate Property 4 (subtraction) and Property 5 (power) in action.

Quiz 1 — Evaluate (37¹⁰³ − 1) % 12
Step 1: Apply subtraction property → (37¹⁰³%12 − 1%12 + 12) % 12
Step 2: Compute 37%12 = 1
Step 3: By power property → 37¹⁰³%12 = (37%12)¹⁰³%12 = 1¹⁰³%12 = 1

Quiz 2 — Evaluate (37¹⁰³) % 12
Step 1: 37 % 12 = 1
Step 2: Power property → (37%12)¹⁰³ % 12 = 1¹⁰³ % 12
Step 3: 1 raised to any power = 1 → 1 % 12 = 1
Key Insight Whenever a base a satisfies a % m == 1, then aᵏ % m == 1 for all positive integers k. This is extremely useful for simplifying large exponentials modulo m without computing the full number.

04Applying Mod to the Fast Power Function

When the constraints are 1 ≤ a, n ≤ 10⁹, computing aⁿ directly overflows even a 64-bit integer. We must take mod at every multiplication step. The key change from the plain fast-power function is:

  • Store the half-result p as a long (not int) to avoid overflow before applying mod.
  • Apply % m at each multiplication: (p * p) % m and ((p * p) % m * a) % m.
Java — Modular Fast Power
// Constraints: 1 ≤ a ≤ 10⁹,  1 ≤ n ≤ 10⁹,  2 ≤ m ≤ 10⁹
int pow(int a, int n, int m) {
    if (n == 0) return 1;

    long p = pow(a, n / 2, m);   // long to prevent overflow before mod

    if (n % 2 == 0) {
        return (int) ((p * p) % m);
    } else {
        return (int) ((p * p % m * a) % m);
    }
}
Why long? When m is up to 10⁹ and p = m−1 ≈ 10⁹, then p×p ≈ 10¹⁸, which overflows a 32-bit int (max ≈ 2.1×10⁹) and even a 32-bit unsigned int. A 64-bit long holds up to ~9.2×10¹⁸, so p * p is safe before the % m reduces it back below 10⁹.

Complexity

TimeO(log N) SpaceO(log N)

05Array Sum % M — Applying Mod in Loops

A common competitive programming pattern: compute the sum of an array modulo some large prime, when both N and individual array values can be up to 10⁹ (so the raw sum could exceed 10¹⁸).

Java — Sum of Array % (10⁹+7)
// Constraints: 1 ≤ N ≤ 10⁹,  1 ≤ arr[i] ≤ 10¹⁰
long modSum(int[] arr, int N) {
    int m = 1_000_000_007;  // 10⁹ + 7
    long sum = 0;

    for (int i = 0; i < N; i++) {
        sum = (sum + arr[i]) % m;   // Apply mod every step — never let sum explode
    }

    return sum % m;
}
Best Practice Apply % m inside the loop rather than only at the end. Taking mod at each addition ensures the accumulator sum stays within [0, m−1], so even with N = 10⁹ additions of values up to 10¹⁰, a long never overflows.

06Count Pairs with Sum Divisible by M

Problem Statement Given an array of N integers and a modulus M, count all pairs (i, j) where i ≠ j and (arr[i] + arr[j]) % m == 0. Note: pair (i, j) is the same as (j, i) — count each once.

Idea 1 — Brute Force O(N²)

Check every pair (i, j) with i < j and test if their sum is divisible by M.

Java — Brute Force
int countPairs(int[] arr, int m) {
    int ans = 0;
    for (int i = 0; i < arr.length; i++) {
        for (int j = i + 1; j < arr.length; j++) {
            if ((arr[i] + arr[j]) % m == 0) ans++;
        }
    }
    return ans;
}
TimeO(N²) SpaceO(1)

Idea 2 — Frequency Array O(N + M)

Key insight: (arr[i] + arr[j]) % m == 0 requires that the two remainders complement each other: if arr[i] % m = r, then arr[j] % m must equal m − r. Special cases: when r = 0 both must be 0; when m is even and r = m/2 both must be m/2.

Pair condition: (r₁ + r₂) % m = 0 → r₁ = 0 and r₂ = 0 → r₁ = 1 and r₂ = m−1 → r₁ = 2 and r₂ = m−2 ... → r₁ = k and r₂ = m−k

Build a frequency array farr[m] where farr[r] counts how many elements have remainder r. Then pair up complementary buckets.

Java — Frequency Array Approach
long countPairsOptimal(int[] arr, int m) {
    // Step 1: Build frequency array of remainders
    int[] farr = new int[m];
    for (int i = 0; i < arr.length; i++) {
        farr[arr[i] % m]++;
    }

    long ans = 0;

    // Step 2a: Count pairs where both remainders are 0
    ans += (long) farr[0] * (farr[0] - 1) / 2;  // C(farr[0], 2)

    // Step 2b: If m is even, count pairs where both remainders are m/2
    if (m % 2 == 0) {
        ans += (long) farr[m / 2] * (farr[m / 2] - 1) / 2;
    }

    // Step 2c: Pair complementary buckets l and r = m - l
    int l = 1, r = m - 1;
    while (l < r) {
        ans += (long) farr[l] * farr[r];
        l++; r--;
    }

    return ans;
}
TimeO(N + M) SpaceO(M)
Why C(n, 2)? When all k elements have remainder 0, every pair among them is valid. The number of ways to choose 2 from k is k(k−1)/2 — this is the combination formula C(k, 2). The same applies to the m/2 bucket when m is even.

07Greatest Common Divisor (GCD)

GCD(A, B) (also called the Highest Common Factor, HCF) is the largest positive integer that divides both A and B without a remainder.

GCD(A, B) = x → A % x = 0 AND B % x = 0, where x is the highest such number.

Quick Examples

GCD(20, 65)

Divisors of 65: 1,5,13,65

Coprime Numbers When GCD(A, B) = 1, the numbers A and B are called coprime (or relatively prime). They share no common prime factor.

08Properties of GCD

①
Commutative
GCD(A, B) = GCD(B, A)
Order of arguments does not matter.
②
Zero Identity
GCD(0, A) = |A|
Zero contributes nothing — the GCD equals the absolute value of the other number.
③
Associative
GCD(A, B, C) = GCD(GCD(A,B), C)
Can be computed pairwise in any order.
④
Subtraction Property ⭐
If A, B > 0 and A > B:
GCD(A, B) = GCD(A−B, B)
This is the key lemma behind the Euclidean algorithm.

Proof of the Subtraction Property

Suppose GCD(A, B) = x. By definition, x divides both A and B. We want to show x also divides (A − B).

A % x = 0 → A = k₁ × x for some integer k₁ B % x = 0 → B = k₂ × x for some integer k₂ (A − B) = (k₁ − k₂) × x → (A − B) % x = 0 ✓

Since x divides both (A−B) and B, x is also a common divisor of (A−B, B). One can further show x is the greatest such divisor, giving GCD(A, B) = GCD(A−B, B).

09The Euclidean Algorithm

The subtraction property works but can be slow — to reduce GCD(23, 5) by subtraction alone requires many steps (23→18→13→8→3→…). The modulo operation is equivalent to repeated subtraction but does it all at once:

GCD(A, B) = GCD(A − B, B) = GCD(A − 2B, B) = GCD(A − 3B, B) ... (repeatedly subtract B) = GCD(A % B, B) ← all in one step!

Combined with commutativity, we get the famous Euclidean recurrence:

GCD(A, B) = GCD(B, A % B)

Worked Example — GCD(23, 5)

gcd(3, 5)returns gcd(5,3)
23 % 5 = 3
gcd(5, 3)returns gcd(3,2)
5 % 3 = 2
gcd(3, 2)returns gcd(2,1)
3 % 2 = 1
gcd(2, 1)returns gcd(1,0)
2 % 1 = 0
gcd(1, 0)b=0 → returns 1

GCD(23, 5) = 1

Worked Example — GCD(24, 16)

gcd(8, 0)b=0 → returns 8
16 % 8 = 0
gcd(16, 8)→ 8
24 % 16 = 8
gcd(24, 16)→ 8

GCD(24, 16) = 8 — only 2 steps!

Why It's Fast Each call to gcd(a, b) reduces the larger argument to a % b, which is always less than b. At least every two steps the smaller argument is halved. The total number of steps is bounded by O(log min(a, b)).

10GCD Function — Recursive Implementation

Java — Euclidean GCD
// Assumption: gcd(a, b) returns GCD of a and b (works for a, b ≥ 0)
int gcd(int a, int b) {
    if (b == 0) return a;      // Base case: GCD(a, 0) = a
    return gcd(b, a % b);       // Euclidean step
}

Complexity

TimeO(log min(a,b)) SpaceO(log min(a,b))
Fibonacci = Worst Case for GCD Consecutive Fibonacci numbers (e.g., gcd(23, 14) where 23=F₈, 14=F₇) achieve the maximum number of steps for a given input size. This is why the Fibonacci sequence proves the O(log min(a,b)) bound for the Euclidean algorithm.

GCD of More than Two Numbers

By the associative property, compute pairwise: GCD(A, B, C) = GCD(GCD(A, B), C).

Java — GCD of Array
int gcdOfArray(int[] arr) {
    int result = arr[0];
    for (int i = 1; i < arr.length; i++) {
        result = gcd(result, arr[i]);
    }
    return result;
}
