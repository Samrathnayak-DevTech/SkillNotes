01

Sum of All Subarray Sums — Brute Force O(N³)

Problem (Google/Facebook): Given arr[N], find the sum of all subarray sums.

Example — arr = [1, 2, 3]

SubarraySum
[1]1
[1, 2]3
[1, 2, 3]6
[2]2
[2, 3]5
[3]3
Total20
java · Idea 1 — O(N³)
long ans = 0;
for (int i = 0; i < N; i++) {
    for (int j = i; j < N; j++) {
        int sum = 0;
        for (int k = i; k <= j; k++) sum += arr[k];
        ans += sum;
    }
}
// T.C = O(N³)   S.C = O(1)
02

Carry Forward the Sum — O(N²)

Instead of recomputing the sum of each subarray from scratch, carry the running sum as we extend the right endpoint. This drops one loop.

java · Idea 3 — Carry Forward O(N²)
long ans = 0;
for (int i = 0; i < N; i++) {
    int sum = 0;
    for (int j = i; j < N; j++) {
        sum += arr[j];    // carry forward: don't restart from i
        ans += sum;
    }
}
// T.C = O(N²)   S.C = O(1)
Still Not EnoughFor N = 10⁶ with constraint 1 ≤ N ≤ 10⁶, O(N²) = 10¹² — still TLE. We need O(N).
03

Contribution Technique — O(N)

Key QuestionInstead of summing subarray-by-subarray, ask: how many times does element arr[i] appear across all subarray sums? Each appearance contributes arr[i] × (count of subarrays containing index i).

In how many subarrays does index i appear?

A subarray containing index i has its start (si) somewhere in [0, i] and its end (ei) somewhere in [i, N–1]. The number of choices is:

Options for si: (i + 1)   ×   Options for ei: (N – i)   =   (i + 1) × (N – i)

Verification — arr = [3, –2, 1, 4]

Index iarr[i](i+1)(N–i) with N=4Contribution
03143 × 4 = 12
1–223–2 × 6 = –12
21321 × 6 = 6
34414 × 4 = 16
Total22 ✓
java · Contribution Technique — O(N)
long ans = 0;
for (int i = 0; i < N; i++) {
    ans += (long) arr[i] * (i + 1) * (N - i);
}
return ans;
// T.C = O(N)   S.C = O(1)
Overflow WarningCast to long before multiplying. With arr[i] up to 10⁹ and N up to 10⁶, the product can reach 10²¹ — well beyond int range.

Number of Subarrays of Length K

For an array of length N, the number of distinct subarrays of a specific length follows a simple pattern:

LengthCount
1N
2N–1
3N–2
KN – K + 1
N1
FormulaNumber of subarrays of length K = N – K + 1
Example: N = 7, K = 4 → 7 – 4 + 1 = 4 subarrays

Max Subarray Sum of Length K — Brute & psum

Problem: Given arr[N] and integer K, find the maximum sum among all subarrays of exactly length K.

arr = [–3, 4, –2, 5, 3, –2, 8, 2, –1, 4], K = 5

sieiSum

3716 ← max

java · Brute Force O(N×K) ≈ O(N²)
int ans = Integer.MIN_VALUE, si = 0, ei = K - 1;
while (ei < N) {
    int sum = 0;
    for (int i = si; i <= ei; i++) sum += arr[i];
    ans = Math.max(ans, sum);
    si++; ei++;
}
return ans;
// T.C = O(N×K)   S.C = O(1)
06

Sliding Window — O(N)

Sliding Window IdeaWhen the window slides one step right, only two elements change: the leftmost element leaves and a new element enters. So instead of recomputing the sum from scratch, update it:

sum = sum – arr[si–1] + arr[ei]

Trace — arr = [–3, 4, –2, 5, 3, –2, 8, 2, –1, 4], K = 5

si=0, ei=4  → sum = –3+4–2+5+3 = 7
si=1, ei=5  → sum = 7 – arr[0] + arr[5] = 7–(–3)+(–2) = 8
si=2, ei=6  → sum = 8 – arr[1] + arr[6] = 8–4+8 = 12
si=3, ei=7  → sum = 12 – arr[2] + arr[7] = 12–(–2)+2 = 16 ← max
si=4, ei=8  → sum = 16 – arr[3] + arr[8] = 16–5+(–1) = 10
si=5, ei=9  → sum = 10 – arr[4] + arr[9] = 10–3+4 = 11
java · Sliding Window — O(N)
// Step 1: Create the first window (sum of first K elements)
int sum = 0;
for (int i = 0; i < K; i++) sum += arr[i];
int ans = sum;

// Step 2: Slide the window — remove leftmost, add rightmost
int si = 1, ei = K;
while (ei < N) {
    sum = sum - arr[si - 1] + arr[ei];
    ans = Math.max(ans, sum);
    si++; ei++;
}
return ans;
// T.C = O(N)   S.C = O(1)
// Sliding Window = Carry Forward + Fixed Subarray Size
Brute Force O(N×K) Sliding Window O(N) Space O(1)
When to Use Sliding WindowUse the sliding window technique whenever the subarray size is fixed. The window moves forward one step at a time, updating in O(1) instead of recomputing in O(K).

Quick Recap

Sum of All Subarray Sums

Three approaches: O(N³) brute, O(N²) carry-forward, O(N) contribution technique.

Contribution Technique

arr[i] appears in (i+1) × (N–i) subarrays. Total sum = Σ arr[i] × (i+1) × (N–i).

Subarrays of Length K

Count = N – K + 1. Each window starts at a unique index si ∈ [0, N–K].

Sliding Window

Fixed window size: build first window in O(K), then slide in O(1) per step. Total O(N).

Window Update Formula

sum = sum – arr[si–1] + arr[ei]. Remove the element that just left, add the new one.

Sliding Window = ?

Carry Forward (reuse previous result) + Fixed Subarray Size = Sliding Window.
