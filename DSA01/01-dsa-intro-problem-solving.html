01

Factors

An integer i is a factor of N if it divides N completely — that is, N % i == 0. Every positive integer N has at least two factors: 1 and itself.

Key Definition i is a factor of N if and only if N % i == 0.

Examples

NFactorsCount
241, 2, 3, 4, 6, 8, 12, 248
101, 2, 5, 104
201, 2, 4, 5, 10, 206

Brute-Force Approach

Check every number from 1 to N. If it divides N, increment the count. This runs in O(N) time.

java · O(N) 
int countFactors(int N) {
    int count = 0;
    for (int i = 1; i <= N; i++) {
        if (N % i == 0) {
            count++;
        }
    }
    return count;
}

Execution Time Analysis

Assuming a server executes roughly 108 iterations per second, the brute-force approach becomes dangerously slow for large N:

NIterationsApproximate Time
1081081 second
10910910 seconds
10181018~317 years
Watch Out An O(N) solution is completely infeasible when N can be as large as 1018. Always check constraints before choosing an algorithm.

Optimised Factor Count — O(√N)

The key observation is that factors come in pairs. If i is a factor of N, then N/i is also a factor. For example, for N = 36:

iN/i = 36/iRunning count
1362
2184
3126
498
66 (same!)9 — perfect square
94← mirror, skip
123
182
361
Key Insight Factors always appear in mirrored pairs around √N. So we only need to iterate from 1 to √N, counting two factors per iteration — unless i == N/i (a perfect square), in which case we count just one.

The condition i ≤ N/i is equivalent to i * i ≤ N, which means i ≤ √N. This reduces iterations from N to √N.

java · O(√N) 
int countFactors2(int N) {
    int count = 0;
    for (int i = 1; i * i <= N; i++) {
        if (N % i == 0) {
            if (i == N / i) {
                count++;          // perfect square — count once (e.g., i=6 for N=36)
            } else {
                count += 2;      // i and N/i are two distinct factors
            }
        }
    }
    return count;
}
Time O(√N) Space O(1)
Impact For N = 1018, the brute-force O(N) approach takes ~317 years. The O(√N) optimisation reduces that to roughly 10 seconds — the same server, a billion times faster.
Golden Rule The main ingredient to improving problem-solving is observation. Look for patterns, symmetry, and redundancy in the problem structure.
03

Prime Numbers

A prime number has exactly two distinct positive factors: 1 and itself. The number 1 is neither prime nor composite by convention.

Definition N is prime ⟺ countFactors(N) == 2

Quick Quiz

Practice From the list [10, 11, 23, 2, 25, 27, 31], identify the prime numbers.
Show Answer

Primes: 11, 23, 2, 31 — four primes. Note: 25 = 5×5, 27 = 3×3×3, 10 = 2×5.

Pseudo Code

java · O(√N) 
boolean checkPrime(int N) {
    if (countFactors2(N) == 2) {
        return true;
    } else {
        return false;
    }
}
// Equivalently: return countFactors2(N) == 2;
Time O(√N) Space O(1)
04

Sum of N Natural Numbers

Gauss derived the formula for the sum of the first N natural numbers using a brilliant symmetry trick.

Analogy — Gauss's Trick Write the sum twice: once forward and once backward. Pair up the columns — each pair sums to (N+1). There are N such pairs, so 2S = N(N+1).
S  = 1  + 2  + 3  + … + (N–1) + N
S  = N + (N–1) + (N–2) + … + 2    + 1
2S = (N+1) + (N+1) + (N+1) + … ← N terms
2S = N(N+1)   ⟹   S = N(N+1) / 2
Formula Sum of first N natural numbers = N × (N + 1) / 2
Example: Sum 1 to 100 = 100 × 101 / 2 = 5050

This is an O(1) formula — no loops needed, regardless of how large N is.

Time O(1) Space O(1)
05

Geometric Progression (GP)

A geometric progression is a sequence where the ratio between any two consecutive terms is constant. That constant is called the common ratio (r).

Example 5, 10, 20, 40, 80, 160, 320, 640 — common ratio r = 2
Terms:   a,   ar,   ar²,   ar³,   …   arN–1
Nth term = arN–1

Sum of First N Terms

Using the multiply-and-subtract trick (multiply S by r, then subtract S from rS to cancel most terms):

rS – S = arN – a
S(r – 1) = a(rN – 1)
S = a(rN – 1) / (r – 1)   (when r ≠ 1)
Why This Matters in DSA Many loop iteration counts form a GP. For example, Quiz-9 in the Time Complexity notes (nested loop where inner runs 2i times) gives total iterations = 2 + 4 + 8 + … + 2N — which is a GP summing to 2(2N – 1), i.e., O(2N).
06

What Is an Iteration?

An iteration is simply the number of times a loop body executes. Counting iterations precisely is the foundation of time complexity analysis.

Core Principle An iteration is one complete execution of the loop body — regardless of how many instructions that body contains.

Iteration Quizzes

Quiz 1 — Count the iterations
pseudocode
for(i = 1; i <= N; i++) {
    if(i == N) { break; }
}

Answer: N iterations. The loop runs i = 1, 2, …, N. Even though the break fires when i = N, the body still executes N times (the break is inside the body on the Nth execution).

Quiz 2 — Count the iterations
pseudocode
s = 0;
for(int i = 0; i <= 100; i++) {
    s = s + i + i*i;
}

Answer: 101 iterations. The range [0, 100] inclusive contains 100 – 0 + 1 = 101 values. The number of operations inside the body does not affect iteration count.

Quiz 3 — Two sequential loops
pseudocode
for(i = 1; i <= N; i++) {
    if(i % 2 == 0) { print(i); }
}
for(j = 1; j <= m; j++) {
    if(j % 2 == 0) { print(j); }
}

Answer: N + m iterations total. Sequential (non-nested) loops add up; they do not multiply.

07

How to Compare Two Algorithms

Execution time on a specific machine is a terrible metric — it depends on hardware, language, operating system, compiler optimizations, and more. Two developers running the same algorithm can get completely different times on different machines.

Analogy Vikas runs Algorithm 1 on Windows XP — 15 seconds. Ankit runs Algorithm 2 on a MacBook Pro — 10 seconds. Does that mean Algorithm 2 is better? No. Running Vikas's algorithm on the same MacBook Pro gives 7 seconds, and on a faster machine it drops to 5 seconds — equal to Ankit's. Execution time tells us about the machine, not the algorithm.
Correct Metric Compare algorithms using the number of iterations as a function of input size N. This is machine-independent, language-independent, and scales predictably.

This is the foundation of asymptotic analysis — which is formalised in the next session on Time Complexity using Big-O notation.

Range Notation

When counting iterations in loops, it is essential to know how many integers fall within a given range. The bracket/parenthesis notation encodes whether endpoints are included or excluded.

NotationMeaningCount
[a, b]Both endpoints included (closed)b – a + 1
[a, b)Left included, right excluded (half-open)b – a
(a, b]Left excluded, right includedb – a
(a, b)Both endpoints excluded (open)b – a – 1

[1, 11]

11 – 1 + 1 = 11 integers: {1, 2, 3, …, 11}

📐

[3, 10]

Quick Recap

Factors — O(N)

Check all i ∈ [1, N]. Simple but slow for large N.

Factors — O(√N)

Factors come in pairs around √N; iterate only to √N counting both at once.

Prime Check

A number is prime iff countFactors(N) == 2. Runs in O(√N).

Gauss Sum

Sum of 1…N = N(N+1)/2. O(1) — no loop needed.

GP Sum

Sum of N terms = a(rN–1)/(r–1). Essential for analysing exponential loops.

Iterations

Count how many times the loop body executes, not instructions inside it.

Range Count

[a,b] → b–a+1  |  [a,b) → b–a  |  (a,b) → b–a–1

Compare Algos

Use iteration count, not execution time. Time varies by machine; iterations don't.
