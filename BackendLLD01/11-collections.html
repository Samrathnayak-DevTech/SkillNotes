LinkedList & Vector

LinkedList

LinkedList is a doubly-linked list — each node stores a value and pointers to the previous and next nodes. Memory is not contiguous. Fast insertion/deletion anywhere (O(1) if you have the node), but O(n) random access since you must traverse from the head.

Vector

Vector is essentially a thread-safe version of ArrayList. Every method is synchronized, which makes it safe for concurrent access but slower than ArrayList for single-threaded use. Prefer ArrayList for single-threaded code, and Collections.synchronizedList or CopyOnWriteArrayList for concurrent access.

When to Choose

Use ArrayList when you need fast random access (get by index). Use LinkedList when you frequently insert/delete in the middle or use it as a queue/deque. Avoid Vector in modern code — it's a legacy class.

bucket = hash(x) % array_length. Collisions resolved by chaining (linked list per bucket). The load factor λ = avg elements per bucket. When λ > 0.75, the array is rehashed (doubled in size).

Java — HashSet add() logic
void add(T x) {
    int bucket = x.hashCode() % A.length;
    boolean found = false;
    for (Node node : A[bucket]) {
        if (node.val.equals(x)) { found = true; break; }
    }
    if (!found) A[bucket].add(new Node(x));  // only add if not already present
}
// TC: O(λ) ≈ O(1) average. When λ > 0.75: rehash → O(n) but amortized O(1).
06

LinkedHashSet & TreeSet

LinkedHashSet

LinkedHashSet extends HashSet with an additional doubly-linked list threading through all nodes in insertion order. It gives you O(1) add/contains like HashSet, but iteration is in the order elements were inserted.

TreeSet

TreeSet is backed by a self-balancing binary search tree (Red-Black tree). All elements are kept in sorted order at all times. Operations are O(log n) instead of O(1), but you get sorted traversal for free.

07

Queue, PriorityQueue & Deque

PriorityQueue

PriorityQueue is a heap-based queue. By default it is a min-heap — poll() always returns the smallest element. Use Collections.reverseOrder() to make it a max-heap.

Java
// Min-heap (default)
PriorityQueue<Integer> minPQ = new PriorityQueue<>();

// Max-heap
PriorityQueue<Integer> maxPQ = new PriorityQueue<>(Collections.reverseOrder());

minPQ.offer(5); minPQ.offer(1); minPQ.offer(3);
minPQ.poll();   // returns 1 (smallest)  — O(log n)
minPQ.peek();   // returns next min without removing  — O(1)

Deque (Double-Ended Queue)

A Deque supports O(1) insertion and removal at both ends — it can act as a stack (LIFO) or a queue (FIFO). ArrayDeque is the preferred concrete implementation (faster than LinkedList for most use cases).

Java
Deque<Integer> deque = new ArrayDeque<>();
deque.addFirst(1);   // front
deque.addLast(2);    // back
deque.pollFirst();   // remove from front
deque.pollLast();    // remove from back

// Use as Stack (LIFO):
deque.push(5);  // same as addFirst
deque.pop();   // same as removeFirst

Quick Recap & Cheat Sheet

ArrayList

Dynamic array. O(1) get by index. O(1) amortized add-at-end. O(n) insert/delete in middle. Default capacity 10, doubles on resize.

LinkedList

Doubly-linked list. O(1) add/remove at head/tail. O(n) get by index. Implements both List and Deque.

Vector

Thread-safe ArrayList (legacy). Every method synchronized. Prefer CopyOnWriteArrayList for modern concurrent code.

HashSet

Hash table backed. O(1) avg add/contains/remove. No order. No duplicates. Rehashes when load factor > 0.75.

LinkedHashSet

HashSet + insertion-order doubly-linked list. O(1) operations + predictable iteration order.

TreeSet

Red-Black BST. Always sorted. O(log n) operations. No nulls allowed. Use when you need sorted unique elements.

PriorityQueue

Min-heap by default. O(log n) offer/poll, O(1) peek. Use for k-th smallest/largest problems.

ArrayDeque

Double-ended queue. O(1) at both ends. Preferred stack/queue implementation over Stack and LinkedList.

HashMap

Hash table for key-value pairs. O(1) avg get/put. No order. Allows one null key. Not thread-safe.

TreeMap

Red-Black BST map. Keys always sorted. O(log n) operations. Useful for range queries: firstKey(), subMap().
