2-D Array Basics

A 2-D array arr[N][M] is a matrix with N rows (indexed 0 to N−1) and M columns (indexed 0 to M−1). The total number of elements is N × M.

Any element is accessed as arr[i][j] — row first, column second. The top-left corner is arr[0][0] and the bottom-right corner is arr[N−1][M−1].

02

How a 2-D Array Is Actually Stored

In Java, int[][] arr = new int[N][M] creates an array of arrays on the heap. The outer array holds N references; each reference points to an independent 1-D inner array of length M.

Analogy Think of the outer array as a bookshelf (N slots). Each slot holds a reference (a label with a shelf address). Each label points to a separate physical shelf (the inner 1-D array) where M actual books live.
Memory Layout Stack: arr → 4K (reference to the outer array).
Heap at 4K: [8K, 4K2, 15K, 20K, …] — N row references.
Heap at 8K: [0, 0, 0, 0] — row 0, M elements.
Heap at 4K2: [0, 0, 0, 0] — row 1, M elements, and so on.

Accessing arr[2][3] involves two pointer dereferences: first, get the address of row 2 from the outer array, then index element 3 in that row. Both steps are O(1).

Row-Wise Sum

Problem Given arr[N][M], print the sum of each row.

Idea: For every row i, iterate over all columns j and accumulate the sum. Reset the accumulator at the start of each row.

void printRowWiseSum(int[][] arr, int N, int M) {
    for (int i = 0; i < N; i++) {
        int sum = 0;
        for (int j = 0; j < M; j++) {
            sum += arr[i][j];
        }
        System.out.println(sum);
    }
}
Time O(N×M) Space O(1)

void printColWiseSum(int[][] arr, int N, int M) {
    for (int j = 0; j < M; j++) {    // outer: column
        int sum = 0;
        for (int i = 0; i < N; i++) { // inner: row
            sum += arr[i][j];
        }
        System.out.println(sum);
    }
}
Time O(N×M) Space O(1)

Print All Diagonals (Right → Left)

Problem Given arr[N][M], print all diagonals going from the top-right direction toward the bottom-left. Total diagonals = N + M − 1.
Key Insight Each diagonal can be identified by its starting point. Diagonals in the first group start from row 0 (columns 0 to M−1). Diagonals in the second group start from column M−1 (rows 1 to N−1). Within each diagonal, move i++, j−− until out of bounds.

void printAllDiagonals(int[][] arr, int N, int M) {

    // Group 1: diagonals starting from row 0, columns 0..M-1
    for (int k = 0; k < M; k++) {
        int i = 0, j = k;
        while (i < N && j >= 0) {
            System.out.print(arr[i][j] + " ");
            i++; j--;
        }
        System.out.println();
    }

    // Group 2: diagonals starting from column M-1, rows 1..N-1
    for (int k = 1; k < N; k++) {
        int i = k, j = M - 1;
        while (i < N && j >= 0) {
            System.out.print(arr[i][j] + " ");
            i++; j--;
        }
        System.out.println();
    }
}
Time O(N×M) Space O(1)
Storing Diagonal Elements If you need to store all diagonals in a 2-D structure, use a dynamic array (e.g., ArrayList<ArrayList<Integer>>) since each diagonal has a different length. Avoid statically sized 2-D arrays for this — the notes correctly flag this as a common mistake.

Transpose of a Square Matrix

Definition The transpose of a matrix swaps rows and columns: element at (i, j) moves to (j, i). For a square matrix this can be done in-place in O(N²) time and O(1) space.

Idea: Only swap elements in the upper triangle (where j > i) with their mirror in the lower triangle. The main diagonal stays fixed.

java — in-place transpose (square only)
void transposeSquare(int[][] arr, int N) {
    for (int i = 0; i < N; i++) {
        for (int j = i + 1; j < N; j++) {  // upper triangle only
            int temp  = arr[i][j];
            arr[i][j] = arr[j][i];
            arr[j][i] = temp;
        }
    }
}
// Note: if j loops from 0 to N-1 (not i+1), you re-swap every pair
//       and end up with the original matrix unchanged — a common bug!
Time O(N²) Space O(1)

int[][] transposeRect(int[][] arr, int N, int M) {
    int[][] t = new int[M][N]; // note: M rows, N columns
    for (int i = 0; i < N; i++) {
        for (int j = 0; j < M; j++) {
            t[j][i] = arr[i][j];
        }
    }
    return t;
}
Time O(N×M) Space O(M×N)

Rotate a Square Matrix 90° Clockwise

Two-Step Algorithm (In-Place) Step 1: Transpose the matrix in-place.
Step 2: Reverse every row in-place.
Why Does This Work? In a 90° clockwise rotation, column j of the original becomes row j of the result, but in reversed order. Transposing first turns column j into row j. Reversing each row then gives the correct right-to-left ordering of what was originally column j.

Rotated 90° clockwise
①
Transpose in-place: for i in [0,N), for j in [i+1,N) → swap arr[i][j] with arr[j][i].
②
Reverse each row: for each row i → two-pointer swap from left and right ends meeting in the middle.
java — rotate 90° clockwise in-place
void rotate90CW(int[][] arr, int N) {
    // Step 1: Transpose
    for (int i = 0; i < N; i++) {
        for (int j = i + 1; j < N; j++) {
            int temp = arr[i][j];
            arr[i][j] = arr[j][i];
            arr[j][i] = temp;
        }
    }
    // Step 2: Reverse each row
    for (int i = 0; i < N; i++) {
        int l = 0, r = N - 1;
        while (l < r) {
            int temp = arr[i][l];
            arr[i][l] = arr[i][r];
            arr[i][r] = temp;
            l++; r--;
        }
    }
}
Time O(N²) Space O(1)

180° Rotation

A 180° clockwise rotation is equivalent to flipping the matrix both horizontally and vertically. The cleanest in-place approach: reverse all rows, then reverse all columns (i.e., reverse each row, then reverse the array of rows itself).

Rotated 180°
Algorithm Step 1: Reverse all rows (reverse the order of rows in the matrix — swap row 0 with row N−1, etc.).
Step 2: Reverse each individual row (reverse left-to-right within each row).
This is equivalent to rotating 90° clockwise twice.

270° Clockwise (= 90° Counter-Clockwise)

For 270° clockwise rotation, apply the reverse steps of 90° CW: reverse each row first, then transpose. Alternatively: transpose first, then reverse all rows top-to-bottom.

Think Carefully The notes deliberately leave 270° as an exercise — work it out from first principles. Take a small example (e.g., 2×2 or 3×3) and trace what happens to each element. Confirm your formula before implementing.

Quick Recap

2-D Array

arr[N][M] — N rows, M columns, N×M total elements. Accessed as arr[i][j], row-major.

Storage in Java

Array of arrays on the heap. The outer array holds N references to independent inner row arrays. Access is two pointer dereferences — O(1).

Row-Wise Sum

Outer loop over rows, inner loop over columns. Reset accumulator per row. O(N×M) time.

Column-Wise Sum

Outer loop over columns, inner loop over rows. Same complexity, different loop order.

Main Diagonal

Elements where i == j. Walk with i++, j++ until i < N.

All Diagonals (R→L)

N+M−1 total. Group 1 starts from row 0 (k=0..M-1); Group 2 from column M-1 (k=1..N-1). Within each: i++, j−−.

Transpose (Square)

Swap arr[i][j] with arr[j][i] for j > i only (upper triangle). O(N²) time, O(1) space.

Rotate 90° CW

Transpose + reverse each row. Both in-place. O(N²), O(1).

Rotate 180°

Reverse row order + reverse each row. Equivalent to two 90° CW rotations.
