01

Atmost 1 Replace — Problem

Question: Given a binary array, you may replace at most one 0 with 1. Find the maximum number of consecutive 1s possible after the replacement.

[1,1,0,1,1,0,1,1,1] → ans = 6
[0,1,1,1,0,1,1,0,1,1,0] → ans = 6
[1,1,1,1,1] → ans = 5 (already all 1s)
[0,0,0,0,0] → ans = 1 (only one 0→1 allowed)

Brute Force Idea

For every 0 in the array, count consecutive 1s on its left (l) and right (r). The answer at that position is l + r + 1.

arr = [0,1,1,1,0,1,1,0,1,1,0]
index 0: l=0, r=3 → 0+3+1=4
index 4: l=3, r=2 → 3+2+1=6 ← max
index 7: l=2, r=2 → 2+2+1=5
index 10: l=2, r=0 → 2+0+1=3
02

Atmost 1 Replace — Solution

Java
int maxConsecutiveOnes(int[] arr) {
    int N = arr.length;
    // Edge cases
    int totalOnes = 0;
    for (int x : arr) if (x == 1) totalOnes++;
    if (totalOnes == N) return N;   // all 1s
    if (totalOnes == 0) return 1;   // all 0s → replace 1

    int ans = 0;
    for (int i = 0; i < N; i++) {
        if (arr[i] == 0) {
            int l = 0, r = 0;
            for (int j = i - 1; j >= 0; j--) {
                if (arr[j] == 1) l++; else break;
            }
            for (int j = i + 1; j < N; j++) {
                if (arr[j] == 1) r++; else break;
            }
            ans = Math.max(ans, l + r + 1);
        }
    }
    return ans;
}
T.C O(N)
S.C O(1)
Why O(N) Not O(N²)? Each element is accessed at most 3 times across all iterations of the three loops — once during the main scan, once from a left expansion, once from a right expansion. So total work is 3N = O(N).
Key Insight Sometimes the brute-force approach IS the most optimal. Deeper analysis of access patterns can reveal that a seemingly nested loop is actually linear.
03

Atmost 1 Swap — Problem

Question: Given a binary array, you may swap a single 0 with a 1 (swapping positions, not replacing). Find the maximum consecutive 1s after at most 1 swap.

The key difference: swapping a 0 with a 1 means the 1 disappears from somewhere else — the total count of 1s is preserved.

arr=[1,1,0,1,1,1] → swap 0 at idx 2 with 1 at idx 5 → ans = 5
arr=[1,1,0,1,1,1,0,1] → totalOnes=5
   At idx 2: l=2, r=3, l+r=5=totalOnes → ans = 5 (no +1)
   At idx 6: l=3, r=1, l+r=4 < 5 → ans = 4+1 = 5
Replace vs Swap Replace: adding a new 1 (from outside), so always +1. Swap: borrowing a 1 from elsewhere. If all 1s are accounted for (l+r == totalOnes), there is no spare 1 to fill the gap — don't add +1.
04

Atmost 1 Swap — Solution

Java
int maxConsecutiveSwap(int[] arr) {
    int N = arr.length;
    int totalOnes = 0, ans = 0;
    for (int x : arr) if (x == 1) totalOnes++;
    if (totalOnes == N) return N;
    if (totalOnes == 0) return 0;

    for (int i = 0; i < N; i++) {
        if (arr[i] == 0) {
            int l = 0, r = 0;
            for (int j = i - 1; j >= 0; j--) {
                if (arr[j] == 1) l++; else break;
            }
            for (int j = i + 1; j < N; j++) {
                if (arr[j] == 1) r++; else break;
            }
            // If l+r uses ALL ones, no spare to fill this gap
            if (l + r == totalOnes) {
                ans = Math.max(ans, l + r);
            } else {
                ans = Math.max(ans, l + r + 1);
            }
        }
    }
    return ans;
}
T.C O(N)
S.C O(1)

Majority Element — Problem & Approaches

Question: Given array of N elements, find the majority element — the element that appears more than N/2 times. You can assume it always exists.

[2,1,4] → No majority
[3,4,3,2,4,4,4,4] → ans = 4
[3,4,3,6,1,3,2,5,3,3,3] N=11 → ans = 3

Key Observation: At Most One Majority Element

Proof by contradiction: if both a and b are majority elements, then freq(a) + freq(b) > N/2 + N/2 = N. But the array only has N elements — contradiction. Therefore at most one majority element exists.

Approach 1 — Brute Force

For every element, count its frequency with a nested loop.

T.C O(N²)
S.C O(1)

Approach 2 — Sorting

Sort the array. The majority element must occupy the middle index N/2, since it appears > N/2 times.

T.C O(N log N)
S.C O(depends on sort)
06

Majority Element — Boyer-Moore Voting

The optimal O(N)/O(1) algorithm exploits the key observation: if you keep deleting two distinct elements, the majority element survives.

Intuition Imagine a battle: whenever two distinct elements meet, they cancel each other out. The majority element (with count > N/2) always has more soldiers than all opponents combined — it always wins the last battle.

Trace on [3,4,3,6,1,3,2,5,3,3,3]

majority=3, freq=1 → 4 (diff) → freq=0
→ 3 (freq=0) → new majority=3, freq=1
→ 6 (diff) → freq=0
→ 1 (freq=0) → new majority=1, freq=1
→ 3 (diff) → freq=0
→ 2 (freq=0) → new majority=2, freq=1
→ 5 (diff) → freq=0
→ 3 (freq=0) → new majority=3, freq=1
→ 3 (same) → freq=2
→ 3 (same) → freq=3   ← final candidate: 3
Verify: freq(3) = 6 > 11/2=5.5 ✓
Java — Boyer-Moore Voting O(N) / O(1)
int majorityElement(int[] arr) {
    int N = arr.length;
    int majority = arr[0], freq = 1;

    for (int i = 1; i < N; i++) {
        if (freq == 0) {
            majority = arr[i];
            freq = 1;
        } else if (arr[i] == majority) {
            freq++;
        } else {
            freq--;   // cancel two distinct elements
        }
    }

    // Verify (needed if majority not guaranteed to exist)
    freq = 0;
    for (int x : arr) if (x == majority) freq++;
    if (freq > N / 2) return majority;
    else return -1;  // no majority element
}
T.C O(N)
S.C O(1)

In-place Approach: Mark Negatives

Since values are non-negative (≥ 0), use −1 as a temporary marker. In two passes: first, mark the row/column entries of any zero as −1 (skipping existing zeros). Second, convert all −1 back to 0.

Why Not Zero Directly? If you write zeros immediately, later iterations can't tell if a zero was original or just added — causing incorrect propagation. Use a distinct sentinel like −1 first.
08

Row-Column Zero — Code

Java — In-place O(N×M) / O(1)
void rowColZero(int[][] arr, int N, int M) {

    // Pass 1a: mark entire row −1 if a zero is found
    for (int i = 0; i < N; i++) {
        boolean flag = false;
        for (int j = 0; j < M; j++) {
            if (arr[i][j] == 0) { flag = true; break; }
        }
        if (flag) {
            for (int j = 0; j < M; j++) {
                if (arr[i][j] > 0) arr[i][j] = -1;
            }
        }
    }

    // Pass 1b: mark entire col −1 if a zero is found
    for (int j = 0; j < M; j++) {
        boolean flag = false;
        for (int i = 0; i < N; i++) {
            if (arr[i][j] == 0) { flag = true; break; }
        }
        if (flag) {
            for (int i = 0; i < N; i++) {
                if (arr[i][j] > 0) arr[i][j] = -1;
            }
        }
    }

    // Pass 2: convert all −1 → 0
    for (int i = 0; i < N; i++)
        for (int j = 0; j < M; j++)
            if (arr[i][j] < 0) arr[i][j] = 0;
}
T.C O(N×M)
S.C O(1)

Quick Recap

Atmost 1 Replace

For each 0, count consecutive 1s on left (l) and right (r). Answer = max(l+r+1). T.C = O(N) — each element accessed ≤ 3 times.

Atmost 1 Swap

Same as replace, but if l+r equals totalOnes, there's no spare 1 to borrow — answer is l+r (not +1). Total 1s stay fixed.

Majority Element Proof

At most one majority can exist: if freq(a) > N/2 and freq(b) > N/2 then their combined count exceeds N — impossible.

Boyer-Moore Voting

Keep a candidate and frequency. On mismatch: freq--. When freq hits 0, switch candidate. Surviving candidate is verified in a second pass.

Row-Column Zero

Two-pass in-place: mark affected rows/columns as −1 (skip existing 0s), then convert all −1 to 0. O(N×M) time, O(1) space.

Brute ≠ Slow

The max-consecutive-ones solution looks like O(N²) but is O(N) — because each element is touched at most 3 times total across all loop iterations.
