What Is Sorting?

Sorting is the arrangement of data in a particular order based on some parameter. The most common parameter is numeric magnitude (ascending or descending), but any comparable property works.

02

Why Sort?

Sorting is a prerequisite for many efficient algorithms. It transforms an unstructured dataset into an ordered one that admits much faster operations.

🔍

Faster Search

Binary search on a sorted array runs in O(log N) instead of O(N) linear scan. Many other lookup optimizations require sorted data.

📊

Easier Analysis

Statistics like median, mode, and percentiles are trivial on sorted data. Range queries, duplicates detection, and frequency counts all simplify.

🗂️

Better Presentation

Leaderboards, search results, file listings — users expect ordered output. Sorting makes data human-readable and actionable.

⚡

Enables Greedy Algorithms

Many greedy strategies (interval scheduling, activity selection, Noble Integers) only work correctly on sorted input.

03

Built-In Sort APIs

In competitive programming and interviews, always use the built-in sort unless you are specifically asked to implement one. Built-in implementations are typically O(N log N) and highly optimized.

Java
int[] arr = {5, 2, 8, 1};

// Sort a primitive array (ascending)
Arrays.sort(arr);               // [1, 2, 5, 8]

// Sort only a subrange [from, to)
Arrays.sort(arr, 1, 3);       // sorts indices 1 and 2

// Sort an ArrayList
Collections.sort(arrayList);   // for List<Integer>

// Sort descending (requires Integer[], not int[])
Arrays.sort(boxedArr, Collections.reverseOrder());
C++
#include <algorithm>
int arr[] = {5, 2, 8, 1};

sort(arr, arr + n);                   // ascending
sort(arr, arr + n, greater<int>());  // descending
Python
arr = [5, 2, 8, 1] arr.sort() # in-place ascending arr.sort(reverse=True) # in-place descending sorted_arr = sorted(arr) # returns new sorted list # Sort by custom key (e.g. by absolute value) arr.sort(key=lambda x: abs(x))
Time (all languages) · O(N log N) Space · O(depends on algo)
Implementation Notes Java uses Dual-Pivot Quicksort for primitives and Timsort for objects. C++ uses Introsort (hybrid quicksort/heapsort). Python uses Timsort. All are O(N log N) average-case.

Elements Removal — Minimum Cost

Given N elements, you remove one element per step. The cost to remove an element equals the current sum of all elements remaining in the array (including the one you are about to remove — add the cost first, then remove). Find the minimum total cost to remove all elements.

Key Observation Each element contributes to the cost of every removal step that happens before or at the step it is removed. If we label elements a > b > c > d (sorted descending), the total cost is a·1 + b·2 + c·3 + d·4. The largest element is multiplied by the smallest coefficient, so we should remove the largest element first (sort descending, then remove in order).

Java
int minCostRemoval(int[] arr) {
    Arrays.sort(arr);              // sort ascending
    int n = arr.length, ans = 0;
    for (int i = 0; i < n; i++) {
        ans += arr[i] * (i + 1);   // arr[i] × its 1-based position
    }
    return ans;
    // Equivalently: sort descending, multiply arr[i] × (n - i)
}
Time · O(N log N) Space · O(depends on sort)
05

Noble Integers — Distinct Data

An element ele in arr[] is called noble if the count of elements in the array that are strictly smaller than ele equals ele itself. Count how many noble integers exist.

Brute Force — O(N²)

Java
int nobleIntegers(int[] arr, int n) {
    int ans = 0;
    for (int i = 0; i < n; i++) {
        int countSmaller = 0;
        for (int j = 0; j < n; j++) {
            if (arr[j] < arr[i]) countSmaller++;
        }
        if (countSmaller == arr[i]) ans++;
    }
    return ans;
}

Optimal — Sort, then check index — O(N log N)

After sorting in ascending order, the count of elements strictly smaller than arr[i] is exactly the index i (since all elements before it are smaller, and data is distinct). So just check if arr[i] == i.

Java
int nobleIntegersOptimal(int[] arr, int n) {
    Arrays.sort(arr);         // ascending
    int ans = 0;
    for (int i = 0; i < n; i++) {
        if (arr[i] == i) ans++; // index i = count of smaller elements
    }
    return ans;
}
Time · O(N log N) Space · O(1)

The algorithm: after sorting, the countSmaller variable updates to the current index i only when we see a new (different) value. We check if arr[i] == countSmaller.

Java
int nobleIntegersWithRepeats(int[] arr, int n) {
    Arrays.sort(arr);          // sort ascending
    int ans = 0, countSmaller = 0;

    if (arr[0] == 0) ans++;     // special check for first element

    for (int i = 1; i < n; i++) {
        if (arr[i] != arr[i - 1]) {
            countSmaller = i;     // new value — its smaller count = i
        }
        if (arr[i] == countSmaller) ans++;
    }
    return ans;
}
Time · O(N log N) Space · O(depends on sort)

Java
void selectionSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        int minIdx = i;
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[minIdx]) minIdx = j;
        }
        // swap arr[i] and arr[minIdx]
        int temp = arr[i];
        arr[i] = arr[minIdx];
        arr[minIdx] = temp;
    }
}
Time (all cases) · O(N²) Space · O(1) Stable? No
Note Selection Sort always makes exactly N-1 swaps (one per pass), regardless of whether the array is already sorted. This makes it predictable but inefficient on nearly-sorted data compared to Insertion Sort.

Insertion Sort

Idea: Think of sorting a hand of playing cards. You hold a sorted portion on the left. At each step, pick the next card from the unsorted right portion and insert it into the correct position in the sorted left portion by shifting elements right.

Analogy — Playing Cards You are dealt cards one by one. Each time you receive a new card (the "key"), you scan your sorted hand from right to left, shifting cards right until you find where the new card belongs, then slot it in.

Java
void insertionSort(int[] arr) {
    int n = arr.length;
    for (int idx = 1; idx < n; idx++) {
        for (int j = idx; j > 0; j--) {
            if (arr[j - 1] > arr[j]) {
                int temp = arr[j];
                arr[j]     = arr[j - 1];
                arr[j - 1] = temp;
            } else {
                break; // already in correct position — stop early
            }
        }
    }
}
Time (worst) · O(N²) Time (best) · O(N) Space · O(1) Stable? Yes
Best Case — Already Sorted If the array is already sorted, the inner loop's break fires immediately on every outer iteration. Insertion Sort runs in O(N) — exactly N-1 comparisons, zero swaps. Selection Sort still runs O(N²).

10

Quick Recap

Sorting

Arranging data by a parameter; magnitude is the default, but any comparable criterion works.

Built-In APIs

Arrays.sort() in Java, sort() in C++, .sort() in Python — all O(N log N), always prefer these.

Elements Removal

Minimum cost = sort descending, multiply each element by its 1-based removal position index.

Noble Integers (Distinct)

Sort ascending; check arr[i] == i. Index equals count of smaller elements for distinct data.

Noble Integers (Repeated)

Track first-occurrence index of each value; check if value equals that first-occurrence index.

Selection Sort

Finds minimum in unsorted region and swaps it to the front. Always O(N²) — never benefits from nearly-sorted input.

Insertion Sort

Inserts each element into its correct position in the sorted prefix. O(N) best case, O(N²) worst — great for nearly-sorted data.

Stability

A stable sort preserves the relative order of equal elements. Insertion Sort is stable; Selection Sort is not.
