01

Linear vs Binary Search

Searching is the task of locating a target element within a collection. Two mental models motivate the two algorithms: searching for a word in a newspaper (read every word — linear search) versus searching for a word in a dictionary (open to the middle, decide which half to keep — binary search).

📰

Linear Search

Iterate every element one by one until the target is found or the end is reached. Works on any array, sorted or not.

📖

Binary Search

Requires a sorted (or monotone) sequence. Every iteration eliminates half the remaining search space, giving O(log N) time.

Key Insight Every binary search problem needs three things: a target to find, a search space to operate on, and a condition that lets you eliminate half the space each step.

Linear Search Code

Time O(N) Space O(1)
Java
for (int i = 0; i < n; i++) {
    if (arr[i] == k) {
        return i;   // found at index i
    }
}
return -1;         // not found
02

Classic Binary Search

Given a sorted array, find the index of element K, or return −1 if absent.

Analogy Think of a number-guessing game: someone thinks of a number 1–100 and tells you "higher" or "lower". You always guess the middle — that is binary search.

How It Works

Maintain two pointers l (left) and r (right) bounding the current search space [0, N−1]. At each step, compute mid = (l + r) / 2 and compare arr[mid] with the target:

🎯
arr[mid] == K — Found it! Return mid.
⬅️
arr[mid] > K — Target is in the left half. Set r = mid − 1.
➡️
arr[mid] < K — Target is in the right half. Set l = mid + 1.
❌
l > r — Search space exhausted, target not present. Return −1.

Java
int l = 0, r = n - 1;
while (l <= r) {
    int mid = (l + r) / 2;
    if (arr[mid] == k) {
        return mid;
    } else if (arr[mid] > k) {
        r = mid - 1;
    } else {
        l = mid + 1;
    }
}
return -1;
Time O(log₂ N) Space O(1)
Common Mistake Writing mid = (l + r) / 2 can overflow when l + r exceeds Integer.MAX_VALUE in Java/C++. Use mid = l + (r - l) / 2 to be safe.

First & Last Occurrence

Given a sorted array where K may appear multiple times, find the first (or last) index at which K occurs. The classic binary search stops as soon as it finds any match — here we keep looking left (or right) even after a match.

First occurrence = index 6; last occurrence = index 10; frequency = 10 − 6 + 1 = 5.

Key Modification

When arr[mid] == K, record ans = mid but keep searching in the relevant direction instead of immediately returning:

Java — First Occurrence
int l = 0, r = n - 1, ans = -1;
while (l <= r) {
    int mid = (l + r) / 2;
    if (arr[mid] == k) {
        ans = mid;
        r = mid - 1;   // keep going LEFT for first occurrence
    } else if (arr[mid] > k) {
        r = mid - 1;
    } else {
        l = mid + 1;
    }
}
return ans;
Tip For last occurrence, change r = mid − 1 (when matched) to l = mid + 1 — this keeps searching to the right. Frequency = last − first + 1.
Time O(log₂ N) Space O(1)
04

Local Minima

A local minimum is any element that is less than or equal to both of its neighbors. Given an unsorted array, find any one local minimum in O(log N) time.

Analogy Imagine a mountain range profile. A valley (local minimum) always exists between any two peaks. Standing at the midpoint, if the left neighbor is smaller, a valley must exist on the left half — go there.

The Four Cases at arr[mid]

🏔

Case 1: Valley

arr[mid] ≤ arr[mid−1] and arr[mid] ≤ arr[mid+1] → return arr[mid].

⬅️

Case 2: Downslope Left

arr[mid−1] ≤ arr[mid] → a local min exists in the left half. Set r = mid − 1.

➡️

Case 3: Downslope Right

arr[mid+1] < arr[mid] → a local min exists in the right half. Set l = mid + 1.

↔️

Case 4: Peak

arr[mid−1] < arr[mid] and arr[mid+1] < arr[mid] → go to either side (we pick right by convention).

Edge Cases Handle the boundary elements specially: if N = 1, return arr[0]. If arr[0] ≤ arr[1], return arr[0]. If arr[N−1] ≤ arr[N−2], return arr[N−1]. Search the interior with l = 1, r = N − 2.
Java
if (n == 1) return arr[0];
else if (arr[0] <= arr[1]) return arr[0];
else if (arr[n-1] <= arr[n-2]) return arr[n-1];

int l = 1, r = n - 2;
while (l <= r) {
    int mid = (l + r) / 2;
    if (arr[mid] <= arr[mid-1] && arr[mid] <= arr[mid+1]) {
        return arr[mid];      // local minimum found
    } else if (arr[mid-1] <= arr[mid]) {
        r = mid - 1;           // left side descends
    } else {
        l = mid + 1;           // right side descends
    }
}
return -1;
Time O(log₂ N) Space O(1)
05

Floor Square Root

Given a non-negative integer N, return ⌊√N⌋ — the largest integer whose square does not exceed N — without using any built-in square-root function.

Examples

N⌊√N⌋Reason
3666² = 36 ≤ 36, 7² = 49 > 36
497exact square root
5077² = 49 ≤ 50, 8² = 64 > 50
648exact square root

Brute Force O(√N)

Start from 1 and walk upward, recording ans = i whenever i × i ≤ N. Stop when i × i > N.

Binary Search Approach O(log N)

The search space is [1, N]. The target is the largest integer mid such that mid × mid ≤ N. The condition is monotone: everything ≤ ⌊√N⌋ satisfies mid² ≤ N, everything above it does not.

Java
if (n <= 1) return n;   // edge cases: 0 and 1

long l = 1, r = n, ans = 1;
while (l <= r) {
    long mid = (l + r) / 2;
    if (mid * mid <= n) {
        ans = mid;         // valid candidate; try larger
        l = mid + 1;
    } else {
        r = mid - 1;
    }
}
return (int) ans;
Overflow Warning Use long for mid * mid — for large N (up to 2 × 10⁹), the product can exceed Integer.MAX_VALUE.
Time O(log N) Space O(1)

06

Quick Recap

Binary Search Template

Maintain l, r; compute mid = (l+r)/2; update pointers based on condition until l > r.

Three Ingredients

Every BS problem needs: (1) a target, (2) a search space, (3) a monotone condition to halve the space.

First / Last Occurrence

Record ans = mid on a match but keep moving left (first) or right (last) instead of stopping.

Local Minima

Not a sorted array problem — yet BS applies because the valley guarantee holds: if arr[mid−1] < arr[mid], a valley exists on the left.

Floor √N

Binary search on integers 1…N; keep the last mid with mid² ≤ N. Use long to avoid overflow.

Complexity

All binary search variants run in O(log N) time and O(1) space — dramatically better than O(N) linear scan.
