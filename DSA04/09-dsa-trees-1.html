Introduction to Trees

A tree is a hierarchical, non-linear data structure where nodes are connected by edges, with no cycles. Unlike arrays or linked lists, a tree branches — each node can have multiple children.

Programmer vs Normal Person For normal people, trees have their root at the bottom. For programmers, the root is at the top and the branches grow downward — exactly how we draw them in computer science diagrams.

Real-World Examples

🏢

Company Org Chart

CEO at the root; CTO, CMO, CFO are children; Engineering Managers are grandchildren, and so on.

💻

File System

"This PC" → Documents, Movies, Books → sub-folders and files. A perfect tree structure.

🌐

HTML DOM

The browser parses HTML into a tree where <html> is the root, with <head> and <body> as children.

03

Binary Tree

A tree where every node has at most 2 children — a left child and a right child.

java — Node class
class Node {
    int val;
    Node left;
    Node right;

    Node(int v) { val = v; }
}
// Each node stores its value and pointers to left and right children.
// null means "no child".
Memory Layout Each Node object uses approximately 12 bytes on the heap: 4 bytes for int val, 4 bytes for the left reference, 4 bytes for the right reference (on 32-bit JVM; 8 bytes per reference on 64-bit without compressed OOPs).

Types of Binary Trees

TypePropertyExample
Full / ProperEvery node has exactly 0 or 2 children (no node has exactly 1 child)Perfect trees, expression trees
CompleteAll levels fully filled except possibly the last, which is filled left to rightHeap
PerfectAll levels fully filled including the last — all leaves at the same depthA perfect binary tree with N = 2ʰ⁺¹ − 1 nodes
BalancedHeight difference of left and right subtrees ≤ 1 for every nodeAVL tree, Red-Black tree

05

Pre-order Traversal — Root → Left → Right

Order Visit the current node first, then recursively traverse the left subtree, then the right subtree.

For the tree with root 1, left child 2 (children 4, 5 where 5 has child 7), right child 3 (children 6 with children 8 and 9):

Pre-order: [1, 2, 4, 5, 7, 3, 6, 8, 9]

java
void preorder(Node root) {
    if (root == null) return;
    System.out.print(root.val + " ");  // ① visit root
    preorder(root.left);               // ② left subtree
    preorder(root.right);              // ③ right subtree
}
// T.C → O(N)   S.C → O(Height of tree)  [recursive call stack]
Time: O(N)
Space: O(H) — H = height
06

In-order Traversal — Left → Root → Right

Order Recursively traverse the left subtree, then visit the current node, then recursively traverse the right subtree.
Special Property In-order traversal of a Binary Search Tree (BST) always yields elements in sorted ascending order. This is the key insight behind many BST problems.

In-order on the same tree: [4, 2, 7, 5, 1, 3, 8, 6, 9]

java
void inorder(Node root) {
    if (root == null) return;
    inorder(root.left);               // ① left subtree
    System.out.print(root.val + " ");  // ② visit root
    inorder(root.right);              // ③ right subtree
}
// T.C → O(N)   S.C → O(H)
07

Post-order Traversal — Left → Right → Root

Order Recursively traverse the left subtree, then the right subtree, then visit the current node last.
Use Case Post-order is natural when you need to process a node after its children — e.g., deleting a tree (delete children before parent), computing subtree sizes, or evaluating expression trees.
java
void postorder(Node root) {
    if (root == null) return;
    postorder(root.left);             // ① left subtree
    postorder(root.right);            // ② right subtree
    System.out.print(root.val + " ");  // ③ visit root
}
// T.C → O(N)   S.C → O(H)
TraversalOrderTypical Use
Pre-orderRoot → L → RCopy / serialize a tree, prefix expression
In-orderL → Root → RBST sorted output, check BST
Post-orderL → R → RootDelete tree, compute subtree aggregates, postfix expression
08

Iterative In-order Traversal

The recursive stack is implicit. We can make it explicit using a manual Stack and a curr pointer.

Algorithm

Maintain a curr pointer starting at root. While curr != null OR stack is non-empty: if curr != null push it and go left. Otherwise pop, print, and go right.

java
void iterativeInorder(Node root) {
    Stack<Node> st = new Stack<>();
    Node curr = root;

    while (curr != null || st.size() > 0) {
        if (curr != null) {
            st.push(curr);
            curr = curr.left;         // go left first
        } else {
            curr = st.pop();
            System.out.print(curr.val + " ");  // visit
            curr = curr.right;        // then go right
        }
    }
}
// T.C → O(N)   S.C → O(H) — explicit stack replaces call stack
Why Iterative? Recursive traversals can cause a stack overflow on extremely deep trees (skewed trees, height ~ N). Iterative traversals are stack-overflow-safe and use the same asymptotic space.
09

Level Order Traversal (BFS)

Idea Visit all nodes level by level from top to bottom, left to right. Use a Queue. For each level, capture its size first (sz = q.size()), process exactly sz nodes, then print a newline.

For the tree: 3 → children 7, 9 → 7's children 5, 6 → 9's child 2 → 2's children 60, 30 → 5's child 40 → 6's child 15, 20:

Level order: 3 / 7 9 / 5 6 2 / 40 15 20 60 30

java
void levelOrder(Node root) {
    Queue<Node> q = new LinkedList<>();
    q.add(root);

    while (!q.isEmpty()) {
        int sz = q.size();               // nodes in current level

        for (int i = 0; i < sz; i++) {
            Node rn = q.poll();
            System.out.print(rn.val + " ");

            if (rn.left  != null) q.add(rn.left);
            if (rn.right != null) q.add(rn.right);
        }
        System.out.println();           // newline after each level
    }
}
// T.C → O(N)   S.C → O(N)  [queue holds at most one full level ~ N/2 nodes]
Time: O(N)
Space: O(N)
10

Left View of a Binary Tree

Problem Statement Print the first (leftmost) node visible at each level when the tree is viewed from the left side.

This is a BFS modification: in the inner loop, print the value only when i == 0 (first node of the level).

java
void leftView(Node root) {
    if (root == null) return;
    Queue<Node> q = new LinkedList<>();
    q.add(root);

    while (!q.isEmpty()) {
        int sz = q.size();

        for (int i = 0; i < sz; i++) {
            Node rn = q.poll();

            if (i == 0) System.out.print(rn.val + " ");  // first node of level

            if (rn.left  != null) q.add(rn.left);
            if (rn.right != null) q.add(rn.right);
        }
    }
}
// Right view: print when i == sz - 1 (last node of each level)
// T.C → O(N)   S.C → O(N)
Right View Change the condition to i == sz - 1 to get the right view. Both are O(N) BFS modifications.
11

Quick Recap

Pre-order RLR

Root → Left → Right. Used for tree serialization and prefix expressions.

In-order LRR

Left → Root → Right. Gives sorted output for BSTs. Key for BST problems.

Post-order LRR

Left → Right → Root. Process children before parent. Used for deletion and aggregation.

Iterative In-order

Explicit stack + curr pointer. Same O(H) space but avoids recursive stack overflow.

Level Order BFS

Queue. Capture level size first. Process sz nodes per level. O(N) time and space.

Left / Right View

BFS variant: print when i==0 (left view) or i==sz-1 (right view) per level.
