Linked List — Basic Problems

Four fundamental linked list operations every engineer must master: search, insertion at any index, deletion of a value, iterative reversal, and the elegant O(N)/O(1) deep copy with random pointers.

Search O(N) Insert O(K) Delete O(N) Reverse O(N) Deep Copy O(N)
01

Search for Element X

Unlike arrays, a linked list has no random access. We cannot jump to a mid-point. The only way to reach a node at index k is to follow next pointers one by one from the head. Consequently, every search is inherently linear.

Meme Moment When someone asks a singly linked list node for the address of its previous node — "Apneko kya maalum bhai?" (How would I know?). A singly linked list node only knows where to go forward, never backward.

Algorithm

📌Let temp = head — use a separate pointer; never move head itself
🔁While temp != null → check temp.val == x
✅If match found → return true
➡️Else → advance: temp = temp.next
❌Loop ended without match → return false

Java
boolean search(Node head, int x) {
    Node temp = head;
    while (temp != null) {
        if (temp.val == x) return true;
        temp = temp.next;
    }
    return false;
}
Time — O(N) Space — O(1)
02

Insertion at K-th Index

Insert a new node with a given value at position K (0-indexed). The existing node at K and all subsequent nodes shift one position to the right.

The Core Linking Trick

After walking to the node at index K−1 (call it temp), link the new node nn in two careful steps — order matters:

①nn.next = temp.next — save the chain that follows
②temp.next = nn — splice new node in
Common Bug If you do step ② first, you lose the reference to the rest of the list before step ①. Always wire up the new node's next pointer before changing any existing next.

Edge Cases

K = 0

Insert at Head

Set nn.next = head, then head = nn. Return the new head. No traversal needed.

K = N

Insert at Tail

Walk to the last node (where next == null), then temp.next = nn. The new node's next is already null from construction.

Java — K ≤ N
Node insertAt(Node head, int k, int val) {
    Node nn = new Node(val);

    if (k == 0) {            // insert at head
        nn.next = head;
        head = nn;
        return head;
    }

    Node temp = head;
    for (int i = 0; i < k - 1; i++) {   // walk K-1 times
        temp = temp.next;
    }
    nn.next = temp.next;   // step ①
    temp.next = nn;        // step ②
    return head;
}
Time — O(K) Space — O(1)

Delete First Occurrence of X

Remove the first node whose value equals X. If the element is not present, leave the list unchanged and return the head as-is.

Core Idea — "Skip" the Target Node

To delete the node at position k, walk to the node at position k−1 and set its next to skip over the unwanted node:

temp.next = temp.next.next;

Edge Case — Delete the Head Node

When head.val == x, simply advance the head: head = head.next. There is no previous node to re-link.

Example 2 — delete(5) from [5 → 7 → 3 → 11]

The target is at the head. Result: [7 → 3 → 11].

Java
Node delete(Node head, int x) {
    if (head == null) return head;

    if (head.val == x) {           // delete the head itself
        head = head.next;
        return head;
    }

    Node temp = head;
    while (temp.next != null) {
        if (temp.next.val == x) {
            temp.next = temp.next.next;   // skip the target node
            break;                        // only first occurrence
        }
        temp = temp.next;
    }
    return head;
}
Time — O(N) Space — O(1)
Variant To delete all occurrences (e.g. the OnePlus manufacturing-line problem), remove the break statement and keep looping until temp.next is null. Also handle consecutive head deletions with a while loop before the main traversal.

Three-Pointer Technique

Use three pointers: prev (starts at null), curr (starts at head), and a temporary currKanext to save the next node before overwriting curr.next.

💾Save: currKanext = curr.next — preserve the forward chain
🔄Reverse: curr.next = prev — flip the pointer
➡️Advance: prev = curr; curr = currKanext
🔁Repeat while curr != null
🏁head = prev — prev is now pointing at the original last node (new head)
Java — Iterative Reverse
Node reverse(Node head) {
    Node prev = null;
    Node curr = head;
    while (curr != null) {
        Node currKanext = curr.next;  // save next
        curr.next = prev;              // flip pointer
        prev = curr;                   // advance prev
        curr = currKanext;             // advance curr
    }
    head = prev;
    return head;
}
Time — O(N) Space — O(1)
05

Copy List with Random Pointers

Each node has a val, a next pointer, and a random pointer that can point to any node in the list (or null). Create a completely independent deep copy of this list in O(N) time and O(1) extra space.

Java — Node Structure
class Node {
    int val;
    Node next;
    Node random;
}

Challenge

When we clone node A and its random pointer points to node C, how do we wire clone-A's random to clone-C without a hash-map? The trick is to interleave clones right after their originals in the same list temporarily.

Three-Phase Algorithm

①

Interleave Clone Nodes

For each original node temp, create a clone cn and insert it right after temp: cn.next = temp.next; temp.next = cn; Then advance by temp = temp.next.next.

②

Set Random Pointers

Now every clone sits right next to its original. For each original temp, the clone is temp.next. Wire the clone's random: temp.next.random = temp.random.next;

③

Separate the Two Lists

Disentangle the interleaved list back into two independent lists by restoring the original next pointers and building the clone list's next chain. Return the clone head.

Key Insight After interleaving, temp.random.next is exactly the clone of temp.random, because every original node X now has its clone at X.next. This lets us set random pointers without any hash map.
Java — Phase ① & ② shown
Node copyRandomList(Node head) {
    if (head == null) return null;

    // Phase ①: interleave clone nodes
    Node temp = head;
    while (temp != null) {
        Node cn = new Node(temp.val);
        cn.next = temp.next;
        temp.next = cn;
        temp = temp.next.next;
    }

    // Phase ②: set random pointers of clones
    temp = head;
    while (temp != null) {
        if (temp.random != null)
            temp.next.random = temp.random.next;
        temp = temp.next.next;
    }

    // Phase ③: separate both linked lists
    temp = head;
    Node cloneHead = head.next;
    Node clone = cloneHead;
    while (temp != null) {
        temp.next = temp.next.next;
        if (clone.next != null)
            clone.next = clone.next.next;
        temp = temp.next;
        clone = clone.next;
    }
    return cloneHead;
}
Time — O(N) Space — O(1) Passes — 3 × O(N)
Simpler Alternative A HashMap<Node, Node> solution maps original → clone in one pass, then sets all pointers in a second pass. That achieves O(N) time and O(N) extra space — acceptable when the O(1) space version seems tricky in an interview.

Quick Recap

Search

Linear traversal with a temp pointer; O(N) — no random access in a linked list.

Insert at K

Walk K−1 steps; link new node's next first, then redirect predecessor's next. K=0 is a special head case.

Delete First Occurrence

Handle head deletion separately; for others, track the predecessor and skip the target node with temp.next = temp.next.next.

Reverse

Three pointers: prev=null, curr=head. Save next, flip pointer, advance both. When curr is null, prev is the new head.

Deep Copy (Random Pointers)

Interleave clones, set random pointers via neighbor trick, then separate lists — all O(N) time, O(1) space.

Golden Rule

Never move the head pointer during traversal. Always use a separate temp pointer.
