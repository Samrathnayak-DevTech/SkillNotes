Binary Search
Tree (BST)

A special binary tree where every node satisfies a strict ordering property, enabling O(Ht) search, insert, and delete — and O(log N) for balanced BSTs.

01

BST Definition

BST Property A binary tree where at every node:

All values in left subtree ≤ node.val < all values in right subtree

This must hold for every single node, not just immediate children — common trap!

The key distinction from a regular binary tree is that BST enforces a global ordering invariant. Just checking left child < node < right child is not sufficient — you must verify that all descendants on the left are smaller and all on the right are larger.

Valid vs Invalid BSTs

Analogy Think of a BST like a phone book hierarchy. At each level, names before the divider go left, names after go right — and this holds recursively at every sub-section.
TreeValid BST?Why?
Root=10, Left=5(children:3,8), Right=13(children:11,16→18)❌ NoNode 8 is in left subtree but has child 11 — 11 > 10 violates BST property
Root=7, Left=3(children:4,6), Right=11(children:8,18)❌ NoNode 4 is right child of 3 but left subtree of 7 — 4 < 7 ✓, but 4 > 3 ✓... actually node 3's right child 4 violates since 4 is not < 7's left
Root=15, Right→11, Right→9, Right→7, Right→6✅ YesRight spine with decreasing values — valid (degenerate BST)
Root=10, Left=5(children:2,8), Right=19(children:25,9)❌ NoNode 9 in right subtree but 9 < 10 violates BST property
02

Search in BST

Leverage the BST property: at each node, we know whether target K is in the left subtree or right subtree — no need to explore both. At every level, only one node is accessed.

🔍
If root == null → return false (not found)
✅
If root.val == K → return true
⬅️
If K < root.val → recurse on root.left
➡️
If K > root.val → recurse on root.right

Recursive Implementation

java 
boolean search(Node root, int K) {
    if (root == null) return false;
    if (root.val == K) return true;
    else if (K < root.val)
        return search(root.left, K);
    else
        return search(root.right, K);
}
Time: O(Ht)
Space: O(Ht)

Iterative Implementation (O(1) Space)

java 
boolean searchIterative(Node root, int K) {
    Node curr = root;
    while (curr != null) {
        if (curr.val == K) return true;
        else if (K < curr.val)
            curr = curr.left;
        else
            curr = curr.right;
    }
    return false;
}
Time: O(Ht)
Space: O(1)
Tip For a balanced BST, Ht = O(log N), making search O(log N). For a degenerate BST (like a sorted linked list), Ht = O(N). Always prefer the iterative version in practice — it saves stack space.
03

Insertion in BST

Insertion follows the same search path — we navigate until we find a null spot that respects the BST property, then place the new node there. New nodes are always inserted as leaf nodes.

Key Insight If the value already exists (root.val == K), we return without inserting — BSTs typically store unique values.
java 
Node insert(Node root, int K) {
    // Base case: found the insertion spot
    if (root == null) {
        Node nn = new Node(K);
        return nn;
    }
    // Duplicate: don't insert
    if (root.val == K) return root;

    else if (K < root.val)
        root.left = insert(root.left, K);
    else
        root.right = insert(root.right, K);

    return root;
}
Time: O(Ht)
Space: O(Ht)
Trace — insert(7) into BST rooted at 5 5 → right(8) → left(6) → right = null → insert 7 here
Result: 7 becomes right child of 6.
04

Find Smallest & Largest

By the BST property, the minimum is always the leftmost node and the maximum is always the rightmost node.

GoalDirectionStop When
Find Smallest (Min)Keep going leftcurr.left == null
Find Largest (Max)Keep going rightcurr.right == null
java 
// Find minimum
int findMin(Node root) {
    Node curr = root;
    while (curr.left != null)
        curr = curr.left;
    return curr.val;
}

// Find maximum
int findMax(Node root) {
    Node curr = root;
    while (curr.right != null)
        curr = curr.right;
    return curr.val;
}
Time: O(Ht)
Space: O(1)
Tip These are used internally during deletion — the "max of left subtree" is found by going right-most in the left child, and the "min of right subtree" by going left-most in the right child.
05

Deletion in BST

Deletion has three cases depending on how many children the target node has. The invariant to maintain: after deletion, the BST property must still hold for all nodes.

Core Insight When a node has two children and is removed, we can replace its value with either the max of left subtree or the min of right subtree — both maintain the BST ordering. The lectures use max of left subtree.

Case 1: Leaf Node

No children. Simply remove it — return null to parent.

Case 2: One Child

Node has only left or only right child. Return that child to replace the deleted node.

Case 3: Two Children

Find max of left subtree (rightmost in left child), swap values with current node, then delete the swapped node from left subtree.

java 
Node delete(Node root, int K) {
    if (root == null) return null;

    if (root.val == K) {
        // Case 1: Leaf node
        if (root.left == null && root.right == null)
            return null;

        // Case 2a: Only right child
        else if (root.left == null && root.right != null)
            return root.right;

        // Case 2b: Only left child
        else if (root.left != null && root.right == null)
            return root.left;

        else {
            // Case 3: Two children
            // Find max of left subtree (rightmost node in left child)
            Node temp = root.left;
            while (temp.right != null) temp = temp.right;

            // Swap values
            swap(temp, root);

            // Delete the swapped node from left subtree
            root.left = delete(root.left, K);
            return root;
        }
    } else if (K < root.val) {
        root.left = delete(root.left, K);
    } else {
        root.right = delete(root.right, K);
    }
    return root;
}
Time: O(Ht)
Space: O(Ht)
Example — delete(8) from tree BST: 8 → left(4: children 1,7), right(12: children 10,15)
Node 8 has two children → find max of left subtree = 7
Swap: root becomes 7, delete old 7 from left subtree (Case 1 — leaf)
Result: 7 is new root.
06

Construct Balanced BST from Sorted Array

Given a sorted array, build the most height-balanced BST possible. The idea: always pick the middle element as root, then recursively build left and right subtrees from the two halves.

Key Insight Choosing the middle element ensures roughly equal elements on both sides → minimum height tree → O(log N) height.

Example

arr = [-1, 3, 4, 6, 7, 8, 10, 13, 14] — indices 0 to 8

mid = (0+8)/2 = 4 → root = arr[4] = 7
Left subtree from arr[0..3] = [-1,3,4,6], mid=1 → root=3
Right subtree from arr[5..8] = [8,10,13,14], mid=6 → root=10

java 
Node constructBST(int[] arr, int l, int r) {
    if (l > r) return null;
    if (l == r) return new Node(arr[l]);

    int mid = (l + r) / 2;
    Node root = new Node(arr[mid]);

    root.left  = constructBST(arr, l, mid - 1);
    root.right = constructBST(arr, mid + 1, r);

    return root;
}

// Call: constructBST(arr, 0, arr.length - 1)
Time: O(N)
Space: O(log N)
Space Note Space is O(log N) — the height of the balanced BST — because that's the maximum recursion depth. This equals O(log₂ N) for a balanced tree.
07

Check if a Binary Tree is a BST

Two clean approaches:

Approach 1 — In-order Traversal

A BST's in-order traversal always produces a strictly sorted array. So do in-order and check if result is sorted.

Key In-order (Left → Root → Right) of a valid BST gives elements in ascending order. If any element is ≤ the previous, it's not a BST.
Time: O(N)
Space: O(N)

Approach 2 — Range Bounds (Optimal)

Pass a valid range (min, max) to each node. At every node, check if min < node.val < max, then recurse with updated bounds.

java 
boolean isBST(Node root, long min, long max) {
    if (root == null) return true;

    if (root.val <= min || root.val >= max) return false;

    return isBST(root.left,  min,      root.val) &&
           isBST(root.right, root.val, max);
}

// Call: isBST(root, Long.MIN_VALUE, Long.MAX_VALUE)
Dry Run — tree: 5→left(2), 5→right(16→left(4)) isBST(5, -∞, +∞) → OK
isBST(2, -∞, 5) → OK
isBST(16, 5, +∞) → OK
isBST(4, 5, 16) → 4 ≤ 5 → FAIL → not a BST
Time: O(N)
Space: O(Ht)
Common Mistake Use Long.MIN_VALUE and Long.MAX_VALUE (not Integer bounds) to handle edge cases where node values are at the extremes of int range.
