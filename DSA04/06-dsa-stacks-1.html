What is a Stack?

A stack is a linear data structure where Last In, First Out (LIFO) discipline is maintained — the most-recently-added element is always the first one removed.

🍽️

Stack of Plates

You always pick up the topmost plate, and you place new plates on top. You never interact with the bottom.

🪑

Stack of Chairs

Chairs are stacked and unstacked from the same end — the top.

⚙️

Call Stack

When fib(5) calls fib(4) calls fib(3) … each frame is pushed; when the deepest call returns, frames pop off in reverse order.

🌐

Browser History

Each visited page is pushed. Clicking "Back" pops the current page to reveal the previous one.

↩️

Undo / Redo

Every edit is pushed onto an undo stack. "Undo" pops from undo and pushes to redo — two co-operating stacks.

Key Property LIFO — Last In, First Out. Only the top element is accessible at any given time. All five core operations run in O(1) time.

03

Array Implementation

Maintain a fixed-size array arr[] and an integer tos (top-of-stack index, initially −1).

Idea tos = −1 → stack is empty.
push(x): increment tos, set arr[tos] = x.
pop(): save arr[tos], decrement tos, return saved value.
peek(): return arr[tos] without changing tos.
java
class Stack {
    int[] arr = new int[1000];
    int tos = -1;

    void push(int x) {
        if (tos == arr.length - 1) { System.out.println("Stack Overflow"); return; }
        arr[++tos] = x;
    }

    int top() {
        if (tos == -1) { System.out.println("Stack Underflow"); return -1; }
        return arr[tos];
    }

    int pop() {
        if (tos == -1) { System.out.println("Stack Underflow"); return -1; }
        int rv = arr[tos--];
        return rv;
    }

    boolean isEmpty() { return tos == -1; }

    int size()     { return tos + 1; }
}
// T.C → O(1) for all operations   S.C → O(N) for the array
All operations: O(1)
Space: O(N)

Linked List Implementation

Four possible combinations of add/remove at head or tail. Only two work efficiently (O(1) for both push & pop):

StrategyPushPopWorks?
AAdd at headRemove from head✅ O(1) both
BAdd at tailRemove from tail❌ O(N) for remove-tail in singly-LL
CAdd at headRemove from tail❌ O(N) for remove-tail
DAdd at tailRemove from head❌ mismatches LIFO (this is a queue)
Correct Strategy — A Use a singly-linked list with a head pointer. Push = addFirst(). Pop = removeFirst(). Both are O(1).
java
class Node {
    int val;
    Node next;
    Node(int v) { val = v; }
}

class LLStack {
    Node head = null;
    int sz = 0;

    void push(int x) {
        Node nn = new Node(x);
        nn.next = head;
        head = nn;
        sz++;
    }

    int pop() {
        if (head == null) return -1;
        int rv = head.val;
        head = head.next;
        sz--;
        return rv;
    }

    int  peek()    { return head == null ? -1 : head.val; }
    boolean isEmpty() { return sz == 0; }
    int  size()    { return sz; }
}
// T.C → O(1) all ops    S.C → O(N)
05

Valid Parentheses

Problem Statement Given a string containing only (, ), {, }, [, ], determine whether the sequence is valid (every opening bracket is closed by the same type of bracket in the correct order).

Examples

StringValid?Reason
()[{}()]✅ YesAll matched in order
()[]✅ YesSequential matching
({)}❌ NoMismatch — top is ( but closing ) expected
({})]❌ NoExtra closing bracket after valid sequence
[(){}❌ NoUnclosed [ at end — stack not empty

Algorithm

🔄
For each character: if it is an opening bracket, push it.
🔍
Closing bracket encountered: if stack is empty → return false (more closing than opening). Otherwise check: does stack.peek() match? If yes, pop. If no → return false (mismatch).
✅
After full scan: return stack.isEmpty(). A non-empty stack means unclosed brackets remain.
java
boolean isValid(String str) {
    Stack<Character> st = new Stack<>();

    for (int i = 0; i < str.length(); i++) {
        char ch = str.charAt(i);

        if (ch == '(' || ch == '{' || ch == '[') {
            st.push(ch);
        } else {
            if (st.isEmpty()) return false; // more closing brackets

            if      (ch == ')' && st.peek() != '(') return false;
            else if (ch == '}' && st.peek() != '{') return false;
            else if (ch == ']' && st.peek() != '[') return false;
            else st.pop();
        }
    }
    return st.isEmpty(); // false if unclosed brackets remain
}
// T.C → O(N)    S.C → O(N)
Time: O(N)
Space: O(N)

Postfix Expression Evaluation

Problem Statement Given a valid postfix (Reverse Polish Notation) expression as an array of strings, evaluate it and return the integer result.

Infix vs Postfix

InfixPostfix (RPN)
2 + 32 3 +
2 + (5 * 3)2 5 3 * +
5 * 2 − 35 2 * 3 − = 7

Algorithm

For each token in the expression: if it is a number, push it. If it is an operator, pop the top two operands (op2 first, then op1), compute op1 ⊕ op2, and push the result. The single remaining stack element is the answer.

Order Matters Pop op2 first, then op1. For non-commutative operators (−, /): result = op1 operator op2.
java
int evalRPN(String[] tokens) {
    Stack<Integer> st = new Stack<>();

    for (String token : tokens) {
        if (isOperand(token)) {
            st.push(Integer.parseInt(token));
        } else {
            int op2 = st.pop();
            int op1 = st.pop();
            st.push(compute(op1, op2, token));
        }
    }
    return st.pop();
}

int compute(int op1, int op2, String op) {
    if      (op.equals("+")) return op1 + op2;
    else if (op.equals("-")) return op1 - op2;
    else if (op.equals("*")) return op1 * op2;
    else                      return op1 / op2;
}
// T.C → O(N)    S.C → O(N)
Time: O(N)
Space: O(N)

Nearest Smaller Element on Left (NSL)

Problem Statement Given an integer array A[], for every index i find the index j such that A[j] < A[i], j < i, and j is maximum (closest to i on the left). Return −1 if no such element exists.

Examples

arr[]nsl[]
[8, 2, 4, 9, 7, 5, 3, 10][−1, −1, 1, 2, 2, 2, 1, 6]
[4, 6, 10, 11, 7, 8, 3, 5][−1, 0, 1, 2, 1, 4, −1, 6]
[4, 5, 2, 10, 8, 2][−1, 0, −1, 2, 2, −1]

Brute Force — O(N²)

For each i, iterate leftward until finding an element smaller than arr[i].

Optimised — O(N) using Stack

Maintain a monotone stack of indices. For each element, pop all indices whose values are ≥ arr[i] (they can never be the "nearest smaller" for future elements either). The peek of the remaining stack is the answer for index i. Then push i.

java
int[] nearestSmallerLeft(int[] arr) {
    int n = arr.length;
    int[] nsl = new int[n];
    Stack<Integer> st = new Stack<>();

    for (int i = 0; i < n; i++) {
        // pop all elements >= arr[i]
        while (st.size() > 0 && arr[st.peek()] >= arr[i]) {
            st.pop();
        }
        // update answer
        nsl[i] = st.isEmpty() ? -1 : st.peek();
        // push current index
        st.push(i);
    }
    return nsl;
}
// T.C → O(N)  (each element pushed and popped at most once)
// S.C → O(N)
Time: O(N)
Space: O(N)
Why O(N)? Even though there is a while loop inside the for, each index is pushed onto the stack at most once and popped at most once. Total push + pop operations = 2N → O(N) overall.
08

Quick Recap

LIFO

Last In First Out — the defining property of a stack. Only the top is accessible.

All O(1) ops

push, pop, peek, isEmpty, size are all constant-time.

Array vs LL

Array: fixed capacity, O(1). LL (push/pop at head): dynamic, O(1).

Valid Parentheses

Push opening brackets; on closing, check & pop. O(N) time & space.

Postfix Eval

Operands → push. Operators → pop two, compute, push result.

NSL — Monotone Stack

Pop all ≥ current, peek for answer, push current. O(N).
