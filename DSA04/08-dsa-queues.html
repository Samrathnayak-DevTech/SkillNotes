What is a Queue?

A queue is a linear data structure where First In, First Out (FIFO) discipline is maintained — elements are inserted at the back and removed from the front.

🎫

Ticket Counter

People join the back of the line and are served from the front. The first person to arrive is the first to be served.

🖨️

Print Queue

Documents are printed in the order they were sent — the first document sent is the first to print.

🌐

BFS Traversal

Breadth-first search uses a queue to explore nodes level by level.

📦

Task Scheduling

OS process queues and CPU scheduling use FIFO (or priority variants).

Key Property — FIFO First In, First Out. Elements are added at the rear and removed from the front. All four core operations run in O(1) time.

03

Array Implementation (Linear)

Two pointers: front and rear, both start at −1. Enqueue increments rear; dequeue increments front.

Limitation Once rear reaches the end of the array, no more enqueues are possible even if the front portion is free — wasted space. Fixed by the circular array variant.
java
class LinearQueue {
    int[] arr = new int[1000];
    int front = -1, rear = -1;

    void enqueue(int n) {
        if (rear == arr.length - 1) { System.out.println("Queue Full"); return; }
        rear++;
        arr[rear] = n;
    }

    int dequeue() {
        if (front == rear) { System.out.println("Queue Empty"); return -1; }
        front++;
        return arr[front];
    }

    int peek() {
        if (front == rear) return -1;
        return arr[front + 1];
    }
}
// All ops: O(1)   Space: O(N)
04

Circular Array Implementation

Use modular arithmetic (% N) so rear wraps around after reaching the end. Track element count with a separate size variable to distinguish empty (size == 0) from full (size == cap).

Ambiguity Problem Without a size variable, front == rear is ambiguous — it could mean empty or full. Using size resolves this cleanly.
java
class CircularQueue {
    int[] arr;
    int front = 0, rear = 0, size = 0, cap;

    CircularQueue(int cap) { this.cap = cap; arr = new int[cap]; }

    void enqueue(int n) {
        if (size == cap) { System.out.println("Queue Full"); return; }
        rear = (rear + 1) % cap;
        arr[rear] = n;
        size++;
    }

    int dequeue() {
        if (size == 0) { System.out.println("Queue Empty"); return -1; }
        front = (front + 1) % cap;
        size--;
        return arr[front];
    }

    int peek()   { if (size == 0) return -1; return arr[(front + 1) % cap]; }
    int size()   { return size; }
}
// All ops: O(1)   Space: O(N)
All operations: O(1)
Space: O(N)

Linked List Implementation

Of the four add/remove combinations, only one achieves O(1) for both enqueue and dequeue in a singly-linked list:

Correct Strategy Enqueue: add at tail (keep a tail pointer → O(1)).
Dequeue: remove at head → O(1).

06

Queue via Stacks

Problem Statement Implement a queue using only stack operations (push, pop, peek, isEmpty, size). The amortised cost of each queue operation should be O(1).

Use two stacks: st1 (inbox) and st2 (outbox). Enqueue always pushes to st1. Dequeue/peek pops from st2 — but if st2 is empty, first drain all of st1 into st2 (reversing the order to achieve FIFO).

Amortised O(1) Each element is moved from st1 → st2 at most once. Total moves = 2N → O(1) amortised per operation.
java
class MyQueue {
    Stack<Integer> st1 = new Stack<>();  // inbox
    Stack<Integer> st2 = new Stack<>();  // outbox

    void enqueue(int x) { st1.push(x); }

    int dequeue() {
        if (st2.size() > 0) return st2.pop();
        while (st1.size() > 0) st2.push(st1.pop());
        return st2.pop();
    }

    int peek() {
        if (st2.size() > 0) return st2.peek();
        while (st1.size() > 0) st2.push(st1.pop());
        return st2.peek();
    }

    int size() { return st1.size() + st2.size(); }
}
// Enqueue O(1)  |  Dequeue O(1) amortised  |  Peek O(1) amortised

O(N) Algorithm — Monotone Deque

Maintain a deque of indices in decreasing order of their values. For each new element, remove from the back all indices whose values are smaller (they can never be the max of any future window). Remove from the front any index that has fallen out of the window. The front always holds the index of the current window's maximum.

3-Step Loop Body 1. Remove from back — all indices whose values < arr[i] (they are dominated).
2. Add current index to back.
3. Remove from front — if front index ≤ i − K (out of window).
Then: if i ≥ K − 1, record arr[dq.getFirst()] as the answer for this window.
java
int[] maxSlidingWindow(int[] arr, int k) {
    int n = arr.length;
    int[] result = new int[n - k + 1];
    Deque<Integer> dq = new ArrayDeque<>();  // stores indices

    // Build first window (indices 0 to k-1)
    for (int i = 0; i < k; i++) {
        while (!dq.isEmpty() && arr[dq.getLast()] < arr[i])
            dq.removeLast();
        dq.addLast(i);
    }
    result[0] = arr[dq.getFirst()];

    // Slide window
    for (int i = k; i < n; i++) {
        while (!dq.isEmpty() && arr[dq.getLast()] < arr[i])
            dq.removeLast();
        dq.addLast(i);
        if (dq.getFirst() <= i - k)   // out of window
            dq.removeFirst();
        result[i - k + 1] = arr[dq.getFirst()];
    }
    return result;
}
// T.C → O(N)   each element added and removed from deque at most once
// S.C → O(N)
Time: O(N)
Space: O(N)

Quick Recap

FIFO

Front removes, rear inserts. Opposite discipline to a stack.

Circular Array

Use % N wrap-around and a size counter. Removes the wasted-space issue of linear arrays.

LL Queue

Add at tail (with tail pointer), remove at head. Both O(1).

Queue via Stacks

Inbox + outbox. Drain inbox into outbox lazily. Amortised O(1).

Deque

Insert/remove at both ends in O(1). Implemented with DLL. Used in sliding-window max.

Sliding Window Max

Monotone deque of indices. Pop dominated from back; pop expired from front. O(N).
