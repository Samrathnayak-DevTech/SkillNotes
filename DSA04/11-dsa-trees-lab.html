01

Equal Tree Partition

Problem Statement Given the root of a binary tree, return true if you can remove exactly one edge to split the tree into two non-empty subtrees with equal sums. Return false otherwise.

Key Observations

Two observations unlock the solution:

1️⃣
If totalSum is odd → impossible to split evenly → return false
2️⃣
If totalSum is even → check if any subtree has sum = totalSum / 2. When such a subtree exists, removing the edge above it splits the tree equally.
Examples Tree: 5→(3→(4,2), 7→(6,1)) — totalSum=28, target=14 → subtree(7→6,1)=14 → true

Tree: 5→(8, 9→(2,3)) — totalSum=27 (odd) → false

Tree: 6→(10→20→7, 10→3) — totalSum=56, target=28 → subtree(10→20→7)=37 ≠ 28, subtree(10→3)=13 ≠ 28 → false
Edge Case If totalSum is 0 and both halves must be 0 — this still works correctly since we're looking for a subtree with sum == 0. But note: we cannot split the root itself (the two parts must both be non-empty).

Implementation

java 
boolean ans = false;

boolean isEqualTreePartition(Node root) {
    if (root == null) return false;

    int totalSum = sum(root);

    if (totalSum % 2 != 0) return false;  // odd sum → impossible

    sum2(root, totalSum / 2);              // DFS looking for target subtree
    return ans;
}

// Compute total sum of tree
int sum(Node root) {
    if (root == null) return 0;
    return root.val + sum(root.left) + sum(root.right);
}

// DFS: compute subtree sums, mark ans=true if any equals target
int sum2(Node root, int target) {
    if (root == null) return 0;

    int lSum = sum2(root.left,  target);
    int rSum = sum2(root.right, target);

    // Check left or right subtree sum — NOT the whole tree (skip root)
    if (lSum == target || rSum == target)
        ans = true;

    return root.val + lSum + rSum;
}
Time: O(N)
Space: O(Ht)
Why check lSum and rSum (not root sum)? We check the subtree sums of left and right children — not the sum including root — because removing an edge means cutting above one subtree. The full tree's sum would always equal totalSum (the root's own subtree), which is not a valid split.
02

Path Sum

Problem Statement Given a binary tree and an integer k, determine if there exists a root-to-leaf path such that the sum of all node values along that path equals k.

Examples

Tree: 5 → (4 → (11 → (7, 2)), 8 → (13, 4 → 1))

k = 22: Path 5→4→11→2 = 22 → true

k = 20: No root-to-leaf path sums to 20 → false

Approach — Subtract & Recurse

Instead of accumulating sum downwards, subtract the current node's value from the target and recurse. At a leaf, check if the remaining target equals zero (meaning this path used up exactly k).

🚫
If root == null → return false
🍃
Leaf check: If left == null AND right == null AND target == root.val → return true
🔁
Recurse left and right with target - root.val. Return lAns || rAns.
Watch Out — Incorrect Base Case Do NOT return target == 0 when root == null. This fails because a null node is not a leaf — it would match paths that end at non-leaf nodes. Always check the leaf condition explicitly.
java 
boolean hasPath(Node root, int target) {
    // Null node — not a valid leaf, return false
    if (root == null) return false;

    // Leaf node reached — check if this path sums to target
    if (root.left == null && root.right == null && target == root.val)
        return true;

    // Recurse left and right with reduced target
    boolean lAns = hasPath(root.left,  target - root.val);
    boolean rAns = hasPath(root.right, target - root.val);

    return lAns || rAns;
}
Time: O(N)
Space: O(Ht)
Trace — k=22, tree root=5 hasPath(5, 22) → hasPath(4, 17) → hasPath(11, 13) → hasPath(7, 2): 7≠2 leaf false; hasPath(2, 2): leaf, 2==2 → true!
03

Check if Binary Tree is Balanced

Definition A binary tree is height-balanced if for every node, the absolute difference between heights of left and right subtrees is ≤ 1.

|height(left) - height(right)| ≤ 1 must hold for ALL nodes.

Naive Approach — O(N²)

For each node, compute heights of left and right subtrees separately, check balance, recurse. This recomputes heights redundantly → O(N²).

Optimal Approach — Single DFS O(N)

Combine height computation with balance checking in a single DFS. Use height = -1 as a sentinel to signal "unbalanced".

📐
If root == null → return height -1 (base: null nodes have height -1)
🔁
Compute lh = height(root.left) and rh = height(root.right)
⚖️
If |lh - rh| > 1 → set ans = false (but continue to return height)
📏
Return Math.max(lh, rh) + 1
java 
boolean ans = true;

boolean isBalanced(Node root) {
    height(root);
    return ans;
}

int height(Node root) {
    if (root == null) return -1;  // null has height -1

    int lh = height(root.left);
    int rh = height(root.right);

    if (Math.abs(lh - rh) > 1)
        ans = false;  // found an unbalanced node

    return Math.max(lh, rh) + 1;
}
Time: O(N)
Space: O(Ht)
Dry Run — Unbalanced Tree: 1→(2→(4→6), 3) height(6)=-1+1=0, height(4)=max(-1,0)+1=1, height(2)=max(1,-1)+1=2
height(3)=-1+1=0, height(1)=max(2,0)+1=3
At node 2: lh=1, rh=-1, |1-(-1)|=2 > 1 → ans=false
Why height of null = -1? With this convention, a single leaf node has height 0 (max(-1,-1)+1=0), which matches the standard definition. Alternatively use height=-1 for null so that a leaf's left and right subtrees (-1 each) differ by 0, which is ≤ 1.
04

Construct Binary Tree from Inorder + Postorder

Problem Statement Given two arrays representing the inorder and postorder traversals of a binary tree (with unique values), reconstruct the original tree.

Key Insight

Two traversals together uniquely determine a binary tree (when all values are unique):

TraversalWhat It Tells Us
PostorderLast element is always the root of the current subtree
InorderRoot splits array into left subtree (elements before root) and right subtree (elements after root)

Algorithm

Example inorder = [4, 2, 7, 5, 1, 3, 6] (indices 0–6)
postorder = [4, 7, 5, 2, 6, 3, 1] (psi=0, pei=6)

Root = post[pei] = 1, idx in inorder = 4
Left subtree: in[0..3]=[4,2,7,5], post[0..3]=[4,7,5,2]
Right subtree: in[5..6]=[3,6], post[4..5]=[6,3]

Index Formulas

Given: isi (inorder start), iei (inorder end), psi (postorder start), pei (postorder end), idx (root's index in inorder)

SubtreeInorder rangePostorder range
Left isi to idx - 1 psi to psi + idx - isi - 1
Right idx + 1 to iei psi + idx - isi to pei - 1
java 
Node construct(int[] in, int[] post,
               int isi, int iei, int psi, int pei) {

    if (isi > iei || psi > pei) return null;

    // Single element: leaf node
    if (isi == iei) return new Node(in[isi]);

    // Root is last element of postorder
    Node root = new Node(post[pei]);

    // Find root in inorder (can optimize with HashMap → O(1))
    int idx = -1;
    for (int i = isi; i <= iei; i++) {
        if (in[i] == post[pei]) { idx = i; break; }
    }

    // Build left subtree
    root.left  = construct(in, post, isi, idx - 1,
                            psi, psi + idx - isi - 1);

    // Build right subtree
    root.right = construct(in, post, idx + 1, iei,
                            psi + idx - isi, pei - 1);

    return root;
}

// Call: construct(inorder, postorder, 0, n-1, 0, n-1)
Time: O(N²)
Space: O(N)

Optimisation — HashMap for O(N) Time

Pre-build a map of value → index for the inorder array. Then the linear search for idx becomes O(1) at each call, reducing total time to O(N).

java 
// Pre-build: Map<value, inorder_index>
Map<Integer, Integer> inMap = new HashMap<>();
for (int i = 0; i < inorder.length; i++)
    inMap.put(inorder[i], i);

// Then replace the linear search with:
int idx = inMap.get(post[pei]);  // O(1) lookup
Time (optimised): O(N)
Space: O(N)
Memory Aid — Index Formula Derivation Left subtree has (idx - isi) nodes. In postorder, left subtree occupies indices psi to psi + (idx - isi) - 1. Right subtree then starts at psi + (idx - isi) and ends at pei - 1 (pei is the root).
