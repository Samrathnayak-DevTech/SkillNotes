Programming Paradigms

A programming paradigm is a style or standard way of writing a program — it is the underlying philosophy that dictates how you structure and think about code.

Without a paradigm, code will be… Less structured, hard to read and understand, hard to test, and difficult to maintain as it grows.

The two most common paradigms are Procedural (instructions executed in sequence) and Object-Oriented (data and behavior bundled into objects). Most modern languages — Java, Python, C++, C#, Kotlin — follow the OOP paradigm. OOP is also one of the most common interview topics.

Note In this module we cover the OOP concepts needed for Data Structures. OOP will be studied in much greater depth — including SOLID principles, design patterns, and hands-on exercises — in the Low-Level Design (LLD) module.
02

Classes & Objects

Without OOP, storing data about multiple students requires parallel arrays for name, marks, and address — hard to synchronize, error-prone when sorting, and brittle when adding new attributes. OOP solves this by grouping related data into a single unit.

Analogy A class is an architectural blueprint for a house — it specifies rooms, windows, and doors but is not a house itself. An object is an actual house built from that blueprint, occupying real land (memory).
📐

Class

A blueprint that defines what attributes (data members) and methods (functions) each object created from it will have. A class has no memory of its own until an object is created.

🏠

Object

A real instance (entity) of a class. Creating an object means allocating memory on the heap and running its constructor to initialize attributes.

Java Example

Java
class Book {
    String title;    // attributes / data members / properties
    String author;

    void read()   { System.out.println("Reading a book"); }
    void cover()  { System.out.println("Covering a book"); }
}

// Creating an object
Book java_b1 = new Book();  // java_b1 holds a reference (address on heap)
java_b1.title  = "Almanack";
java_b1.author = "Naval";
System.out.println(java_b1.title + " - " + java_b1.author);

Python Equivalent

Python
class Book:
    title  = ""
    author = ""

    def read(self):   return "Reading a book"
    def cover(self):  return "Covering a book"

python_b1 = Book()
python_b1.title  = "Almanack"
python_b1.author = "Naval"
03

Constructors

When you write new Student(), what executes? A special method called a constructor runs automatically. It has the same name as the class, no return type, and is responsible for initializing the object's attributes.

Default Constructor

If you do not write any constructor, the language provides a default constructor that takes no parameters and sets every attribute to its type's default value: 0 for int, null for String, 0.0 for double, false for boolean.

Important Rule The moment you define your own constructor, the language stops providing the default one. If you still need a no-argument constructor alongside a parameterized constructor, you must declare both explicitly.
Java — Default Constructor
class Student {
    String name;   // defaults to null
    int    marks;  // defaults to 0
    double psp;    // defaults to 0.0

    // Compiler provides:  Student() { this.name = null; this.marks = 0; ... }
}

Student s1 = new Student();  // s1 points to a heap object with default values
Student s2 = s1;              // SHALLOW COPY — s2 points to the SAME heap object
Student s3 = new Student();  // DEEP COPY scaffold — separate heap object

Parameterized Constructor

Accepts arguments at object-creation time so you can initialize attributes immediately, without calling setters afterward.

Java — Parameterized Constructor
class Student {
    String name;
    int    marks;
    double psp;

    Student(String n, int m, double p) {
        this.name  = n;
        this.marks = m;
        this.psp   = p;
    }
}

Student s1 = new Student("Jatin", 47, 73.5);
// Student() would now cause a compile error — no default constructor exists

04

The this Keyword (Java) / self (Python)

Inside a constructor or instance method, this (Java) or self (Python) refers to the current object — the specific instance on which the method was called. It is passed implicitly by the runtime and allows you to distinguish instance attributes from local parameters with the same name.

Java
class Student {
    String name;
    int    marks;
    double psp;

    double getNormalizedScore() {
        return this.marks * this.psp; // 'this' refers to the calling object
    }
}

Student s1 = new Student();
s1.name  = "Jatin";
s1.marks = 47;
s1.psp   = 73.5;
System.out.println(s1.getNormalizedScore()); // 'this' inside == s1
Python — Constructor with self
class Student: def __init__(self, name, age, psp, univ_name): self.name = name # self = current object self.age = age self.psp = psp self.univ_name = univ_name
05

Shallow Copy vs Deep Copy

Suppose you have a Student object and want to create a new, independent copy of it. There are two ways — one of which silently shares memory and can cause bugs.

🪞

Shallow Copy

Student s2 = s1; — Both variables now point to the same heap object. Changing s2.name also changes s1.name. Not a true independent copy.

📦

Deep Copy

Student s2 = new Student(s1.name, s1.marks, s1.psp); — A new heap object is created with all the same values. Changes to s2 do not affect s1.

Copy Constructor

A special constructor that takes another object of the same class and copies all its fields — a clean, reusable way to perform deep copies.

Java — Copy Constructor
class Student {
    String  name;
    int     age, marks;
    double  psp, attendance;
    boolean isActive;

    // Parameterized constructor
    Student(String n, int a, int m, double p, double att, boolean active) {
        this.name       = n;
        this.age        = a;
        this.marks      = m;
        this.psp        = p;
        this.attendance = att;
        this.isActive   = active;
    }

    // Copy constructor — accepts another Student object
    Student(Student o) {
        this.name       = o.name;
        this.age        = o.age;
        this.marks      = o.marks;
        this.psp        = o.psp;
        this.attendance = o.attendance;
        this.isActive   = o.isActive;
    }
}

Student s1 = new Student("Jatin", 27, 47, 90.5, 75, true);
Student s2 = new Student(s1);  // deep copy via copy constructor
Warning Never write s2 = s1 when you want an independent copy of an object in Java. You will get a shallow copy that shares the heap reference.
06

Linked List — Introduction

A linked list is a collection of nodes where each node holds a value and a reference (pointer) to the next node. Unlike arrays, elements are not stored in contiguous memory — each node can be anywhere on the heap.

The Node Class

Java — Node Definition
class Node {
    int  val;         // the data
    Node next;        // reference to the next Node (null for last node)

    Node(int val) {
        this.val  = val;
        this.next = null;  // initialized to null; wired later
    }
}

Building a List

Java
Node a    = new Node(10);   // a is at address 10K
Node head = a;              // head always points to the first node
Node b    = new Node(20);
a.next = b;                 // a.next.next = c also works
Node c    = new Node(30);
b.next = c;                 // c.next is still null — end of list

Why Use a Linked List Instead of an Array?

✅

Dynamic Size

No need to declare size upfront. Nodes are allocated on demand — memory grows and shrinks as needed.

✅

O(1) Insertion / Deletion

Adding or removing at the head (or at a known pointer position) only requires rewiring two pointers — no element shifting.

❌

No Random Access

To reach the Kth element you must traverse from the head — O(K) time, unlike O(1) array indexing.

❌

Extra Memory

Every node stores a pointer in addition to data, using more memory than a plain array of the same size.

07

Traversal & Accessing the Kth Node

Golden Rule Never move the head pointer. If you lose head, you lose the entire list. Always use a temporary pointer (temp) for traversal.

Printing All Elements

Java — Traversal
Node temp = head;             // temp is the cursor; head stays put
while (temp != null) {
    System.out.print(temp.val + " → ");
    temp = temp.next;
}
System.out.println("null");
Time O(N) Space O(1)

Accessing the Element at Index K

The next pointer of the last node points to null. The entire list is accessible only from the first node (head). To reach index K, traverse K steps from head:

Java — Get Kth Node
Node getKth(Node head, int k) {
    Node temp = head;
    for (int i = 0; i < k; i++) {
        if (temp == null) return null; // k out of bounds
        temp = temp.next;
    }
    return temp;
}
08

Quick Recap

Class vs Object

Class = blueprint (no memory). Object = instance (heap memory allocated on new).

Default Constructor

Compiler-provided; takes no args; sets defaults. Lost as soon as you write your own constructor.

this / self

Reference to the current object; passed implicitly. Disambiguates instance fields from local parameters.

Shallow Copy

s2 = s1 — two variables, one heap object. Changes through either affect both.

Deep Copy

new Student(s1) (copy constructor) — a separate heap object with copied values.

Linked List Node

A self-referential class: int val + Node next. Last node's next = null.

Never Move Head

Use a temp pointer for traversal. Losing head means losing the entire list.

Random Access Cost

Getting the Kth element is O(K) in a linked list vs O(1) in an array.
