Unique Element Among Pairs

Every element in the array appears exactly twice, except for one element which appears once. Duplicates are adjacent to each other, but the array is not sorted. Find the unique element.

Answer: 6

Brute Force

XOR all elements together. Because x ^ x = 0 for any x and x ^ 0 = x, all paired elements cancel out, leaving only the unique one. Time: O(N), Space: O(1).

Binary Search Approach O(log N)

The key observation: before the unique element, every pair starts at an even index. After the unique element, every pair starts at an odd index. Use this parity flip as the condition to halve the search space.

Key Insight At index mid, determine which of the two neighbors is the pair partner. If the pair occupies [mid, mid+1] (i.e., arr[mid] == arr[mid+1]), then mid is the first of a pair at an even index — the unique element is to the right. If the pair occupies [mid−1, mid], then mid is the second of a pair and the unique element is to the left.
Java
if (n == 1) return arr[0];
else if (arr[0] != arr[1]) return arr[0];
else if (arr[n-1] != arr[n-2]) return arr[n-1];

int l = 1, r = n - 2;
while (l <= r) {
    int mid = (l + r) / 2;
    if (arr[mid] != arr[mid-1] && arr[mid] != arr[mid+1]) {
        return arr[mid];        // unique found
    }
    int fo = mid;               // index of first occurrence in the pair
    if (arr[mid] == arr[mid-1]) fo = mid - 1;

    if (fo % 2 == 0) {
        l = mid + 1;            // pair starts at even → unique is to the right
    } else {
        r = mid - 1;            // pair starts at odd  → unique is to the left
    }
}
return -1;
Time O(log N) Space O(1)

Key Insight — Two Sorted Halves A rotated sorted array always contains one sorted half and one unsorted half. Determine which half arr[mid] falls in using arr[mid] ≥ arr[0] (first sorted half) or arr[mid] < arr[0] (second sorted half). Then check whether the target lies within the sorted half; if it does, recurse into it; otherwise, recurse into the other half.

Decision Logic at each step

✅

Match

arr[mid] == target → return mid.

➡️

mid in 1st half, target in 2nd

arr[mid] ≥ arr[0] and target < arr[0] → go right: l = mid + 1.

⬅️

mid in 2nd half, target in 1st

arr[mid] < arr[0] and target ≥ arr[0] → go left: r = mid − 1.

↔️

Both in same half

Apply standard binary search: if arr[mid] < target go right; else go left.

Java
int l = 0, r = n - 1;
while (l <= r) {
    int mid = (l + r) / 2;
    if (arr[mid] == target) return mid;

    // mid is in the first (left) sorted half
    if (arr[mid] >= arr[0]) {
        if (target < arr[0]) {          // target in 2nd half
            l = mid + 1;
        } else if (arr[mid] < target) {  // both in 1st, go right
            l = mid + 1;
        } else {                           // both in 1st, go left
            r = mid - 1;
        }
    } else {  // mid is in the second (right) sorted half
        if (target >= arr[0]) {          // target in 1st half
            r = mid - 1;
        } else if (arr[mid] < target) {  // both in 2nd, go right
            l = mid + 1;
        } else {                           // both in 2nd, go left
            r = mid - 1;
        }
    }
}
return -1;
Time O(log N) Space O(1)

Median of Two Sorted Arrays

Given two sorted arrays A (size N) and B (size M), find the median of the combined sorted array. A naïve merge takes O(N + M) time and O(N + M) space; the optimal solution uses binary search and runs in O(log(N + M)) time, O(1) space.

Intuition The median divides the combined array into two halves of equal size. We need to find a partition in A and a corresponding partition in B such that all elements in the left half are ≤ all elements in the right half.

Partition Idea

Let x = (N + M) / 2 be the number of elements we want on the left side. If we choose α elements from A on the left, we take x − α elements from B. We binary search on α (the number of elements from A that go left).

A: 1, 3, 4|7, 10, 12
B: 2, 3|6, 15

Left side: {1, 2, 3, 3, 4} — Right side: {6, 7, 10, 12, 15}. Median = (4 + 6) / 2 = 5.

Validity Condition

A partition is valid when: l₁ ≤ r₂ and l₂ ≤ r₁, where l₁, r₁ are the boundary elements of A's partition and l₂, r₂ are the boundary elements of B's partition.

Boundary Convention When a partition takes 0 elements from the left, use l = −∞. When it takes all elements, use r = +∞. These sentinel values ensure the validity check never throws an array out-of-bounds error.

Median Formula

Once a valid partition is found, let a = max(l₁, l₂) and b = min(r₁, r₂). If the combined length is even: median = (a + b) / 2.0. If odd: median = b (the min of the right boundary).

Java
double medianValue(int[] A, int[] B, int n, int m) {
    int x = (n + m) / 2;
    int l = 0, r = Math.min(n, x);

    while (l <= r) {
        int c1 = (l + r) / 2;      // elements from A on the left
        int c2 = x - c1;              // elements from B on the left

        long l1 = (c1 >= 1) ? A[c1-1] : Long.MIN_VALUE;
        long l2 = (c2 >= 1) ? B[c2-1] : Long.MIN_VALUE;
        long r1 = (c1 < n) ? A[c1]   : Long.MAX_VALUE;
        long r2 = (c2 < m) ? B[c2]   : Long.MAX_VALUE;

        if (l1 <= r2 && l2 <= r1) {   // valid partition
            if ((n + m) % 2 == 0)
                return (Math.max(l1, l2) + Math.min(r1, r2)) / 2.0;
            else
                return Math.min(r1, r2);
        } else if (l1 > r2) {
            r = c1 - 1;                // too many from A; go left
        } else {
            l = c1 + 1;                // too few from A; go right
        }
    }
    return -1;
}
Time O(log(N+M)) Space O(1)

Quick Recap

Unique in Pairs

Use parity of the first occurrence index to determine which side to search — pairs before the unique start at even indices.

Rotated Array Search

One half of a rotated sorted array is always fully sorted — identify it using arr[mid] ≥ arr[0], then decide which half to enter.

Median of Two Arrays

Binary search on the number of elements from A that belong on the left; complement the remaining from B. Check cross-boundary validity.

Sentinel Values

Use ±∞ sentinels when one array contributes 0 or N elements — prevents index errors and simplifies the validity check.
