NSL & NSR — Quick Revision

Two canonical monotone-stack patterns appear repeatedly in hard problems:

PatternIterateStack condition to popAnswer for index i
NSL — Nearest Smaller on LeftLeft → RightPop while arr[st.peek()] ≥ arr[i]st.isEmpty() ? −1 : st.peek()
NSR — Nearest Smaller on RightRight → LeftPop while arr[st.peek()] ≥ arr[i]st.isEmpty() ? N : st.peek()
NGL — Nearest Greater on LeftLeft → RightPop while arr[st.peek()] ≤ arr[i]st.isEmpty() ? −1 : st.peek()
NGR — Nearest Greater on RightRight → LeftPop while arr[st.peek()] ≤ arr[i]st.isEmpty() ? N : st.peek()
Example — NSL arr = [5, 4, 2, 7, 6, 9, 15, 3]
nsl = [−1, −1, −1, 2, 2, 4, 5, 2]
java — NSL (iterate left to right)
int[] nsl(int[] arr) {
    int n = arr.length;
    int[] ans = new int[n];
    Stack<Integer> st = new Stack<>();
    for (int i = 0; i < n; i++) {
        while (st.size() > 0 && arr[st.peek()] >= arr[i]) st.pop();
        ans[i] = st.isEmpty() ? -1 : st.peek();
        st.push(i);
    }
    return ans;
}

int[] nsr(int[] arr) {           // NSR — iterate right to left
    int n = arr.length;
    int[] ans = new int[n];
    Stack<Integer> st = new Stack<>();
    for (int i = n - 1; i >= 0; i--) {
        while (st.size() > 0 && arr[st.peek()] >= arr[i]) st.pop();
        ans[i] = st.isEmpty() ? n : st.peek();   // use n as sentinel
        st.push(i);
    }
    return ans;
}
// Both: T.C → O(N)   S.C → O(N)
Time: O(N)
Space: O(N)
02

Remove Equal Pairs of Consecutive Elements

Problem Statement Given a string, repeatedly remove any two adjacent identical characters until no more removals are possible. Return the resulting string.

Examples

InputStepsOutput
abcddremove ddabc
abcddcabcddc → abcc → abab
abbcbbcacxchains of removals → cxcx
abbbabbb → abab

Algorithm — Stack

Push characters one by one. If the character matches the stack's top, pop (removing the pair). Otherwise push. At the end, pop everything from the stack, reverse, and return.

java
String removeEqualPairs(String s) {
    Stack<Character> st = new Stack<>();

    for (int i = 0; i < s.length(); i++) {
        if (st.isEmpty() || st.peek() != s.charAt(i)) {
            st.push(s.charAt(i));
        } else {
            st.pop();               // remove the pair
        }
    }

    StringBuilder sb = new StringBuilder();
    while (!st.isEmpty()) sb.append(st.pop());
    sb.reverse();
    return sb.toString();
}
// T.C → O(N)   S.C → O(N)
Time: O(N)
Space: O(N)

Largest Rectangle Area in Histogram

Problem Statement Given an array of bar heights in a histogram, find the area of the largest rectangle that can be formed using consecutive bars. Each bar has width 1.

ans = 20 (bar at index 6, height 5 extending 4 bars wide)

Key Insight

For every bar i treated as the minimum height bar of the rectangle, the width extends from just after the nearest smaller bar on the left (NSL) to just before the nearest smaller bar on the right (NSR).

Formula Area for bar i = arr[i] × (nsr[i] − nsl[i] − 1)
Answer = max over all i.

Algorithm — O(N) using NSL + NSR

Compute nsl[] (nearest strictly-smaller index on left, default −1) and nsr[] (nearest smaller-or-equal index on right, default N) using two monotone-stack passes. Then compute the area for each bar and track the maximum.

Tie-breaking To handle duplicate heights correctly: for NSL use strict < (pop on ≥), for NSR use non-strict ≤ (pop on ≥ from right, sentinel N). This avoids double-counting rectangles of equal height.
java
public int largestRectangleArea(int[] arr) {
    int n = arr.length;
    int[] nsl = new int[n];
    int[] nsr = new int[n];

    // NSL — nearest smaller on left
    Stack<Integer> st = new Stack<>();
    for (int i = 0; i < n; i++) {
        while (st.size() > 0 && arr[st.peek()] >= arr[i]) st.pop();
        nsl[i] = st.isEmpty() ? -1 : st.peek();
        st.push(i);
    }

    // NSR — nearest smaller on right
    st = new Stack<>();
    for (int i = n - 1; i >= 0; i--) {
        while (st.size() > 0 && arr[st.peek()] >= arr[i]) st.pop();
        nsr[i] = st.isEmpty() ? n : st.peek();
        st.push(i);
    }

    // Compute max area
    int ans = 0;
    for (int i = 0; i < n; i++) {
        ans = Math.max(ans, arr[i] * (nsr[i] - nsl[i] - 1));
    }
    return ans;
}
// T.C → O(N)   S.C → O(N)
Time: O(N)
Space: O(N)

Sum of (Max − Min) for All Subarrays

Problem Statement Given an array of N integers, find the sum of (max − min) over all possible subarrays. Since the answer can be large, return it modulo 10⁹ + 7. (1 ≤ N ≤ 10⁵)

Example — arr = [1, 2, 3]

Subarraymaxminmax − min
[1]110
[1,2]211
[1,2,3]312
[2]220
[2,3]321
[3]330
Total4

Key Observation

Σ(max − min) = Σmax − Σmin. We need the total contribution of each element as a maximum and as a minimum across all subarrays.

Contribution Technique For element at index i, the number of subarrays where it is the maximum equals (i − ngl[i]) × (ngr[i] − i), where ngl = nearest greater on left and ngr = nearest greater on right.

Similarly, subarrays where it is the minimum = (i − nsl[i]) × (nsr[i] − i).
java
int sumSubarrayMaxMin(int[] arr) {
    int n = arr.length;
    long ans = 0;
    int mod = (int)1e9 + 7;

    int[] nsl = nsl(arr);  // nearest strictly smaller left  (default -1)
    int[] nsr = nsr(arr);  // nearest smaller-equal right   (default n)
    int[] ngl = ngl(arr);  // nearest strictly greater left  (default -1)
    int[] ngr = ngr(arr);  // nearest greater-equal right    (default n)

    for (int i = 0; i < n; i++) {
        long maxContrib = (long) arr[i] * (i - ngl[i]) * (ngr[i] - i);
        long minContrib = (long) arr[i] * (i - nsl[i]) * (nsr[i] - i);
        ans = (ans + (maxContrib - minContrib) + mod) % mod; // check
    }
    return (int) ans;
}
// T.C → O(N)   S.C → O(N)
Time: O(N)
Space: O(N)
Tie-breaking Rule To avoid double-counting when equal elements exist: for max — NGL strict (<), NGR non-strict (≤). For min — NSL strict (<), NSR non-strict (≤). Apply this consistently.
05

Quick Recap

NSL / NSR

Iterate left→right (NSL) or right→left (NSR). Pop while top ≥ current. O(N).

Remove Pairs

Push char if different from top; else pop. O(N) single pass.

Largest Histogram

Formula: arr[i] × (nsr[i] − nsl[i] − 1). Requires NSL + NSR. O(N).

Sum Max−Min

Contribution technique: count subarrays where each element is max/min. O(N).

Monotone Stack

Each element pushed and popped at most once → amortised O(N) total.

Tie-breaking

Strict vs non-strict comparisons in NSL/NSR avoid double-counting equal elements.
