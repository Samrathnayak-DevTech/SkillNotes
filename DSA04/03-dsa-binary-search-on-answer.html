Binary Search
on the Answer

When the answer space is monotone — feasible on one side, infeasible on the other — you can binary search the answer itself. Two canonical problems: minimizing maximum load and maximizing minimum distance.

01

The "Binary Search on Answer" Pattern

Many optimization problems ask for the minimum or maximum of some value subject to constraints. If the answer space is continuous and the feasibility function is monotone (once it becomes feasible, it stays feasible), binary search works on the answer itself rather than the array index.

The Three Ingredients (1) Target — the optimal answer (minimum time, maximum distance, etc.). (2) Search Space — the range of possible answers (often [max element, sum of all elements] or [1, max − min]). (3) Feasibility Condition — a function that checks: "is it possible to achieve answer ≤ mid (or ≥ mid)?"
Analogy You're buying a laptop and your budget ranges from ₹30 000 to ₹200 000. You want the best possible within budget. Instead of checking every price, you guess the midpoint, see if it buys what you need, and adjust — that is binary search on the answer.
02

Painter's Partition Problem

Given N boards with lengths arr[0..N−1] and P painters, find the minimum time to paint all boards, where each painter takes 1 unit of time per unit of board length, a board can only be painted by one painter, and each painter can only paint a contiguous segment of boards.

Example — arr = [5, 3, 6, 1, 9], P = 2

All possible ways to split between 2 painters and their costs:

Painter 1Painter 2Max (time taken)
[5][3, 6, 1, 9]max(5, 19) = 19
[5, 3][6, 1, 9]max(8, 16) = 16
[5, 3, 6][1, 9]max(14, 10) = 14 ✓ minimum
[5, 3, 6, 1][9]max(15, 9) = 15
[5, 3, 6, 1, 9][]max(24, 0) = 24

Binary Search Approach

The search space is [max(arr), sum(arr)]: the minimum possible time is the largest single board (if one painter per board), and the maximum is the total length (if one painter does all). For each candidate time mid, check if P painters can finish all boards within that time.

Monotone Property If P painters can finish within time T, they can also finish within time T+1. So the feasibility function is monotone-increasing. Binary search for the smallest T where it is feasible — go left when feasible, right when not.

isPossible Helper

Simulate greedily: assign boards to the current painter until adding the next board would exceed the time limit T — at that point, start a new painter. Count total painters needed; return true if ≤ P.

Java — isPossible
boolean isPossible(int[] arr, int p, int T) {
    int timeLeft = T, painters = 1;
    for (int i = 0; i < arr.length; i++) {
        if (arr[i] <= timeLeft) {
            timeLeft -= arr[i];          // current painter takes this board
        } else {
            painters++;                  // need a new painter
            timeLeft = T - arr[i];
        }
    }
    return painters <= p;
}   // T.C → O(N), S.C → O(1)
Java — Main Binary Search
int max = arr[0], sum = arr[0];
for (int i = 1; i < n; i++) {
    max = Math.max(arr[i], max);
    sum += arr[i];
}

int l = max, r = sum, ans = -1;
while (l <= r) {
    int mid = (l + r) / 2;
    if (isPossible(arr, p, mid)) {
        ans = mid;          // feasible; try smaller
        r = mid - 1;
    } else {
        l = mid + 1;      // not feasible; need more time
    }
}
return ans;
Time O(N · log(sum − max)) Space O(1)
Real-World Variant The same pattern solves "Split Array Largest Sum" (LeetCode 410) and the workload-balancing problem: given N emails with complexity scores and K handlers, divide them into K contiguous blocks minimizing the maximum block sum — identical to Painter's Partition with K = P.

Binary Search on Minimum Distance

The search space for the minimum distance is [1, arr[N−1] − arr[0]] (minimum adjacent gap to maximum possible span). For each candidate minimum distance mid, check if M cows can be placed such that every consecutive pair is at least mid apart.

Monotone Property If we can place M cows with minimum distance ≥ d, we can also place them with minimum distance ≥ d−1. So the feasibility is monotone-decreasing as d increases — binary search for the largest d where it is feasible. Go right when feasible, left when not.

isPossible Helper

Place the first cow at arr[0]. Greedily place each subsequent cow at the earliest stall that is at least dist away from the last placed cow. Count cows placed; return true if ≥ M.

Java — isPossible
boolean isPossible(int[] arr, int m, int dist) {
    int lastPos = arr[0], cowsPlaced = 1;
    for (int i = 1; i < arr.length; i++) {
        if (arr[i] - lastPos >= dist) {
            cowsPlaced++;
            lastPos = arr[i];
        }
    }
    return cowsPlaced >= m;
}   // T.C → O(N), S.C → O(1)
Java — Main Binary Search
int l = 1;
int r = arr[n-1] - arr[0];
// Narrow l to the minimum adjacent gap for tighter search
for (int i = 0; i < n-1; i++)
    l = Math.min(l, arr[i+1] - arr[i]);

int ans = -1;
while (l <= r) {
    int mid = (l + r) / 2;
    if (isPossible(arr, m, mid)) {
        ans = mid;          // feasible; try larger distance
        l = mid + 1;
    } else {
        r = mid - 1;      // distance too large; reduce
    }
}
return ans;
Time O(N · log(max distance)) Space O(1)

Common Observation

Both Painter's Partition and Aggressive Cows share the exact same algorithmic skeleton. The difference is only in the objective and the feasibility check direction.

PropertyPainter's PartitionAggressive Cows
ObjectiveMinimize maximum timeMaximize minimum distance
Search space[max element, sum of elements][1, arr[N−1] − arr[0]]
FeasibilityCan P painters finish in ≤ T?Can M cows maintain ≥ d gap?
On successans = mid; r = mid − 1 (go smaller)ans = mid; l = mid + 1 (go larger)
Graph shapeMonotone decreasing (more time ⇒ easier)Monotone decreasing (larger dist ⇒ harder)
Recognition Pattern Whenever a problem says "minimize the maximum" or "maximize the minimum" of something, binary search on the answer. Write an O(N) feasibility checker that tests a candidate answer, and wrap it in the binary search template.
05

Quick Recap

BS on Answer

Binary search over the answer space, not the array index, whenever feasibility is monotone.

Painter's Partition

Minimize max load. Search space = [max board, total length]. Go left on success.

Aggressive Cows

Maximize min distance. Search space = [1, max − min]. Go right on success.

Feasibility Checker

Always O(N) greedy simulation — that is what keeps the overall algorithm at O(N log answer).
